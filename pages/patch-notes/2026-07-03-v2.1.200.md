## 변경사항

- `AskUserQuestion` 대화를 더 이상 기본적으로 자동 계속하지 않도록 변경했으며, `/config`를 통해 유휴 timeout을 선택할 수 있습니다
- "default" permission 모드를 CLI, `--help`, VS Code, JetBrains 전체에서 "Manual"로 변경했습니다. `--permission-mode manual` 및 `"defaultMode": "manual"`은 `default` 외에도 허용됩니다
- `.claude.json`의 `disabledMcpServers` 또는 `enabledMcpServers`가 배열이 아닌 값으로 설정되었을 때 시작 시 충돌이 발생하는 문제를 수정했습니다
- sleep/wake 후 또는 stalled 세션을 다시 열 때 background 세션이 turn 중간에 자동으로 중지되는 문제를 수정했습니다
- stall respawn 후 Esc로 취소된 turn을 background 세션이 다시 실행하는 문제를 수정했습니다
- 충돌로 인해 stale `daemon.lock`이 남겨져 OS가 PID를 재사용한 후 background agent가 다시 시작되지 않는 문제를 수정했습니다
- background-agent daemon handover를 수정하여 재설치된 이전 빌드가 더 이상 daemon을 인수인계할 수 없도록 했습니다. 빌드 recency는 이제 version에 embedded된 build timestamp로 판단됩니다
- background-agent roster 문제를 수정했습니다: transient corruption이 orphan cleanup을 영구적으로 비활성화하는 문제, 이전 바이너리가 새 버전에 의해 작성된 필드를 보존하지 않는 문제, daemon restart 중 socket auth token이 제거되는 문제
- rate limit에 의해 text output을 생성하기 전에 차단된 subagent가 깔끔하게 실패하는 대신 빈 결과를 반환하는 문제를 수정했습니다
- background-agent 출력의 control byte가 agent view의 터미널에 도달하는 문제를 수정했습니다
- `--permission-mode manual` 플래그가 `agents` 뒤에 배치되었을 때 `claude agents --plugin-dir <dir>`이 agent view에서 plugin의 agent 및 skill을 표시하지 않는 문제를 수정했습니다
- project-scoped plugin이 같은 repository의 git worktree에서 올바르게 로드되지 않는 문제를 수정했습니다
- `/mcp` server 목록이 screen reader 및 magnifier의 focus를 추적하지 않는 문제를 수정했습니다
- voice dictation에서 recording이 audio를 캡처하지 못할 때 오해의 소지가 있는 "Voice connection failed" 메시지를 표시하는 문제를 수정했습니다
- tmux 3.4+ 이상에서 synchronized terminal output을 활성화하여 rendering flicker를 수정했습니다
- screen-reader 출력을 개선했습니다: decorative glyph는 이제 숨겨지고, transcript symbol은 짧은 레이블로 읽혀지며, nested table은 `Header: value.` 줄로 읽혀집니다
- install script를 개선하여 system의 memory 부족으로 설치가 중단되는 경우를 설명하도록 했습니다
