> 원문: [Claude Code v2.1.207 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.207)

## 변경사항

- Auto mode가 이제 Bedrock, Vertex AI, Foundry에서 `CLAUDE_CODE_ENABLE_AUTO_MODE` opt-in 없이 사용 가능합니다. 설정의 `disableAutoMode`를 통해 비활성화할 수 있습니다.
- 매우 긴 목록, 표, 단락 또는 코드 블록을 포함한 응답을 스트리밍하는 동안 터미널이 멈추고 키 입력이 지연되는 문제를 수정했습니다.
- 비대화형 실행(`claude -p`, SDK)에서 원격 관리 설정이 보안 동의 대화상자를 표시하지 않고 영구적으로 동의된 것으로 기록되는 문제를 수정했습니다.
- 양성 시스템 생성 대화 업데이트로 인해 트리거되는 거짓 prompt-injection 경고를 수정했습니다.
- Auto-updater가 릴리스할 때마다 `~/.local/bin/claude`의 사용자 정의 launcher 스크립트 또는 symlink를 덮어쓰는 문제를 수정했습니다. `/doctor`는 이제 외부에서 관리되는 launcher를 보고합니다.
- 유일한 출력 리디렉션이 `/dev/null`인 경우 `cd`가 포함된 복합 명령어가 권한을 요청하는 문제를 수정했습니다.
- 응답 스트리밍이 완료될 때 트랜스크립트가 답변의 시작 위로 점프하는 문제를 수정했습니다.
- 마지막 `worktree.sparsePaths` worktree가 제거된 후 `extensions.worktreeConfig`가 repo의 `.git/config`에 남아있어 go-git tools(`tea` 같은)를 중단시키는 문제를 수정했습니다.
- rules globs, skill paths, `.ignore`, `.worktreeinclude`의 잘못된 bracket 패턴이 파일 읽기, 파일 제안, worktree 생성을 중단시키는 문제를 수정했습니다.
- 잘못된 teammate mailbox 메시지로 인해 mailbox 파일을 수동으로 삭제할 때까지 매초 반복되는 오류가 발생하는 agent teams의 crash loop 문제를 수정했습니다.
- 계획을 수락하여 auto-named된 백그라운드 세션이 agent-view 행에 해당 이름을 표시하지 않는 문제를 수정했습니다.
- git worktree에 진입한 백그라운드 세션이 agent list에서 cold reopen 후 빈 상태로 재개되는 문제를 수정했습니다.
- 네트워크 인터럽션 또는 credential 새로고침에서 연결이 복구될 때 Remote Control 작업 상태 업데이트가 손실되는 문제를 수정했습니다.
- 데스크탑 app에서 호스트된 Remote Control 세션이 모바일 및 웹에서 background agent 및 workflow 진행률을 표시하지 않는 문제를 수정했습니다.
- Deep research 실행이 모든 Fetch-phase agent를 "unknown"으로 라벨링하는 문제를 수정했습니다. 이제 칩은 소스 hostname을 표시합니다.
- Bedrock이 모든 API 요청에서 IAM Identity Center의 AWS SSO credentials를 반복적으로 요청하는 문제를 수정했습니다.
- Agent view 개선: 동일한 텍스트를 다시 붙여넣으면 이제 축소된 `[Pasted text #N]` placeholder를 확장하여 두 번째를 추가하지 않습니다.
- Agent view 개선: 차단된 세션 peeks가 이제 질문으로 시작하고 동일한 타임스탐프 대신 표현된 staleness clock(`waiting 3m`)을 표시합니다.
- Bedrock, Vertex, Claude Platform on AWS가 기본값을 Claude Opus 4.8로 변경했습니다.
- Auto mode가 더 이상 `.claude/settings.local.json`(repo-resident)에서 `autoMode`를 읽지 않습니다. 대신 `~/.claude/settings.json`을 사용하세요.
- Windows에서 AWS credential 해결이 stall될 때(예: 멈춘 `credential_process`) 무한 hang이 발생하는 문제를 수정했습니다. 60초 stall guard가 이제 영구 대기 대신 작동합니다.
- Plugin hooks/monitors/MCP headersHelper: shell-form 명령어의 `${user_config.*}`가 이제 거부됩니다(shell-injection 수정). Hooks: exec form(`args` 배열) 또는 `$CLAUDE_PLUGIN_OPTION_<KEY>`를 사용하세요. monitors 및 headersHelper: 스크립트 내부에서 값을 읽으세요(config 파일 또는 server의 `env` 블록).
- Plugin option values(`pluginConfigs`)가 더 이상 project-level `.claude/settings.json`에서 읽혀지지 않습니다. user, `--settings`, managed settings만 적용됩니다.
- `/usage-credits` amount 입력이 더 이상 잘못된 값(예: 붙여넣은 timestamp)을 자동으로 숫자로 제거하지 않습니다. 잘못된 amount는 이제 오류로 거부되며, $1,000를 초과하는 amount는 입력된 확인이 필요합니다.
