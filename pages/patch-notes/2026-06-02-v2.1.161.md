## 변경사항

- `OTEL_RESOURCE_ATTRIBUTES` 값이 이제 metric datapoint의 레이블로 포함되므로 team 또는 repo와 같은 사용자 지정 차원으로 사용량 metrics를 분할할 수 있습니다
- `claude agents` 행은 이제 work가 fan out될 때 detail 앞에 `done/total`을 표시합니다. peek는 가장 오래 실행 중인 항목을 표시합니다
- `/mcp`는 이제 로그인한 적이 없는 claude.ai connector를 "Show unused connectors" 행 뒤에 축소합니다
- Parallel tool calls: 실패한 Bash 명령어는 더 이상 같은 batch의 다른 호출을 취소하지 않습니다 — 각 tool은 자체 결과를 독립적으로 반환합니다
- Fullscreen mode: clipboard는 이제 Linux에서 사용 가능할 때 `wl-copy`/`xclip`/`xsel`을 사용하고, clipboard와 PRIMARY selection 모두에 복사하며, "hold {key} for native selection" hint는 이제 terminal별로 올바른 key를 표시합니다
- `/effort` dialog, workflow animations, prompt keyword shimmer가 "Reduce motion" 설정을 무시하지 않도록 수정했습니다
- `forceLoginOrgUUID`/`forceLoginMethod` managed-settings policies가 org pin과 함께 third-party provider sessions (Bedrock, Vertex, Foundry, Mantle)를 차단하는 문제를 수정했습니다 (2.1.146의 regression)
- `--output-format text` 또는 `json`을 사용할 때 background subagent output이 `claude -p` stdout을 손상시키는 문제를 수정했습니다
- `/usage-credits`가 organization의 usage settings page를 가리키는 대신 Team 및 Enterprise admin에 대해 re-login을 시작하는 문제를 수정했습니다
- `/autofix-pr`이 session이 git worktree 또는 다른 repository 내에 있을 때 "cannot run on the default branch"를 보고하는 문제를 수정했습니다
- `--resume` picker가 git worktree가 아닐 때 (예: jj workspaces) 현재 directory의 session을 표시하지 않는 문제를 수정했습니다
- bash를 명시적으로 호출하는 Windows hooks (예: `/usr/bin/bash script.sh`)가 "command not found" 또는 "cannot execute binary file"로 실패하는 문제를 수정했습니다
- OpenTelemetry log events (`user_prompt`, `api_request`, `tool_result`, `tool_decision`)가 telemetry initialization이 완료되기 전에 발행될 때 조용히 삭제되는 문제를 수정했습니다
- `claude mcp` list/get/add가 terminal에 secrets를 인쇄하는 문제를 수정했습니다: `${VAR}` 참조는 더 이상 확장되지 않으며, credential headers와 URL secrets는 redacted됩니다
- `isolation: "worktree"`로 spawned된 Workflow agents가 background sessions에서 자신의 worktree 내부 파일을 편집하는 것이 차단되는 문제를 수정했습니다
- `claude agents`에서 dispatched된 background sessions가 settings.json의 model 대신 daemon의 environment에서 stale model로 booting되는 문제를 수정했습니다
- session을 resume한 후 Write tool results를 rendering할 때 잠재적인 crash를 수정했습니다
- completed subagents가 결과를 finalize하는 동안 오류가 발생할 때 running으로 stuck되는 문제를 수정했습니다
- `CLAUDE_CODE_TMPDIR`이 deep path로 설정될 때 `$TMPDIR` 아래에서 Unix sockets를 bind하는 tools로부터의 `EADDRINUSE` errors를 수정했습니다
- layout engine의 JIT compilation profile을 stabilize하여 terminal rendering performance를 개선했습니다
- large file writes의 rendering performance를 개선했습니다
- [VSCode] garbled glyphs를 수정하기 위해 terminal GPU acceleration을 비활성화하거나 `/terminal-setup`을 실행할 것을 제안하는 tip을 추가했습니다
