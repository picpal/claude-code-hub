> 원문: [Claude Code v2.1.144 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.144)

## 변경사항

- `/resume`이 background 세션을 지원하도록 추가됨 — `claude --bg` 또는 agent view를 통해 시작된 세션이 이제 `bg`로 표시되며 대화형 세션 옆에 나타남
- Background subagent 완료 알림에 경과 시간이 추가됨 (예: "Agent completed · 3h 2m 5s")
- `/plugin` browse 및 discover pane이 플러그인이 마지막으로 업데이트된 시간을 표시하도록 변경됨
- `/model`이 현재 세션만 모델을 변경하도록 변경됨; 모델 선택기에서 `d`를 눌러 새 세션의 기본값을 설정할 수 있음
- CLI 전체에서 "extra usage"를 "usage credits"로 이름 변경; `/extra-usage`는 이제 `/usage-credits`임 (이전 이름도 작동함)
- `api.anthropic.com`에 접근할 수 없을 때 시작이 최대 75초 동안 지연되는 문제를 수정함 (captive portal, firewall, VPN 문제) — side-channel API 호출이 이제 15초 후 시간 초과됨
- 놓친 window-resize 이벤트 후 터미널 출력이 손상되는 문제를 수정함 (예: VS Code split-pane divider 드래그) — 이제 Ctrl+L을 요구하는 대신 다음 frame에서 자동으로 복구됨
- 매우 긴 세션에서 나타날 수 있었던 점진적 터미널 표시 손상 (낡은/손상된 glyph)을 수정함; 이제 터미널 크기 조정이나 재시작 시에만 제거됨
- VS Code에서 spinner animation color count를 줄임으로써 터미널 렌더링 결함을 감소시킴
- Full Disk Access로 보호된 폴더 아래에 프로젝트가 있을 때 macOS background 세션이 "exit 1 before init"으로 충돌하는 문제를 수정함 (v2.1.143에서 발생한 regression)
- 이미지 확장명이 내용과 일치하지 않는 파일을 읽을 때 복구할 수 없는 conversation을 수정함 (예: .png로 저장된 HTML) — 이제 텍스트로 폴백됨
- 검색 중 발생하는 tool 오류를 감소시킴: `head`/`tail` 파일 views가 이제 read-before-edit 체크를 만족하고, `egrep`, `fgrep`, `git grep`, 또는 `git diff`의 "no matches" 결과 (exit code 1)가 더 이상 command failure로 보고되지 않음
- worktree에 진입하거나 일부 background 세션에서 `/branch`가 "No conversation to branch" 오류로 실패하는 문제를 수정함
- AskUserQuestion notes field에서 Escape를 누르면 turn을 중단하는 대신 answer 선택으로 돌아가도록 수정함
- IDE model picker나 시작 후 `applyFlagSettings`를 통해 변경할 때 model 선택이 적용되지 않는 문제를 수정함
- Resumed 세션이 이제 다른 세션의 `/model` 선택을 선택하는 대신 사용하던 model을 유지함
- Bedrock 및 Vertex 사용자가 `/model` picker에서 "Opus (1M context)"를 선택할 수 없는 문제를 수정함 (v2.1.129에서 발생한 regression)
- `forceLoginMethod` 및 `forceLoginOrgUUID`가 설정된 사용자의 원격 세션 로그인이 "Can't access this organization"으로 실패하는 문제를 수정함
- paginated `tools/list` 응답이 있는 MCP 서버가 첫 페이지만 반환하고 tools를 자동으로 삭제하는 문제를 수정함
- 지원되지 않는 MIME types (예: SVG)의 MCP 이미지가 conversation을 끊는 문제를 수정함 — 이제 disk에 저장되고 tool result에서 참조됨
- skill directory 내에서 build가 실행될 때 file descriptor 고갈 문제를 수정함 — 비`.md` 파일이 더 이상 skill 재로드를 트리거하지 않음
- session title이 사용자의 첫 prompt 대신 plugin monitor 출력으로 생성되는 문제를 수정함
- Headless mode에서 Skill tool이 permission error로 실패하는 문제를 수정함 (v2.1.141에서 발생한 regression)
- 자신의 설정에서 활성화된 플러그인이 fresh machine의 첫 로드 후 "not cached" 오류를 표시하는 문제를 수정함; 프로젝트의 `.claude/settings.json`에서만 활성화된 플러그인은 이제 실행 가능한 `claude plugin install` 힌트를 표시함
- `.mcp.json`을 파싱할 수 없을 때 `claude mcp list`가 자동으로 서버가 없다고 보고하는 문제를 수정함 (예: VS Code의 `"servers"` key를 `"mcpServers"` 대신 사용) — 이제 configuration 오류를 표시함
- custom `ANTHROPIC_BASE_URL` setups 및 Bedrock Mantle의 background side-queries가 Haiku를 사용하지 않는 문제를 수정함 — 이제 first-party API key가 설정되었거나 Haiku model이 설정되지 않았을 때 올바르게 폴백됨
- Windows에서 attached background 세션의 스크롤 문제를 수정함 — PgUp/PgDn, mouse wheel, 및 Ctrl+O transcript navigation이 이제 작동함
- background 세션에 attached 상태에서 터미널을 닫을 때 발생하는 crash를 수정함
- `! <cmd>` exec 세션이 attached 상태에서 Ctrl+C에 응답하지 않는 문제를 수정함 — 이제 실행 중인 command를 중단함
- Agent view shell-command rows가 완료 후에도 Working 아래에 남아있고, 완료된 row에서 Enter를 누르면 출력이 만료된 후 command가 재실행되는 문제를 수정함
- Windows에서 `claude agents`의 ← 키를 누르면 list가 keyboard input에 응답하지 않는 문제를 수정함
- Windows Terminal의 CJK 내용으로 Agent View의 pane 전환 시 왼쪽 가장자리에 ghost characters가 나타나는 문제를 수정함
- `/bg` 및 `←`-detach이 이제 `/add-dir`을 통해 추가된 directories를 유지함
- 이미 editing in place 중이던 세션을 detach한 직후 Edit/Write가 "background session hasn't isolated its changes yet"으로 거부하는 문제를 수정함
- 중지된 background 세션의 `claude respawn <id>`가 "running" 대신 "stopped"를 표시하는 문제를 수정함
- `/resume` picker가 background 세션에서 forked된 sessions를 표시하지 않는 문제를 수정함
- background service가 응답하지 않을 때 `claude agents`에서 session을 열거나 `claude logs <id>`를 실행하면 hang되는 문제를 수정함 — 이제 10초 후 복구 힌트와 함께 시간 초과됨
- Subagents가 spawn한 background Bash tasks가 프로세스 종료 후에도 SDK task panels에서 "Running"으로 유지되는 문제를 수정함
- 완료되거나 중지된 background 세션이 wake에 실패하면 startup crash로 영구적으로 표시되는 문제를 수정함
- `claude agents` attached 세션의 markdown links가 clickable hyperlinks 대신 plain text로 렌더링되는 문제를 수정함
- Custom `spinnerVerbs`가 turn 후 duration message에 적용되는 문제를 수정함 — "Worked for 5s"와 같은 과거형 built-ins이 복구됨
- `claude agents` / `--bg` rejection messages가 generic message 대신 특정 gate (non-TTY, env var, 또는 setting)를 이름으로 표시함
- `claude --bg --name <label>`이 이제 post-spawn confirmation에서 이름을 표시함
- `claude agents`: background 세션을 Ctrl+R로 이름 변경하면 attached 세션의 banner를 즉시 업데이트함
- Background session worktree isolation guard가 이제 `WorktreeCreate` hooks가 설정된 non-git VCS 사용자에게 적용됨
- Plugin marketplace add/update가 이제 `CLAUDE_CODE_PLUGIN_PREFER_HTTPS`를 준수함
- `/plugin`이 플러그인을 활성화, 비활성화, 또는 제거한 후 설치된 list로 돌아감
- `/doctor`가 command hook에 `command` field가 없을 때 exec-form 예제를 표시함
- Skill-listing truncation이 더 이상 startup notification으로 표시되지 않음 — full breakdown은 `/doctor`를 실행할 것
- 드문 pre-response stream stalls에서의 복구가 개선됨 — 느린 non-streaming request로 폴백하는 대신 streaming을 한 번 재시도함
- SDK/headless MCP 시작이 개선됨: pre-wait가 첫 turn 전에 startup을 차단하는 대신 겹치므로 (slow MCP 서버로 최대 2초 빠름)
- Post-survey follow-up 힌트가 이제 context-aware copy와 함께 모든 non-dismiss survey 응답 후에 나타나므로 /feedback을 통해 더 많은 세부 사항을 공유하기가 더 쉬워짐
