> 원문: [Claude Code v2.1.80 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.80)

## 변경된 사항
- Claude.ai 사용량 제한 현황을 표시하기 위해 statusline 스크립트에 `rate_limits` 필드를 추가했습니다(`used_percentage` 및 `resets_at`을 포함한 5시간 및 7일 단위 창).
- `source: 'settings'` 플러그인 마켓플레이스 소스를 추가했습니다 — settings.json 파일 내에서 플러그인 항목을 직접 선언할 수 있습니다
- 파일 패턴 일치 기능 외에도 플러그인 팁에 CLI 도구 사용법 안내 기능을 추가했습니다
- 스킬 및 슬래시 명령어가 호출될 때 모델의 노력 수준을 재정의할 수 있도록 `effort` 프론트매터 지원을 추가했습니다.
- `--channels` 옵션 추가 (연구용 미리보기) — MCP 서버가 사용자의 세션으로 메시지를 전송할 수 있도록 합니다
- `--resume` 옵션 사용 시 병렬 도구 결과가 누락되던 문제를 수정했습니다. 이제 병렬 도구 호출이 포함된 세션에서는 `[Tool result missing]` 자리 표시자가 표시되는 대신 모든 tool_use/tool_result 쌍이 복원됩니다.
- 브라우저 외 TLS 지문에서 Cloudflare의 봇 탐지로 인해 발생하던 음성 모드 WebSocket 오류 문제를 수정했습니다.
- API 프록시, Bedrock 또는 Vertex를 통해 세분화된 도구 스트리밍을 사용할 때 발생하는 400 오류를 수정했습니다.
- 게이트웨이 및 타사 제공업체 배포 환경에서 정상적으로 작동할 수 없는 상황에서 `/remote-control`이 표시되던 문제를 수정했습니다.
- `/sandbox` 탭 전환 시 Tab 키나 방향 키가 반응하지 않던 문제를 수정했습니다
- 대규모 Git 저장소에서 `@` 파일 자동 완성 기능의 반응 속도 개선
- 상태 표시줄의 표시와 일치하도록, 현재 'auto'가 어떤 값으로 해결되는지 확인할 수 있도록 `/effort` 기능을 개선했습니다
- `/permissions` 기능 개선 — 이제 목록 내에서 Tab 키와 화살표 키를 사용하여 탭을 전환할 수 있습니다
- 백그라운드 작업 패널 개선 — 이제 왼쪽 화살표 버튼을 누르면 목록 보기에서 닫힙니다
- 두 단계의 과정을 거치는 대신 단일 `/plugin install` 명령어를 사용할 수 있도록 플러그인 설치 절차를 간소화했습니다
- 대용량 저장소의 시작 시 메모리 사용량이 감소했습니다(파일 25만 개 규모의 저장소에서 약 80MB 절감).
- 이전 세션에서 `remote-settings.json`이 캐시된 경우, 시작 시 관리 설정(`enabledPlugins`, `permissions.defaultMode`, 정책 세트 환경 변수)이 적용되지 않던 문제를 수정했습니다.
