> 원문: [Claude Code v2.1.147 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.147)

## 변경사항

- Workflow 도구가 추가되었습니다. 이는 결정론적 다중 에이전트 오케스트레이션을 위한 것이며 기본적으로 비활성화되어 있습니다 — `CLAUDE_CODE_WORKFLOWS=1`을 설정하여 활성화할 수 있습니다
- 고정된 백그라운드 세션(`claude agents`의 `Ctrl+T`)이 이제 유휴 상태일 때 활성 상태로 유지되며, Claude Code 업데이트를 적용하기 위해 제자리에서 재시작되고, 메모리 압박 상황에서도 고정되지 않은 세션 이후에만 제거됩니다
- `/simplify`를 `/code-review`로 이름을 변경했습니다. 이제 선택한 노력 수준에서 정확성 버그를 보고합니다(예: `/code-review high`). `--comment`를 전달하면 findings을 인라인 GitHub PR 주석으로 게시합니다. 기존의 cleanup-and-fix 동작은 제거되었습니다
- prototype-pollution 및 thenable 기반 탈출에 대해 REPL 및 Workflow 도구 샌드박스를 강화했습니다
- 자동 업데이트 개선: 일시적 네트워크 오류를 재시도하고, 실패 시 특정 오류 범주 및 OS 오류 코드를 보고하며, 업데이트 실패 시 현재 버전을 표시합니다
- 대용량 파일 편집에 대한 diff 렌더링 성능을 개선했습니다
- Prompt 기록이 더 이상 연속 중복 항목을 기록하지 않습니다 — 화살표 위로 프롬프트를 회수한 후 다시 제출해도 다른 복사본이 추가되지 않습니다
- 엔터프라이즈 로그인 제한(`forceLoginOrgUUID` 및 `forceLoginMethod` managed-settings)이 타사 제공자 및 API 키 세션에 대해 적용되지 않는 문제를 수정했습니다
- `!` 명령 출력에서 `&`가 `&amp;`로 표시되어 headless 머신에서 `gcloud auth login` 같은 명령의 URL을 복사-붙여넣기할 수 없는 문제를 수정했습니다
- headless/SDK 모드에서 알 수 없는 slash 명령이 자동으로 아무것도 하지 않는 문제를 수정했습니다 — 이제 오류 메시지를 표시합니다
- fullscreen 모드가 아닐 때 작은 터미널에서 `/help`가 손상된 탭 헤더를 렌더링하고 페이지당 한 명령만 표시하는 문제를 수정했습니다
- 단일 밑줄로 시작하는 이름을 가진 사용자 함수를 drop하는 shell snapshot 문제를 수정했습니다. 이는 이러한 함수를 참조하는 aliases를 손상시켰습니다
- `tools:` frontmatter에서 여러 `Agent(...)` 타입을 선언하는 plugin 에이전트가 마지막 항목을 제외한 모든 항목을 drop하는 문제를 수정했습니다
- `PowerShell(git push*)`와 같은 hook `if` 조건이 절대 일치하지 않는 문제를 수정했습니다 — 오직 `PowerShell(*)`만 작동했습니다
- PowerShell 도구가 기본 formatter에 의존하는 명령의 출력을 drop하는 문제를 수정했습니다
- 수정: Windows에서 PowerShell 스크립트 호출에 대해 "Yes, and don't ask again"을 선택하면 이제 후속 실행에서 실제로 일치하는 규칙을 작성합니다
- Windows에서 `pwsh`가 winget 또는 Microsoft Store를 통해 설치된 경우 PowerShell 도구가 종료 코드 1로 실패하는 문제를 수정했습니다
- `/effort`가 잘못된 수준의 슬라이더로 열리는 문제를 수정했습니다 — 이제 현재 노력 수준에서 시작합니다
- MCP 서버 페이지를 매기면서 page 1 이후의 resources, templates, prompts를 drop하는 문제를 수정했습니다
- Windows Terminal의 attached 백그라운드 세션에서 Claude가 streaming할 때 전체 화면이 깜빡이는 문제를 수정했습니다
- 수정: Windows에서 백그라운드 작업 worktree를 제거해도 더 이상 NTFS junctions이 main repo를 따라가지 않습니다
- `/background`가 유일한 typed input이 skill 또는 custom slash 명령인 세션을 거부하는 문제를 수정했습니다
- 자동 모드가 사용자 또는 skill이 명시적으로 의존하는 경우에 `AskUserQuestion`을 억제하는 문제를 수정했습니다. auto-mode classifier는 이제 사용자의 답변을 intent signal로 봅니다
- `/theme` "New custom theme" 및 color editor 대화 상자가 Esc에 응답하지 않는 문제를 수정했습니다
- Agent SDK를 통해 실행할 때 streaming 세션의 끝에서 발생하는 uncaught exception 문제를 수정했습니다
- Windows에서 scroll이 settle될 때까지 기다릴 때 발생하는 드문 hang 문제를 수정했습니다
- 백그라운드 세션 결과에 wide(CJK) 문자가 포함되어 있을 때 Windows의 agent view list에서 stale 및 doubled 행이 표시되는 문제를 수정했습니다
- 붙여넣은 텍스트가 에이전트에게 실제 콘텐츠 대신 읽을 수 없는 `[Pasted text #N]` placeholder로 전달되는 문제를 수정했습니다
- plugin의 manifest가 default directories와 겹치는 경로를 나열할 때 `claude plugin details` 및 `/plugin`의 plugin component 수가 doubled되는 문제를 수정했습니다
- backgrounded 세션이 "don't ask again"으로 이미 부여한 도구 권한에 대해 다시 묻는 문제를 수정했습니다
- GNOME Terminal 오른쪽 클릭 및 중간 클릭 붙여넣기가 텍스트를 삽입하지 않는 문제를 수정했습니다
- `CLAUDE_CODE_SUBAGENT_MODEL`이 agent teams에 의해 생성된 teammate 프로세스에 적용되지 않는 문제를 수정했습니다
- tab 또는 newline이 뒤따르는 slash 명령이 unknown 명령으로 처리되는 문제를 수정했습니다
- `/plugin`, `/status`, `/mobile`, `/sandbox`, `/permissions` 메뉴의 여러 spacing 및 layout glitches를 수정했습니다
- stripped images가 더 이상 존재하지 않는 media를 반복적으로 다시 읽도록 모델을 prompting하는 문제를 수정했습니다
