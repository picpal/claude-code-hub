## 변경사항

- `/color` (인자 없음)가 이제 무작위 세션 색상을 선택합니다
- `/mcp`가 이제 연결된 서버의 도구 개수를 표시하고 0개의 도구로 연결된 서버에 플래그를 지정합니다
- `--plugin-dir`이 이제 디렉토리 외에 `.zip` plugin 아카이브를 수락합니다
- `--channels`이 이제 console (API key) authentication과 작동합니다 — 관리되는 설정이 있는 console org는 이를 활성화하기 위해 `channelsEnabled: true`를 설정해야 합니다
- 업데이트된 `/model` picker: 중복 Opus 4.7 항목을 축약하고, 현재 Opus가 "Opus 4.7" 대신 "Opus"로 표시됩니다
- Subprocess (Bash, hooks, MCP, LSP)가 더 이상 `OTEL_*` 환경 변수를 상속받지 않으므로, Bash tool을 통해 실행되는 OTEL-instrumented 앱이 더 이상 CLI의 자체 OTLP endpoint를 수신하지 않습니다
- MCP: `workspace`는 이제 예약된 서버 이름입니다 — 해당 이름을 가진 기존 서버는 경고와 함께 건너뜁니다
- MCP 서버 재연결 시 더 이상 매 재연결마다 전체 tool-name 목록으로 대화를 범람시키지 않습니다 — 재공지된 도구는 서버 prefix별로 요약됩니다
- SDK hosts는 이제 Bash 권한 프롬프트에 대한 지속적인 `localSettings` 제안을 받으므로 "Always allow"가 `.claude/settings.local.json`에 기록됩니다
- `EnterWorktree`가 이제 문서화된 대로 local HEAD에서 새 branch를 생성합니다. `origin/<default-branch>` 대신 — 푸시되지 않은 커밋이 더 이상 삭제되지 않습니다
- Auto mode: classifier가 action을 평가할 수 없을 때, 오류에는 이제 힌트(retry, `/compact`, 또는 `--debug` 실행)가 포함됩니다
- Focus mode가 더 이상 새 프롬프트를 제출할 때 이전 응답을 간단히 어둡게 합니다
- Kitty 및 OSC 9를 notification으로 해석하는 다른 터미널에서 모든 `/exit`에 대해 stray "4;0;" desktop notification을 수정했습니다
- Remote Control이 rate limit 시 실행 가능한 upsell 옵션 대신 빈 "Opening your options…" 메시지를 표시하는 것을 수정했습니다
- 이미지 읽기가 실패할 때 drag-and-drop 이미지 업로드가 "Pasting text…"에서 대기하는 것을 수정했습니다
- stdin을 통해 `claude -p`로 매우 큰 입력(>10 MB)을 파이프할 때 crash loop를 수정했습니다
- fullscreen mode에서 모든 wrapped row의 긴 URL이 개별적으로 클릭 가능하지 않은 것을 수정했습니다
- `/plugin` Components panel이 `--plugin-dir`을 통해 로드된 plugin에 대해 "Marketplace 'inline' not found"를 표시하는 것을 수정했습니다
- MCP tool 결과가 서버가 structured content와 content blocks을 모두 반환할 때 이미지를 드롭하는 것을 수정했습니다
- 목록 항목 내의 fenced code blocks가 copy-paste에서 clipboard로 leading whitespace를 전달하는 것을 수정했습니다
- `/config`의 tab navigation이 focus를 stranding하는 것을 수정했습니다 — tab header가 이제 focused 상태로 유지되어 arrows와 Esc가 계속 작동합니다
- OSC 8 hyperlink support가 없는 터미널에서 markdown link labels이 손실되는 것을 수정했습니다 — 링크가 이제 URL만 아닌 `label (url)`로 렌더링됩니다
- 더 작은 autocompact window를 가진 1M-context 모델의 세션이 실제 API limit에 도달하기 전에 거짓으로 "Prompt is too long"으로 차단되는 것을 수정했습니다
- Parallel shell tool 호출: read-only 명령(grep, git diff, ls)이 실패할 때 더 이상 sibling calls를 취소하지 않습니다
- Banner가 effort를 지원하지 않는 모델에서 "with X effort"를 표시하는 것을 수정했습니다
- `/fast`가 3P provider에서 관련 없는 skill로 fuzzy-matching되는 것을 수정했습니다 — "not available"을 대신 표시합니다
- Bedrock default model이 region-appropriate prefix 대신 `global.*`로 resolve되는 것을 수정했습니다
- Vim mode: NORMAL mode의 `Space`가 이제 표준 vi/vim 동작과 일치하도록 커서를 오른쪽으로 이동합니다
- Terminal progress indicator (OSC 9;4)가 tool calls 사이에서 깜박거리는 것을 수정했습니다 — 전체 turn에 걸쳐 visible 상태를 유지합니다
- `/rename`이 인자 없이 마지막 항목이 compact boundary인 resumed 세션에서 실패하는 것을 수정했습니다
- Stale "remote-control is active" status lines이 `--resume`/`--continue` 후 이전 세션에서 나타나는 것을 수정했습니다
- Stale `installed_plugins.json` 항목이 삭제된 cache 디렉토리를 가리키며 PATH를 오염시키는 것을 수정했습니다
- MCP stdio servers가 `CLAUDE_CODE_SHELL_PREFIX`가 설정되고 인자에 spaces 또는 shell metacharacters이 포함될 때 corrupted 인자를 받는 것을 수정했습니다
- Sub-agent progress summaries가 prompt cache를 누락하는 것을 수정했습니다 (~3× `cache_creation` 감소)
- `/plugin update`가 npm-sourced plugin의 새 버전을 결코 감지하지 못하는 것을 수정했습니다
- Sub-agent summaries가 sub-agent의 transcript가 static인 동안 반복해서 발생하는 것을 수정했습니다. 유휴 sub-agent에 대한 worst-case token 비용을 제한합니다
- Headless `--output-format stream-json`: `init.plugin_errors`이 이제 dependency demotions 외에 `--plugin-dir` load failures를 포함합니다
