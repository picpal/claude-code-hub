> 원문: [Claude Code v2.1.117 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.117)

## 변경사항

- Forked subagents는 이제 `CLAUDE_CODE_FORK_SUBAGENT=1`을 설정하여 external builds에서 활성화할 수 있습니다
- Agent frontmatter `mcpServers`는 이제 `--agent`를 통해 main-thread agent sessions에 로드됩니다
- `/model` 개선: 선택 항목이 프로젝트가 다른 모델을 고정한 경우에도 재시작 후에도 유지되며, 시작 헤더는 활성 모델이 프로젝트 또는 managed-settings 핀에서 나온 경우를 표시합니다
- `/resume` 명령어는 이제 기존 `--resume` 동작과 일치하도록 stale하고 큰 sessions를 다시 읽기 전에 요약할 것을 제안합니다
- 로컬 및 claude.ai MCP servers가 모두 구성된 경우 더 빠른 시작 (concurrent connect가 기본값으로 적용됨)
- 이미 설치된 플러그인에서 `plugin install`은 이제 "already installed"에서 멈추는 대신 누락된 dependencies를 설치합니다
- Plugin dependency 오류는 이제 설치 힌트와 함께 "not installed"로 표시되며, `claude plugin marketplace add`는 이제 구성된 marketplaces에서 누락된 dependencies를 자동으로 해결합니다
- Managed-settings `blockedMarketplaces` 및 `strictKnownMarketplaces`는 이제 plugin install, update, refresh, 및 autoupdate에 적용됩니다
- Advisor Tool (실험적): 다이얼로그는 이제 "experimental" 레이블, learn-more 링크, 및 활성화 시 시작 알림을 표시합니다. sessions는 더 이상 모든 prompt 및 `/compact`에서 "Advisor tool result content could not be processed" 오류로 중단되지 않습니다
- `cleanupPeriodDays` retention sweep는 이제 `~/.claude/tasks/`, `~/.claude/shell-snapshots/`, 및 `~/.claude/backups/`도 포함합니다
- OpenTelemetry: `user_prompt` events는 이제 slash commands에 대해 `command_name` 및 `command_source`를 포함합니다. `cost.usage`, `token.usage`, `api_request`, 및 `api_error`는 이제 모델이 effort levels을 지원할 때 `effort` attribute를 포함합니다. Custom/MCP command 이름은 `OTEL_LOG_TOOL_DETAILS=1`이 설정되어 있지 않으면 redacted됩니다
- macOS 및 Linux의 native builds: `Glob` 및 `Grep` tools은 이제 Bash tool을 통해 사용 가능한 embedded `bfs` 및 `ugrep`으로 대체됩니다 — 별도의 tool round-trip 없이 더 빠른 검색 (Windows 및 npm-installed builds는 변경 없음)
- Windows: 더 빠른 subprocess 시작을 위한 프로세스당 cached `where.exe` executable lookups
- Opus 4.6 및 Sonnet 4.6의 Pro/Max 구독자에 대한 기본 effort는 이제 `high`입니다 (이전: `medium`)
- Plain-CLI OAuth sessions가 access token이 session 중간에 만료될 때 "Please run /login"으로 종료되는 문제 수정 — 이제 token이 401에서 reactively하게 새로고쳐집니다
- `WebFetch`가 매우 큰 HTML pages에서 hanging되는 문제 수정 — HTML-to-markdown 변환 전에 input을 truncating합니다
- proxy가 HTTP 204 No Content를 반환할 때 crash되는 문제 수정 — 이제 `TypeError` 대신 명확한 오류를 표시합니다
- `CLAUDE_CODE_OAUTH_TOKEN` env var로 시작했을 때 `/login`이 효과가 없고 그 token이 만료되는 문제 수정
- prompt-input undo (`Ctrl+_`)가 입력 직후에 아무것도 하지 않고 각 undo 단계에서 상태를 건너뛰는 문제 수정
- `NO_PROXY`가 Bun 아래에서 실행될 때 remote API requests에 대해 존중되지 않는 문제 수정
- 느린 연결을 통해 key 이름이 coalesced text로 도착할 때 rare spurious escape/return triggers 문제 수정
- SDK `reload_plugins`가 모든 user MCP servers를 serially하게 reconnecting하는 문제 수정
- Bedrock application-inference-profile requests가 thinking disabled된 Opus 4.7로 지원될 때 400으로 실패하는 문제 수정
- MCP `elicitation/create` requests가 print/SDK mode에서 server가 mid-turn에 연결을 마칠 때 auto-cancelling되는 문제 수정
- subagents가 main agent와 다른 모델을 실행할 때 파일 읽기를 malware warning으로 잘못 표시하는 문제 수정
- background tasks가 있을 때 idle re-render loop 수정 — Linux에서 memory growth를 줄임
- [VSCode] 여러 개의 큰 marketplaces가 구성될 때 "Manage Plugins" panel이 breaking되는 문제 수정
- Opus 4.7 sessions가 inflated `/context` percentages를 표시하고 자동으로 너무 일찍 autocompacting되는 문제 수정 — Claude Code가 Opus 4.7의 native 1M context window 대신 200K context window에 대해 계산하고 있었습니다
