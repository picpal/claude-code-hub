> 원문: [Claude Code v2.1.169 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.169)

## 변경사항

- `--safe-mode` 플래그(및 `CLAUDE_CODE_SAFE_MODE`)가 추가되어 문제 해결을 위해 모든 커스터마이제이션(CLAUDE.md, 플러그인, 스킬, 훅, MCP 서버)을 비활성화한 상태로 Claude Code를 시작할 수 있습니다
- `/cd` 명령어가 추가되어 세션 중간에 prompt cache를 손상시키지 않고 새로운 작업 디렉토리로 세션을 이동할 수 있습니다
- `disableBundledSkills` 설정 및 `CLAUDE_CODE_DISABLE_BUNDLED_SKILLS` 환경 변수가 추가되어 번들된 스킬, 워크플로우, 내장 slash 명령어를 모델에서 숨길 수 있습니다
- 길게 감싼 입력 줄의 명령어 히스토리를 넘어 Up/Down 화살표가 점프하는 문제가 수정되었습니다 — 이제 각 시각적 행을 먼저 이동하고 히스토리 회수는 가장자리에서 시작합니다
- enterprise managed MCP 정책(`allowedMcpServers`/`deniedMcpServers`)이 재연결, IDE 유형 설정, 설치 후 첫 세션 중 `--mcp-config` 서버, 또는 원격 설정이 로드되기 전에 적용되지 않는 문제가 수정되었습니다. 또한 원격 설정이 없는 조직의 느린 콜드 스타트도 수정되었습니다
- macOS 사용자가 claude.ai 자격증명으로 로그인했을 때 각 차례 시작 시 ~30-50ms UI 정지 현상이 수정되었습니다
- Windows에서 `claude -p`가 느리거나 slash 명령어/스킬 스캔을 기다리는 동안 응답 없음 상태로 보이는 문제가 수정되었습니다(2.1.161의 회귀)
- OAuth 토큰 새로 고침이 동시에 발생했을 때 세션을 재개한 후 Remote Control이 "재연결 중"에 멈추는 문제가 수정되었습니다
- Windows 시작 시 캐시된 자격증명 없이 백그라운드 git 명령어가 실행될 때 Git Credential Manager의 "GitHub에 연결" 팝업이 나타나는 문제가 수정되었습니다
- 커스텀 statusline이 있는 사용자에 대해 footer 힌트(예: "esc to interrupt")가 표시되지 않는 문제가 수정되었습니다
- 워커가 기다리는 동안 죽은 원격 세션에 재연결할 때마다 오래된 권한 및 대화상자 프롬프트가 다시 나타나는 문제가 수정되었습니다
- `claude agents --json`이 차단되고 방금 발송된 백그라운드 세션을 생략하는 문제가 수정되었습니다. 완료된 세션을 포함하기 위해 `--all`이 추가되었고, 새로운 `id` 및 `state` 필드가 추가되었습니다
- WSL의 Windows Terminal에서 에이전트에서 돌아간 후 에이전트 뷰가 오래된/손상된 프레임을 남기는 문제가 수정되었습니다
- 백그라운드 에이전트가 미리 준비된 워커에 발송될 때 프로젝트 수준 설정 `env` 값(예: `ANTHROPIC_MODEL`)을 무시하는 문제가 수정되었습니다
- Windows에서 MCPB 플러그인 캐시가 거짓으로 무효화되어 불필요한 재추출이 발생하는 문제가 수정되었습니다
- 플러그인 `.in_use` PID 잠금 파일이 무한정 누적되는 문제가 수정되었습니다. 충돌한 세션의 오래된 마커는 이제 하루에 한 번씩 제거됩니다
- 신뢰할 수 없는 프로젝트 설정이 신뢰 확인 없이 OTEL 클라이언트 인증서 경로를 설정할 수 있는 문제가 수정되었습니다
- `/workflows`가 차례가 진행 중인 동안에도 즉시 열립니다
- `TaskCreate` 신뢰성이 개선되었습니다: 잘못된 형식의 입력이 자동으로 복구되고 로드되지 않은 도구의 검증 오류에 스키마가 포함됩니다
- 조직이 API 키 인증을 비활성화했을 때 표시되는 오류 메시지가 개선되었으며, 활성 API 키의 출처를 기반으로 한 안내가 포함됩니다
- 응답이 스트림되고 spinner 애니메이션이 진행되는 동안 CPU 사용량이 감소합니다
- Vertex/Foundry의 기본 5분 유휴 시간 초과가 복원되어 정지된 스트림이 무한정 중단되지 않고 중단됩니다. `API_FORCE_IDLE_TIMEOUT=0`으로 설정하여 거부할 수 있습니다
- 잘못된 항목이 있는 원격 관리 설정은 이제 나머지 유효한 정책을 적용하고 검증 오류를 표시하며, 전체 페이로드를 조용히 삭제하지 않습니다
- 백그라운드 세션은 이제 retire→wake 전반에 걸쳐 `--ide`, `--chrome`, `--bare`, `--remote-control` 및 기타 플래그를 유지하고, respawn 상태 검증이 강화되었습니다
- 백그라운드 세션은 shared-checkout 편집이 워크트리에 들어갈 때까지 차단되었음을 알려져 `EnterWorktree` 전에 낭비되는 거부된 편집을 피합니다
- "CLAUDE.md는 너무 깁니다" 경고 임계값이 이제 모델의 context window에 따라 확장됩니다
- Windows의 자동 업데이터는 이제 `claude.exe`가 다른 프로세스에 의해 보유된 후 세션 내에서 재시도를 중지합니다
- slash 명령어 메뉴의 스킬 태그의 색상 대비가 개선되었습니다
- Apple/Google 청구 구독자의 결제 수단이 없는 프로모 크레딧 청구가 이제 결제 수단을 추가할 위치를 설명합니다
- 여러 동시 세션을 실행할 때 `claude agents`를 제안하는 팁이 추가되었습니다
