> 원문: [Claude Code v2.1.110 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.110)

## 변경사항

- `/tui` 명령어 및 `tui` 설정을 추가했습니다 — `/tui fullscreen`을 실행하여 동일한 대화에서 깜빡임이 없는 렌더링으로 전환할 수 있습니다
- `Ctrl+O`를 normal과 verbose transcript 간의 토글로 변경했습니다; focus view는 이제 새로운 `/focus` 명령어로 별도로 토글됩니다
- `autoScrollEnabled` config를 추가하여 fullscreen 모드에서 대화 자동 스크롤을 비활성화할 수 있습니다
- `Ctrl+G` external editor에서 Claude의 마지막 응답을 주석 처리된 context로 표시하는 옵션을 추가했습니다 (`/config`를 통해 활성화)
- `/plugin` Installed 탭을 개선했습니다 — 주의가 필요한 항목과 즐겨찾기가 상단에 표시되고, 비활성화된 항목은 fold 뒤에 숨겨지며, `f`로 선택된 항목을 즐겨찾기에 추가합니다
- `/doctor`를 개선하여 MCP server가 다른 endpoint를 가진 여러 config scope에 정의되어 있을 때 경고를 표시합니다
- `--resume`/`--continue`는 이제 만료되지 않은 예약된 작업을 복구합니다
- `/autocompact`, `/context`, `/exit`, `/reload-plugins`는 이제 Remote Control (mobile/web) 클라이언트에서 작동합니다
- Write tool은 이제 IDE diff에서 제안된 content를 수락하기 전에 편집할 때 모델에 알립니다
- Bash tool은 이제 문서화된 최대 timeout을 시행하며 임의로 큰 값을 허용하지 않습니다
- SDK/headless 세션은 이제 환경에서 `TRACEPARENT`/`TRACESTATE`를 읽어 분산 trace linking을 수행합니다
- Session recap은 이제 telemetry가 비활성화된 사용자(Bedrock, Vertex, Foundry, `DISABLE_TELEMETRY`)에 대해 활성화됩니다. `/config` 또는 `CLAUDE_CODE_ENABLE_AWAY_SUMMARY=0`을 통해 opt out할 수 있습니다
- SSE/HTTP transports에서 server connection이 response 중간에 끊어질 때 MCP tool call이 무한정 행(hang)하는 문제를 수정했습니다
- API에 연결할 수 없을 때 non-streaming fallback retries가 수 분 동안 행(hang)하는 문제를 수정했습니다
- Session recap, local slash-command output, 기타 시스템 상태 라인이 focus mode에서 표시되지 않는 문제를 수정했습니다
- Fullscreen에서 tool이 실행되는 동안 text가 선택되었을 때 높은 CPU 사용량 문제를 수정했습니다
- Plugin install이 marketplace entry에서 생략된 `plugin.json`에서 선언된 의존성을 준수하지 않는 문제를 수정했습니다; `/plugin` install은 이제 자동 설치된 의존성을 나열합니다
- `disable-model-invocation: true`를 가진 skills이 message 중간에 `/<skill>`을 통해 호출될 때 실패하는 문제를 수정했습니다
- `--resume`가 여전히 실행 중이거나 비정상적으로 종료된 세션에서 `/rename` 이름 대신 첫 번째 prompt를 표시하는 경우가 있는 문제를 수정했습니다
- 대기 중인 메시지가 multi-tool-call turns 중에 잠깐 두 번 표시되는 문제를 수정했습니다
- Session cleanup이 subagent transcripts를 포함한 전체 session directory를 제거하지 않는 문제를 수정했습니다
- CLI가 재시작된 후(예: `/tui`, provider setup wizards) 키 입력이 손실되는 문제를 수정했습니다
- macOS Terminal.app 및 synchronized output를 지원하지 않는 기타 terminals에서 깨진 시작 렌더링 문제를 수정했습니다
- 신뢰할 수 없는 파일명에서의 command injection으로부터 "Open in editor" 작업을 강화했습니다
- `PermissionRequest` hooks이 `updatedInput`를 반환할 때 `permissions.deny` rules에 대해 다시 확인되지 않는 문제를 수정했습니다; `setMode:'bypassPermissions'` updates는 이제 `disableBypassPermissionsMode`를 준수합니다
- `PreToolUse` hook의 `additionalContext`이 tool call이 실패했을 때 제거되는 문제를 수정했습니다
- Stdio MCP servers가 stdout에 외부 non-JSON 라인을 출력할 때 첫 외부 라인에서 연결이 끊기는 문제를 수정했습니다 (2.1.105에서의 regression)
- `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC` 또는 `CLAUDE_CODE_DISABLE_TERMINAL_TITLE`이 설정되어 있을 때 headless/SDK 세션 auto-title이 추가 Haiku request를 발생시키는 문제를 수정했습니다
- Piped (non-TTY) Ink output이 단일 매우 넓은 라인을 포함할 때 과도한 메모리 할당 가능성을 수정했습니다
- `/skills` 메뉴가 fullscreen mode에서 list가 modal을 초과할 때 스크롤되지 않는 문제를 수정했습니다
- Remote Control 세션이 세션이 너무 오래되었을 때 re-login을 요청하는 대신 일반적인 오류를 표시하는 문제를 수정했습니다
- claude.ai에서의 Remote Control 세션 rename이 제목을 local CLI 세션에 유지하지 않는 문제를 수정했습니다
