> 원문: [Claude Code v2.1.92 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.92)

## 변경사항

- `forceRemoteSettingsRefresh` policy 설정을 추가했습니다. 이 설정이 활성화되면 CLI가 원격 관리 설정을 새로 가져올 때까지 시작을 차단하며, 가져오기 실패 시 종료됩니다(fail-closed)
- 로그인 화면에서 "3rd-party platform"을 선택할 때 접근 가능한 대화형 Bedrock 설정 마법사를 추가했습니다. 이 마법사는 AWS 인증, 리전 구성, 자격 증명 검증, 모델 고정을 안내합니다
- 구독 사용자를 위해 `/cost`에 모델별 및 캐시 히트 분석을 추가했습니다
- `/release-notes`가 이제 대화형 버전 선택기입니다
- Remote Control 세션 이름이 이제 호스트명을 기본 접두사로 사용합니다(예: `myhost-graceful-unicorn`). `--remote-control-session-name-prefix`로 재정의할 수 있습니다
- Pro 사용자는 이제 프롬프트 캐시가 만료된 후 세션으로 돌아올 때 바닥글 힌트를 확인할 수 있으며, 다음 턴이 캐시되지 않은 상태로 전송할 토큰 수를 대략적으로 표시합니다
- 장기 실행 세션 중 tmux 윈도우가 종료되거나 번호가 다시 지정된 후 subagent 생성이 "Could not determine pane count" 오류로 영구적으로 실패하는 문제를 수정했습니다
- 작은 빠른 모델이 `ok:false`를 반환할 때 prompt-type Stop 훅이 잘못 실패하는 문제를 수정했으며, non-Stop prompt-type 훅에 대해 `preventContinuation:true` 의미론을 복원했습니다
- 스트리밍이 배열/객체 필드를 JSON 인코딩 문자열로 내보낼 때 tool input validation 실패 문제를 수정했습니다
- extended thinking이 실제 콘텐츠와 함께 공백만 포함된 텍스트 블록을 생성했을 때 발생할 수 있는 API 400 오류를 수정했습니다
- auto-pilot 키 입력 및 연속 프롬프트 숫자 충돌로 인한 실수로 인한 피드백 설문조사 제출을 수정했습니다
- 전체 화면 모드에서 처리 중 텍스트 선택이 존재할 때 "esc to clear" 옆에 오류로 나타나는 "esc to interrupt" 힌트를 수정했습니다
- Homebrew 설치 업데이트 프롬프트를 cask의 릴리스 채널을 사용하도록 수정했습니다(`claude-code` → stable, `claude-code@latest` → latest)
- multiline 프롬프트에서 이미 줄의 끝에 있을 때 `ctrl+e`가 다음 줄의 끝으로 이동하는 문제를 수정했습니다
- 전체 화면 모드에서 위로 스크롤할 때 같은 메시지가 두 위치에 나타날 수 있는 문제를 수정했습니다(iTerm2, Ghostty 및 DEC 2026을 지원하는 다른 터미널)
- idle-return "/clear to save X tokens" 힌트가 현재 context 크기 대신 누적 세션 토큰을 표시하는 문제를 수정했습니다
- 세션 시작 시 인증되지 않은 claude.ai connector를 중복하는 plugin MCP 서버가 "connecting"에 고착되는 문제를 수정했습니다
- 큰 파일에 대한 Write tool diff 계산 속도를 개선했습니다(탭/`&`/`$`가 있는 파일에서 60% 더 빠름)
- `/tag` 명령어를 제거했습니다
- `/vim` 명령어를 제거했습니다(vim 모드는 `/config` → Editor mode에서 토글)
- Linux sandbox는 이제 npm 및 native 빌드에서 모두 `apply-seccomp` 헬퍼를 포함하여 샌드박스 명령어에 대한 unix-socket 차단을 복원합니다
