## 변경된 사항
- `/context` 명령어에 실행 가능한 제안 사항을 추가했습니다. — 컨텍스트 사용량이 많은 도구, 메모리 과다 사용, 용량 관련 경고 사항을 식별하고 구체적인 최적화 팁을 제공합니다.
- 자동 메모리 저장을 위한 사용자 지정 디렉터리를 설정할 수 있도록 `autoMemoryDirectory` 설정을 추가했습니다
- 제너레이터가 조기에 종료될 때 스트리밍 API 응답 버퍼가 해제되지 않아 Node.js/npm 코드 경로에서 RSS가 무한정 증가하는 메모리 누수 문제를 수정했습니다.
- 사용자 `allow` 규칙이나 스킬 `allowed-tools`에 의해 관리형 정책 `ask` 규칙이 우회되던 문제를 수정했습니다.
- 에이전트 프론트매터의 `model:` 필드 및 `--agents` JSON 구성에서 전체 모델 ID(예: `claude-opus-4-5`)가 아무런 오류 메시지 없이 무시되던 문제를 수정했습니다. 이제 에이전트는 `--model` 옵션과 동일한 모델 값을 허용합니다.
- 콜백 포트가 이미 사용 중인 경우 MCP OAuth 인증이 응답 없이 멈춰 있던 문제를 수정했습니다
- HTTP 200 오류를 반환하는 OAuth 서버(예: Slack)의 경우, 리프레시 토큰이 만료된 후에도 MCP OAuth 리프레시 시 재인증 요청이 표시되지 않던 문제를 수정했습니다.
- 터미널에 마이크 권한이 한 번도 부여된 적이 없는 사용자의 경우, macOS 네이티브 바이너리에서 음성 모드가 아무런 오류 메시지 없이 실패하던 문제를 수정했습니다. 이제 해당 바이너리에 `audio-input` 권한이 포함되어 macOS에서 올바르게 권한 요청을 표시합니다.
- `hook.timeout` 설정과 관계없이 종료 시 `SessionEnd` 훅이 1.5초 후에 강제 종료되던 문제를 수정했습니다. 이제 `CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MS`를 통해 이 시간을 설정할 수 있습니다.
- 로컬 소스를 사용하는 마켓플레이스 플러그인의 경우, REPL 내에서 `/plugin install` 명령어가 실패하는 문제를 수정했습니다.
- 마켓플레이스 업데이트 시 Git 서브모듈이 동기화되지 않던 문제를 수정했습니다. 이제 업데이트 후 서브모듈 내 플러그인 소스 코드가 더 이상 손상되지 않습니다.
- 인수가 포함된 알 수 없는 슬래시 명령어가 입력 내용을 아무런 알림 없이 무시하던 문제를 수정했습니다. 이제 해당 입력 내용이 경고 메시지로 표시됩니다.
- Windows 터미널, conhost 및 VS Code 통합 터미널에서 히브리어, 아랍어 및 기타 RTL 텍스트가 제대로 표시되지 않던 문제를 수정했습니다.
- 잘못된 형식의 파일 URI로 인해 Windows에서 LSP 서버가 작동하지 않던 문제를 수정했습니다.
- `--plugin-dir` 옵션을 변경하여, 이제 로컬 개발용 복사본이 동일한 이름의 설치된 마켓플레이스 플러그인을 우선 적용하도록 했습니다(단, 관리 설정에 의해 해당 플러그인이 강제 활성화된 경우는 제외).
- [VSCode] ‘Untitled’ 세션에서 삭제 버튼이 작동하지 않던 문제를 수정했습니다.
- [VSCode] 터미널 인식 가속도 기능을 통해 통합 터미널의 스크롤 휠 반응성을 개선했습니다
