> 원문: [Claude Code v2.1.208 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.208)

## 변경사항

- Screen reader 모드 추가: screen reader 사용자를 위한 옵트인 평문 렌더링. `claude --ax-screen-reader` 실행, `CLAUDE_AX_SCREEN_READER=1` 설정, 또는 설정에 `"axScreenReader": true` 추가
- `vimInsertModeRemaps` 설정 추가: vim 모드에서 `jj`와 같은 두 자리 insert-mode 시퀀스를 Escape에 매핑
- `CLAUDE_CODE_PROCESS_WRAPPER` 추가: agent view와 background service가 이제 모든 Claude Code self-spawn을 필수 wrapper executable을 통해 실행하여 corporate launcher를 준수
- Fullscreen 모드에서 multi-select 메뉴 및 "Other" input 행에 대한 마우스 클릭 지원 추가
- Fast mode가 해당 모드를 지원하는 모델로 전환한 후에도 꺼진 상태로 유지되는 문제 수정 — 설정에서 활성화되면 이제 자동으로 복원됨
- Background agent에 입력한 회신이 전달 실패 시 손실되는 문제 수정 — 텍스트가 이제 저장되고 세션이 재시작될 때 전달됨
- 업데이트로 인해 실행 중인 `claude agents` 프로세스가 시작된 바이너리가 교체된 후 background-session attach가 영구적으로 실패하는 문제 수정 ("Couldn't start the background daemon")
- CLI 자동 업데이트 후 context window(및 auto-compact 표시기)가 일시적으로 200k로 재설정되어 장시간 context 세션을 재개할 때 거짓된 "100% context used"를 유발하는 문제 수정
- 서버가 요청 진행 중에 HTTP/2 연결을 GOAWAY로 종료할 때 supervised 및 background 세션이 충돌하는 문제 수정
- `claude -p`에서 대용량 응답을 piping할 때 truncated stream-json/JSON 출력 및 결과 메시지 누락 문제 수정
- `CLAUDE_CODE_MAX_OUTPUT_TOKENS` 및 유사 env vars가 scientific-notation 값의 mantissa를 자동으로 사용하는 문제 수정 (`1e6`이 `1`이 됨)
- 매우 큰 markdown 테이블이 렌더링을 stall하거나 과도한 메모리를 사용하는 문제 수정; 200행 이상의 테이블은 처음 200행을 "… N more rows" 공지와 함께 표시
- 대상 텍스트가 여전히 고유하게 일치할 때 Edit tool이 읽기 후 수정된 파일에서 실패하는 문제 수정
- Read가 빈 파일을 "shorter than offset"으로 보고하고, Grep이 잘못된 regex 패턴에 대해 자동으로 "No files found"를 반환하고, Grep count 모드가 페이지 처리 시 총합을 과소 보고하고, Glob이 pattern, path 또는 working directory에 null byte가 포함되어 있을 때 불명확한 오류로 충돌하는 문제 수정
- `apiKeyHelper` 스크립트 실패가 ~10회의 자동 재시도 후 일반 401 오류로 숨겨지는 문제 수정; 스크립트의 자체 오류가 이제 3회 시도 이내에 표시됨
- Gateway가 응답을 변환할 때 Bedrock streaming 요청이 오해의 소지가 있는 "Truncated event message received"로 실패하는 문제 수정 — 이제 오류가 content-type을 명시하고 proxy를 가리킴
- 브라우저가 열기에 실패할 때 `/upgrade`가 upgrade URL 대신 로그인 흐름을 표시하는 문제 수정
- Windows 스타일 SDK 호스트로부터 blank CRLF 또는 whitespace 전용 라인이 stream-json 입력을 kill하는 문제 수정
- `control_request`가 non-string `set_model` 페이로드를 전달할 때 headless stream-json 세션이 무한정 hang되는 문제 수정; CLI가 이제 오류 응답으로 답함
- 세션 재개 시 반복되는 "No completion record was found" 공지 문제 수정 — orphaned background 작업이 이제 단일 요약으로 축소됨
- Remote Control 클라이언트가 terminal에 호스팅된 세션에 attach할 때 작업이 시작되거나 중지될 때까지 background agents 및 workflow 진행 상황을 볼 수 없는 문제 수정
- Agent tool이 subagent의 `tools` 목록이 nothing으로 해석될 때 tools 없이 시작되는 문제 수정 — 이제 인식되지 않은 항목을 명시하는 명확한 오류를 반환
- `/usage`가 더 신선한 데이터 위에 오래된 캐시된 bar를 표시하고, `/mcp`가 설정 편집 후 placeholder 서버를 재분류하지 않는 문제 수정
- SDK 호스트의 "Change directory" (예: Claude Desktop)가 실행 중인 background 작업이 있는 idle 세션에서 "A turn is in progress" 오류로 실패하는 문제 수정
- Workflow 저장 대화가 user-scope 저장을 위해 `CLAUDE_CONFIG_DIR` 위치 대신 `~/.claude/workflows/`를 표시하는 문제 수정
- `/release-notes`가 본 notes를 모델의 context에 추가하는 문제 수정 — "Show all"이 이전에는 전체 changelog를 모든 후속 요청에 주입함
- 붙여넣은 이미지가 peek 회신을 보낸 후 screen의 lifetime 동안 유지되는 agent view의 메모리 누수 수정
- Plugin refresh가 클라이언트 attach 전에 실행될 때 SDK 세션이 initialize 요청을 통해 정의된 agents를 잃는 문제 수정
- 장시간 세션에서의 여러 메모리 누수 수정: MCP stdio server stderr이 서버당 최대 64 MB까지 누적, LSP 문서가 무한정 열린 상태 유지 (이제 50-doc cap의 LRU), backgrounding 후 async hook 출력 유지, headless/SDK 세션의 large tool-result 페이로드로부터 무제한 증가
- Offset/limit를 사용하여 극도로 긴 단일 라인이 있는 파일을 읽을 때 메모리 폭발 문제 수정 — 이제 read가 전체 라인을 로드하는 대신 명확한 오류를 반환
- 많은 permission deny/ask 규칙이 있는 세션에서 turn당 multi-second slowdown 문제 수정 — rule matcher가 이제 한 번 컴파일되고 캐시됨
- Agent 작업 목록이 업데이트되는 동안 입력 responsiveness 개선 — 작업 업데이트가 더 이상 전체 UI를 re-render하지 않음
- 많은 MCP tools이 있는 print/SDK 세션에서 tool-pool 어셈블리 캐싱으로 per-tool-call CPU 오버헤드 감소 (높은 tool 수에서 최대 7배 빠른 tool rounds)
- 1,000개의 전체 파일을 고정하는 대신 16 MB로 file edit read 캐시를 제한하여 메모리 사용량 감소
- edit 집약적 세션에서 최대 79배까지 세션 transcript 크기 감소 및 superseded file-history 백업을 제거하여 checkpoint 디스크 사용량 제한
- Background agents 또는 대용량 대화에서 spawned된 forks가 있는 세션을 재개할 때 메모리 사용량 감소
- Completed background agents가 이제 종료되는 순간 사라지는 대신 cleanup까지 `/tasks`에 나열된 상태 유지
- Stopped background agent에 attach할 때 이제 빈 "Session is starting" 화면 대신 세션이 워밍업되는 동안 transcript를 즉시 표시
- Background 세션: 더 이전 daemon이 더 이상 자동으로 newer 버전이 spawned한 workers를 더 이전 바이너리로 재시작하지 않음
- Agent view: Ctrl+X가 이제 renamed-branch worktrees를 삭제하고, unpushed commits을 절대 파괴하지 않으며, worktree가 유지될 때 세션 행을 유지하고, 재사용된 worktree 이름이 현재 base로 재설정됨
- Catastrophic removals (예: `rm -rf ~`)을 포함하는 `$(…)`/backticks/`<(…)` commands가 이제 `--dangerously-skip-permissions` 및 auto 모드에서 prompt되고, plain 형식과 일치하도록 자동 수행
- `/install-github-app` 및 `/mcp` 설정 메뉴가 더 이상 background 세션에서 열리지 않음
- Empty URL로 구성된 MCP servers가 이제 config 오류 대신 `/mcp`에서 "not configured"으로 표시
- `/usage`가 이제 usage endpoint가 rate-limited될 때 오류 화면 대신 "as of" 공지와 함께 마지막으로 알려진 usage bars를 표시
- Bedrock auth가 sso_region이 Bedrock 지역과 다른 AWS SSO profiles에 대해 "Session token not found or invalid"로 실패하는 문제 수정 (2.1.207 regression)
