## 변경사항

- session transcript 파일 변조를 차단하는 auto mode rule이 추가되었습니다
- `--json-schema`가 schema가 유효하지 않을 때 구조화되지 않은 output을 조용히 생성하는 문제가 수정되었으며, `format` keyword를 사용하는 schema가 거부되는 문제가 수정되었습니다
- Claude가 작업 중일 때 전송된 메시지가 `--max-turns` 제한에서 turn이 끝날 때 조용히 손실되는 문제가 수정되었습니다
- Windows worktree 제거 시 worktree 내에 NTFS junction 또는 directory symlink가 있으면 worktree 외부의 파일이 삭제되는 문제가 수정되었습니다
- background agent가 `SendMessage`로 재개된 후 agent list에서 "failed" 또는 "completed"로 계속 표시되는 문제가 수정되었습니다
- agent의 turn에 읽을 수 있는 텍스트가 없을 때 background job이 agent list에서 "needs input"에서 "working"으로 다시 바뀌는 문제가 수정되었습니다
- background agent가 upgrade restart 중일 때 `claude attach`가 오류를 발생하는 대신 대기하도록 수정되었습니다
- session-to-PR linking이 output이 30K inline 제한을 초과하는 Bash call에서 생성된 PR을 놓치는 문제가 수정되었습니다
- `claude mcp add-from-claude-desktop`이 server name에 지원되지 않는 문자가 포함되어 있을 때 멈추는 문제가 수정되었으며, 이제 유효하지 않은 이름이 보고되고 나머지 server는 계속 import됩니다
- plugin LSP server가 초기화에 실패하면 같은 file extension을 처리하는 다른 plugin의 유효한 LSP server가 작동하지 못하는 문제가 수정되었습니다
- Claude가 시작된 directory가 command 실행 중에 삭제되거나 locked되거나 unmount될 때 Windows crash가 수정되었습니다
- file watcher가 closed될 때 directory scan이 여전히 진행 중인 crash가 수정되었습니다
- project verify skill이 documented command가 변경되었을 때만이 아니라 매 session마다 rewrite되는 문제가 수정되었습니다
- agent view가 한 줄 너무 높이 rendering되고 job list가 화면을 약간 넘칠 때 header가 clipping되는 문제가 수정되었습니다
- web 및 mobile Remote Control panel의 background task가 모든 membership change에서 full task state를 forward하여 stale "Running" status를 표시하는 문제가 수정되었습니다
- auto mode가 context에서 resolve할 수 없는 variable에서 `rm -rf`를 실행하기 전에 물어보도록 개선되었습니다
- Auto-update binary download는 이제 memory에 buffer하지 않고 disk로 stream하여 updater의 peak memory usage을 약 400 MB 줄입니다
- Background task notification은 이제 human input이 발생하지 않았음을 명시적으로 나타내어 transcript에서 fabricated approval이 실행되는 것을 방지합니다
- Agent view가 개선되었습니다: 기존 PR을 edit, merge, comment하거나 push하는 session은 이제 `claude agents`에서 link됩니다
- Agent view가 개선되었습니다: 각 row는 raw tool call text 대신 colored state word 및 classifier-written headline을 표시하며, peek는 blocked session의 exact ask를 포함한 full status로 열립니다
- `/doctor`는 이제 issue를 진단하고 수정할 수 있는 full setup checkup이며, `/checkup`은 그 alias입니다
- Claude Desktop pane rename을 앞두고 "Claude Browser" MCP server name이 예약되었습니다("Claude Preview"와 함께); user-configured MCP server는 더 이상 두 이름 중 하나로 등록할 수 없습니다
- CLI 2.1.203+에서 Cowork VM-mode local-agent session이 "Not logged in · Please run /login"으로 시작되지 않는 문제가 수정되었습니다
