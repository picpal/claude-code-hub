## 변경사항

- Plugin 종속성 강제 적용 추가: `claude plugin disable`은 이제 다른 활성화된 plugin이 대상에 종속될 때 거부하며(복사-붙여넣기 가능한 disable-chain hint 제공), `claude plugin enable`은 전이적 종속성을 강제로 활성화합니다
- `/plugin` marketplace browse pane에 예상 context 비용(턴별 및 호출별 token 추정)을 추가했습니다
- `worktree.bgIsolation: "none"` 설정을 추가하여 background sessions이 `EnterWorktree` 없이 working copy를 직접 편집할 수 있도록 했습니다. worktrees가 비실용적인 repos의 경우를 위합니다
- PowerShell tool이 이제 `-ExecutionPolicy Bypass`를 전달합니다. `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1`로 거부할 수 있습니다
- Background sessions이 이제 유휴 상태에서 깨어난 후에도 설정한 model과 effort level을 유지합니다
- Attached agent sessions에서 Shift+Tab이 이제 auto mode를 순환에 포함합니다
- Corrupt된 `.credentials.json`(비-array `scopes` 값 포함)이 시작 시 CLI를 hanging하거나 OAuth token refresh를 조용히 중단하는 문제를 수정했습니다
- Windows Terminal 및 WSL의 `claude agents`에서 마우스 오른쪽 클릭 paste를 수정했습니다
- 반복적으로 loop를 차단하는 stop hooks가 영원히 loop하는 문제를 수정했습니다. 이제 8번의 연속 차단 후 경고와 함께 turn이 종료됩니다(`CLAUDE_CODE_STOP_HOOK_BLOCK_CAP`로 override 가능)
- Claude가 반복 사이에 유휴 상태일 때 Esc/Ctrl+C가 pending `/loop` wakeup을 취소하지 못하는 문제를 수정했습니다
- Background shells 또는 delegated subagents이 여전히 실행 중인 동안 `/goal` evaluator가 발동하는 문제를 수정했습니다
- Settings.json의 `env`에서 `NO_COLOR`/`FORCE_COLOR`가 Claude Code의 UI colors을 제거하는 문제를 수정했습니다. 이제 subprocesses에만 적용됩니다
- Agent view가 Windows에서 sessions을 나열할 때 반복적으로 PowerShell processes를 생성하는 문제를 수정했습니다
- Prompt 없이 `/bg`를 사용할 때 forked session에 "continue"를 보내는 문제를 수정했습니다. 이제 fork가 input을 대기합니다
- `--agent <name>`이 `plugin:` prefix 없이 plugin-contributed agents를 찾지 못하는 문제를 수정했습니다
- Agent view에서 session을 삭제할 때 transcript file을 제거하지 않는 문제를 수정했습니다
- Windows Terminal의 attached background sessions에서 scrolling할 때 stale-fragment가 rendering되는 문제를 수정했습니다
- Host sleep 또는 macOS App Nap 후 background agents의 false-positive worker-stall detection storm을 수정했습니다
- 5xx error messages가 status.claude.com을 가리키는 대신 configured gateway 또는 cloud provider 이름을 나타내는 문제를 수정했습니다
- PowerShell tool은 이제 Bedrock, Vertex, Foundry 사용자의 Windows에서 기본적으로 활성화됩니다. `CLAUDE_CODE_USE_POWERSHELL_TOOL=0`으로 거부할 수 있습니다
- `claude agents`가 이제 `--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir`을 허용하고 dashboard와 dashboard에서 dispatched된 background sessions에 적용합니다
- `claude agents`가 `--permission-mode`, `--model`, `--effort`, `--dangerously-skip-permissions`을 허용하여 dashboard에서 dispatched된 sessions의 기본값을 설정합니다
- `claude --bg --dangerously-skip-permissions`이 이제 retire→wake 간에 유지됩니다
- Background sessions이 조용히 IDE file references를 warm spare의 input에 캡처하여 reference가 `claude agents`에서 dispatched된 다음 prompt의 앞에 prepend되는 문제를 수정했습니다
- Worktree cleanup이 더 이상 `git worktree remove`가 실패할 때 `rm -rf`로 fallback하지 않으므로 gitignored 또는 in-progress files의 손실을 방지합니다
- macOS의 background-job sessions이 Full Disk Access를 부여받았음에도 불구하고 `~/Documents`, `~/Desktop`, `~/Downloads` 아래의 files를 읽을 때 "Operation not permitted" errors를 받는 문제를 수정했습니다
- `/bg`가 이제 `--mcp-config`, `--settings`, `--add-dir`, `--plugin-dir`, `--strict-mcp-config`를 보존하므로 backgrounded sessions이 respawn 간에 MCP servers 및 settings을 유지합니다
- `claude agents`에서 launched된 background sessions이 이제 settings.json의 `permissions.defaultMode`를 준수합니다(이전에는 auto mode로 override되었습니다)
- Windows에서 response가 streaming되는 동안 `claude agents`에서 ← 키를 누르면 agents list가 모든 input에 반응하지 않게 되는 문제를 수정했습니다
- `/bg` 및 ←-detach가 이제 `--fallback-model`을 보존하므로 backgrounded workers가 overload 시 fallback model로 degrading되고 hard-failing되지 않습니다
- `/bg` 및 ←-detach가 이제 `--allow-dangerously-skip-permissions`를 보존하므로 forked worker가 Shift+Tab cycle에서 bypass-permissions을 계속 사용할 수 있습니다
- Background daemon spawn이 `~/.local/bin/claude` launcher가 누락되었거나 non-executable일 때 이제 running binary로 fallback됩니다
- `claude agents --allow-dangerously-skip-permissions`가 dispatched sessions을 bypass mode로 defaulting하는 대신 permission cycle에서 사용 가능하게 하는 문제를 수정했습니다
