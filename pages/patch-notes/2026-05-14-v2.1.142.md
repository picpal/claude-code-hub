## 변경사항

- 새로운 `claude agents` 플래그 추가: `--add-dir`, `--settings`, `--mcp-config`, `--plugin-dir`, `--permission-mode`, `--model`, `--effort`, `--dangerously-skip-permissions`를 통해 dispatched background sessions를 구성할 수 있습니다
- Fast mode는 이제 기본적으로 Opus 4.7을 사용합니다 (이전: Opus 4.6). Fast mode를 Opus 4.6으로 고정하려면 `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1`을 설정하세요
- 루트 수준의 `SKILL.md`를 가지고 있으며 `skills/` 하위 디렉토리가 없는 플러그인은 이제 skill로 표시됩니다
- `/plugin` 세부 정보 pane과 `claude plugin details`는 플러그인이 제공하는 LSP servers를 표시합니다
- `/web-setup`은 기존 GitHub App 연결을 교체하기 전에 경고를 표시합니다
- `MCP_TOOL_TIMEOUT`이 remote HTTP 및 SSE MCP servers에 대한 per-request fetch timeout을 올리지 못하여 구성된 값에 관계없이 tool calls이 60초로 제한되는 문제를 수정했습니다
- Background sessions이 기존 git worktrees를 인식하지 못하여 Edit을 차단하고 EnterWorktree가 중복 생성을 거부하는 문제를 수정했습니다
- macOS sleep/wake 후 background sessions이 사라지고 daemon reconnect가 실패하는 문제를 수정했습니다 — daemon이 경과한 idle time으로 취급하는 대신 clock jumps를 감지합니다
- 바이너리 업그레이드 후 (예: `brew upgrade`) daemon이 정리되지 않아 dispatched agents이 삭제된 경로에서 crash-loop되는 문제를 수정했습니다
- Claude-in-Chrome extension이 shared tab 없이 연결되었을 때 background agents가 crash-loop되는 문제를 수정했습니다
- 첨부된 `claude agents` session에서 링크를 클릭할 때 문제를 수정했습니다 — background worker의 headless browser shim이 attached된 동안 더 이상 적용되지 않습니다
- `claude agents` "v to open in editor"가 shell의 `$EDITOR`/`$VISUAL` 대신 daemon의 기본 editor를 사용하는 문제를 수정했습니다
- `claude agents`가 network-drive working directories가 있는 Windows에서 deadlock되는 문제를 수정했습니다; Ctrl+C는 이제 startup 중에 작동합니다
- Apple Terminal 또는 다른 256-color-only terminals에서 `claude agents` session에 attach할 때 background-color bleed 문제를 수정했습니다
- `claude --bg --dangerously-skip-permissions`이 retire/wake 간 지속되지 않는 문제를 수정했습니다
- 첫 번째 메시지가 링크일 때 session titles이 URL에서 파생되는 문제를 수정했습니다
- Remote clients의 redundant `set_model` requests가 transcript에 duplicate `/model` breadcrumbs를 주입하는 문제를 수정했습니다
- `skills: ["./"]`를 사용하는 플러그인이 false "path escapes plugin directory" 오류를 표시하는 문제를 수정했습니다
- Plugin cache cleanup이 설치 메타데이터가 없을 때 active plugin version directory를 삭제하는 문제를 수정했습니다
- `/plugin` browse pane이 새로 게시된 플러그인에 대해 "0 installs"를 표시하는 문제를 수정했습니다
- Plugin advisories가 default folder를 shadow하는 모든 `plugin.json` 키를 명명하지 않는 문제를 수정했습니다
- Reactive compaction 개선: 첫 번째 summarize 시도는 이제 원래 request의 overflow size에서 seed되어 낭비된 near-full-context retry를 피합니다
- Hook 구성 오류 개선: `SessionStart`/`Setup`/`SubagentStart`에 대해 prompt- 또는 agent-type hook을 구성하면 이제 명확한 "use a command-type hook instead" 오류를 표시합니다
- Usage Policy refusal 메시지에서 stale `/model claude-sonnet-4-20250514` 제안을 제거했습니다
