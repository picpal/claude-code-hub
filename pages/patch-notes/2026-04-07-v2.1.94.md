> 원문: [Claude Code v2.1.94 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.94)

## 변경사항

- Amazon Bedrock이 Mantle으로 지원되도록 추가, `CLAUDE_CODE_USE_MANTLE=1` 설정
- API-key, Bedrock/Vertex/Foundry, Team, Enterprise 사용자를 위해 기본 effort 레벨을 medium에서 high로 변경 (`/effort`로 제어 가능)
- Slack MCP send-message tool call을 위해 클릭 가능한 채널 링크가 있는 compact한 `Slacked #channel` 헤더 추가
- plugin 출력 스타일에 대한 `keep-coding-instructions` frontmatter 필드 지원 추가
- `UserPromptSubmit` hook에 session title을 설정하기 위한 `hookSpecificOutput.sessionTitle` 추가
- `"skills": ["./"]`를 통해 선언된 plugin skill이 이제 directory basename 대신 skill의 frontmatter `name`을 invocation name으로 사용하여 install 방법 전반에 걸쳐 안정적인 이름 제공
- 긴 Retry-After 헤더와 함께 429 rate-limit 응답 후 agent가 stuck으로 나타나는 문제 수정 — 오류가 이제 즉시 표시되고 조용히 대기하지 않음
- macOS에서 login keychain이 잠겨 있거나 password가 동기화되지 않았을 때 Console login이 조용히 "Not logged in"으로 실패하는 문제 수정 — 오류가 이제 표시되고 `claude doctor`가 수정 방법을 진단함
- YAML frontmatter에 정의된 plugin skill hook이 조용히 무시되는 문제 수정
- `CLAUDE_PLUGIN_ROOT`가 설정되지 않았을 때 plugin hook이 "No such file or directory"로 실패하는 문제 수정
- startup 시 local-marketplace plugin에 대해 `${CLAUDE_PLUGIN_ROOT}`가 설치된 cache 대신 marketplace 소스 directory로 resolve되는 문제 수정
- long-running session에서 동일한 diff가 반복되고 blank page가 표시되는 scrollback 문제 수정
- transcript의 multiline user prompt가 `❯` caret 아래가 아닌 텍스트 아래에서 wrapped line을 들여쓰는 문제 수정
- search input에서 Shift+Space가 공백 문자 대신 literal word "space"를 삽입하는 문제 수정
- xterm.js 기반 terminal(VS Code, Hyper, Tabby)에서 실행되는 tmux 내부에서 hyperlink를 클릭할 때 두 개의 브라우저 탭이 열리는 문제 수정
- content height 변경이 mid-scroll에서 발생할 때 compounding ghost line을 남길 수 있는 alt-screen rendering bug 수정
- `settings.json` `env`를 통해 설정될 때 `FORCE_HYPERLINK` environment variable이 무시되는 문제 수정
- dialog에서 native terminal cursor이 선택된 tab을 추적하지 않아 screen reader와 magnifier가 tab navigation을 따를 수 없는 문제 수정
- Sonnet 3.5 v2의 `us.` inference profile ID를 사용하여 Bedrock invocation 수정
- interrupted mid-stream일 때 conversation history에서 partial assistant response를 preserve하지 않는 SDK/print mode 수정
- `--resume`을 개선하여 `cd` command를 출력하는 대신 동일한 repo의 다른 worktree에서 session을 직접 resume
- stream-json input/output에서 chunk boundary가 UTF-8 sequence를 분할할 때 CJK 및 기타 multibyte text가 U+FFFD로 손상되는 문제 수정
- [VSCode] session 시작 시 cold-open subprocess 작업 감소
- [VSCode] mouse가 list 위에 있으면서 typing 또는 arrow key를 사용할 때 dropdown menu가 잘못된 item을 선택하는 문제 수정
- [VSCode] `settings.json` 파일 parse에 실패할 때 warning banner를 추가하여 사용자가 permission rule이 적용되지 않고 있음을 알 수 있도록 함
