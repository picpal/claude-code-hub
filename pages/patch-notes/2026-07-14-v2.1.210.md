> 원문: [Claude Code v2.1.210 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.210)

## 변경사항

- 축소된 tool 요약 줄에 실시간 경과 시간 카운터를 추가했으므로 오래 실행되는 tool 호출이 멈춘 것처럼 보이지 않고 눈에 띄게 진행됩니다
- `Write(path)`, `NotebookEdit(path)`, 및 `Glob(path)` permission 규칙에 대한 시작 경고를 추가했습니다 — 대신 `Edit(path)` 또는 `Read(path)`를 사용하십시오
- `isolation: 'worktree'` subagent가 자신의 격리된 worktree 대신 메인 repo checkout에 대해 git 변경 명령을 실행할 수 있던 문제를 수정했습니다
- webhook 페이로드 및 릴레이된 PR 코멘트와 같은 인간이 아닌 출처의 입력에 대해 `ultracode` 키워드 opt-in이 작동하던 문제를 수정했습니다
- UI 컴포넌트가 스타일된 text 요소 외부의 콘텐츠를 반환할 때 렌더링된 text 조각이 crash 텔레메트리로 유출되던 문제를 수정했습니다
- Claude Code에서 열린 외부 편집기로 paste marker가 유출되어 붙여넣은 텍스트 주변에 임의의 È/É 문자로 나타날 수 있던 문제를 수정했습니다
- `claude attach`가 session 전환 중 "job not found" 또는 "agent is still starting" 오류로 인해 실패하는 경우가 있던 문제를 수정했습니다 — attach는 이제 daemon이 안정화될 때까지 대기하며, 느린 attach 중 터미널 크기 조정은 완료되면 적용됩니다
- tool의 결과 renderer가 UI 요소 대신 숫자 bigint 값 또는 일반 text를 반환했을 때 session crash가 발생하던 문제를 수정했습니다
- hook callback 타임아웃이 사용자 거부로 모델에 잘못 보고되어 무인 session이 중지되고 대기하던 문제를 수정했습니다
- `cd` 명령이 background로 이동한 후 Claude가 `cd`가 적용된 것으로 가정하던 문제를 수정했습니다; tool 결과는 이제 working directory가 변경되지 않음을 명시합니다
- MCP server를 mid-session에 재동기화할 때 plugin에서 제공한 MCP server가 종료되던 문제를 수정했습니다
- 편집 없이 plan을 승인할 때 "(edited by user)"로 표시되고 stale snapshot으로 plan 파일을 덮어쓰던 문제를 수정했습니다
- `/doctor`가 Bedrock, Vertex, 및 Foundry에서 auto-mode-default 제안을 건너뛰던 문제를 수정했습니다(이들 플랫폼에서는 auto mode가 더 이상 opt-in을 필요로 하지 않습니다)
- Grep content mode가 결과 끝을 넘어 페이지를 매길 때 "No matches found"라고 표시하던 문제를 수정했습니다
- skill 및 command의 일치하지 않는 `$1`/`$2` 위치 placeholder가 자동으로 제거되던 문제를 수정했습니다; 이제 그대로 유지됩니다
- plugin 캐시 쓰기가 실패 시 temp 파일을 남기고 Windows 및 network filesystem에서 잠긴 파일 이름 변경에 실패하던 문제를 수정했습니다
- client가 background 서비스로의 연결을 재설정할 때 background worker가 crash loop하던 문제를 수정했습니다
- `claude agents --effort ultracode`가 dispatched session에 도달하지 못하던 문제를 수정했습니다; 값이 자동으로 삭제되었습니다
- ← 키를 눌러 agents 보기를 열 때 session으로 반환할 때 task tracker가 삭제되던 문제를 수정했습니다
- agents dashboard가 session 삭제 후 버려진 reply draft에서 붙여넣은 이미지를 유지하던 문제를 수정했습니다
- 종료된 background session이 영구적인 `git worktree lock`을 남기던 문제를 수정했습니다; 주기적 sweep은 이제 소유 프로세스가 없는 lock을 해제합니다
- SDK MCP server가 `initialize` control request를 통해 등록될 때 다음 turn까지 연결을 기다리던 문제를 수정했습니다
- session에서 agents 보기로 반환할 때 `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`로 인해 겹치는 ghost frame이 남던 문제를 수정했습니다
- 늦게 나타나는 `.claude/*` symlink가 sandbox deny-write 목록으로 조정되지 않던 문제를 수정했습니다
- Agent tool을 subagent가 읽은 콘텐츠를 통한 간접 prompt injection으로부터 강화했습니다
- Bash/PowerShell tool 메시지를 개선했습니다 — 명령이 timeout에 도달하고 자동으로 background될 때 모델이 hang과 명시적 background 요청을 구분할 수 있습니다
- auto mode를 개선했습니다: permission classifier는 이제 외부 session에 대해 기본적으로 Sonnet 5를 사용하며, session의 첫 요청에서 검증되고 session에 대해 고정됩니다
- 번들된 dataviz skill의 chart 색상 검증을 perceptual OKLab 색상 차이로 개선하고 색맹 임계값을 재보정했습니다
- MEMORY.md 인덱스를 읽기 제한을 초과하여 남기는 memory 쓰기는 이제 자동 truncation 대신 명시적 오류를 생성합니다
- Screen reader mode는 이제 Shift+Tab으로 permission mode를 순환할 때 permission mode 변경을 음성으로 알립니다
- agents footer 힌트는 이제 input을 기다리고 있는 background agent의 수를 표시하며, 수가 변경될 때 간단한 색상 강조를 표시합니다
- Agent view: ← 키를 누른 session은 마우스 hover 또는 화살표 키로 선택을 이동한 후에도 시각적으로 표시된 상태를 유지합니다
- Fable advisor 오류를 유발하는 server-side 문제가 수정되는 동안 Fable은 일시적으로 advisor picker에서 사용 불가로 표시됩니다
