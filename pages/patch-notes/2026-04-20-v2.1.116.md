> 원문: [Claude Code v2.1.116 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.116)

## 변경사항

- 큰 세션에서 `/resume`이 훨씬 빨라졌으며(40MB 이상 세션에서 최대 67%), 많은 dead-fork 항목이 있는 세션을 더 효율적으로 처리합니다
- 여러 stdio server가 구성되어 있을 때 MCP 시작이 더 빨라졌으며, `resources/templates/list`는 이제 첫 번째 `@`-mention까지 지연됩니다
- VS Code, Cursor, Windsurf 터미널에서 더 부드러운 전체화면 스크롤 — `/terminal-setup`은 이제 에디터의 스크롤 민감도를 구성합니다
- Thinking spinner는 이제 진행 상황을 인라인으로 표시하고("still thinking", "thinking more", "almost done thinking"), 별도의 hint 행을 대체합니다
- `/config` 검색은 이제 옵션 값과도 일치합니다(예: "vim"을 검색하면 Editor mode 설정을 찾습니다)
- `/doctor`는 이제 Claude가 응답하는 동안에도 현재 턴이 끝날 때까지 기다릴 필요 없이 열 수 있습니다
- `/reload-plugins` 및 백그라운드 플러그인 자동 업데이트는 이제 이미 추가한 마켓플레이스에서 누락된 플러그인 종속성을 자동 설치합니다
- Bash tool은 이제 `gh` 명령이 GitHub의 API rate limit에 도달했을 때 힌트를 표시하므로, agent가 재시도하지 않고 백오프할 수 있습니다
- Settings의 Usage 탭은 이제 5시간 및 주간 사용량을 즉시 표시하며, usage endpoint가 rate-limited될 때 더 이상 실패하지 않습니다
- Agent frontmatter `hooks:`는 이제 `--agent`를 통해 main-thread agent로 실행할 때 발동합니다
- Slash command 메뉴는 이제 필터가 0개의 결과를 반환할 때 사라지는 대신 "No commands match"를 표시합니다
- 보안: sandbox auto-allow는 더 이상 `/`, `$HOME` 또는 다른 중요 시스템 디렉토리를 대상으로 하는 `rm`/`rmdir`에 대한 dangerous-path 안전 검사를 우회하지 않습니다
- Devanagari 및 기타 Indic script가 터미널 UI에서 손상된 열 정렬로 렌더링되는 문제를 수정했습니다
- Kitty keyboard protocol을 사용하는 터미널(iTerm2, Ghostty, kitty, WezTerm, Windows Terminal)에서 Ctrl+-이 실행 취소를 트리거하지 않는 문제를 수정했습니다
- Kitty keyboard protocol을 사용하는 터미널(Warp fullscreen, kitty, Ghostty, WezTerm)에서 Cmd+Left/Right이 줄 시작/끝으로 이동하지 않는 문제를 수정했습니다
- Claude Code가 wrapper 프로세스(예: `npx`, `bun run`)를 통해 시작될 때 Ctrl+Z가 터미널을 행(hang)시키는 문제를 수정했습니다
- 터미널 크기 조정 또는 대량 출력 폭발로 인해 이전 대화 기록이 반복되는 inline mode의 scrollback 중복 문제를 수정했습니다
- 짧은 터미널 높이에서 modal search 대화상자가 화면을 오버플로우하여 검색 상자와 키보드 힌트를 숨기는 문제를 수정했습니다
- 스크롤 중 VS Code integrated 터미널에서 산재된 빈 셀과 사라지는 composer chrome 문제를 수정했습니다
- 병렬 요청이 요청 설정 중에 완료될 때 발생할 수 있는 cache control TTL 순서와 관련된 간헐적 API 400 오류를 수정했습니다
- `/branch`가 50MB보다 큰 transcript가 있는 대화를 거부하는 문제를 수정했습니다
- `/resume`이 로드 오류를 보고하지 않고 큰 세션 파일에서 빈 대화를 자동으로 표시하는 문제를 수정했습니다
- `/plugin` Installed 탭이 Needs attention 또는 Favorites에 나타날 때 동일 항목을 두 번 표시하는 문제를 수정했습니다
- 세션 중간에 worktree에 들어간 후 `/update` 및 `/tui`가 작동하지 않는 문제를 수정했습니다
