## 변경사항

- organization 기본 모델 지원 추가 — 관리자가 org console에서 설정하며, 직접 선택하지 않았을 때 `/model`에서 "Org default"(또는 "Role default")로 표시됨
- 세션 시작 시 읽기 쉬운 기본 이름 추가 — 세션을 더 쉽게 식별하고 메시지를 보낼 수 있음
- 채팅에서 클릭 가능한 파일 첨부 추가 — Cmd/Ctrl-click으로 Finder/Explorer에서 파일 표시
- 보안: `claude mcp list`/`get`이 더 이상 repo가 committed `.claude/settings.json`을 통해 자체 승인한 `.mcp.json` 서버를 실행하지 않으며, 신뢰되지 않은 workspace에는 `⏸ Pending approval` 표시
- 백그라운드 작업 깨우기 시 transcript probe가 실제 transcript를 잘못 읽었을 때 대화를 영구적으로 삭제하고 원본 prompt를 다시 실행하는 문제 수정 — 파일을 격리하여 절대 삭제하지 않음
- 여러 병렬 요청이 사용량 제한에 도달한 순간에 진행 중일 때 rate-limit 경고가 깜빡이고 rate-limit telemetry가 과도하게 계산되는 문제 수정
- 백그라운드 세션의 turn 후 중복된 recap 줄 수정: schema가 거부한 StructuredOutput 시도가 더 이상 재시도와 함께 렌더링되지 않음
- PowerShell `git diff`/`git grep`, `egrep`/`fgrep`, 및 `|`를 포함하는 따옴표 처리된 search pattern이 exit 1일 때 실패로 보고되는 문제 수정 — Bash 동작과 일치
- 여러 `claude agents` side panel 문제 수정: agent 열기 시 키보드 focus가 stuck되는 문제, 백그라운드 작업이 매번 열 때마다 subagent 유형을 잃는 문제, 세션이 활동 중일 때 잘못된 상태를 표시하는 문제
- `claude agents --dangerously-skip-permissions`이 bypass disclaimer를 표시하고 생성된 agent에 bypass 모드를 적용하는 대신 자동 모드로 자동 폴백하는 문제 수정
- Remote 세션에 대한 mid-turn crash recovery 수정 — 서버 재시작으로 중단된 세션이 이제 다음 worker에서 자동 재개됨
- 이전 경로에 특수 문자가 포함되었을 때 `/cd`로 이동한 세션이 graceful하지 않은 종료 후 이전 디렉터리의 resume 목록에 다시 나타나는 문제 수정
- source가 "."인 로컬 plugin을 건너뛰고 첫 번째 error class 후 멈추는 `claude plugin validate` 수정
- idle prompt에서 Esc Esc가 rewind menu를 열지 않는 문제 수정 (regression) — Ctrl+C 또는 Ctrl+X Ctrl+K를 사용하여 백그라운드 agent 중지
- scope가 지정되지 않았을 때 MCP OAuth가 authorization server의 전체 `scopes_supported` catalog을 요청하여 GitLab self-hosted 및 기타 enterprise IdP에서 `invalid_scope` 실패를 유발하는 문제 수정
- `/context`이 Bedrock에서 모든 tool group에 대해 0 token을 표시하는 문제 수정
- `/deep-research`이 verifier 실패를 `unverified` 대신 "all claims refuted"로 잘못 보고하는 문제 수정
- plugin dependency version pin이 marketplace를 git repo를 기반으로 한 로컬 폴더 경로로 추가했을 때 적용되지 않는 문제 수정
- `claude agents` 세션 상태 수정: 완료된 행이 더 이상 "Done"과 "Needs your input" 사이를 번갈아 보이지 않으며, stalled agent는 이제 "Needs attention"으로 레이블되고, PR을 언급한 결과는 클릭 가능한 링크 표시
- voice dictation이 공백을 삼키고 voice 모드가 활성화되었을 때 매우 빠른 타이핑 중 spuriously recording을 시작하는 문제 수정
- 백그라운드 세션 안정성 개선: 장시간 실행되는 command와 workflow가 이제 세션의 process가 중지, 재시작 또는 업데이트되더라도 생존함 — Windows에서도, 백그라운드 shell이 죽이는 대신 handed off됨
- 백그라운드 agent 개선: daemon restart로 인해 죽은 worker는 이제 다음 번 agent view가 열릴 때 자동으로 중단한 지점에서 재개됨
- `/code-review` workflow 개선: 5개의 cleanup finder를 1개로 통합하여 token 사용량을 약 25% 감소
- streaming 중 no-op subtree walk를 건너뛰어 terminal UI의 per-frame rendering 작업 감소
- streaming idle watchdog이 이제 모든 provider에 대해 기본적으로 활성화됨 — response stream이 5분 동안 이벤트를 생성하지 않으면 abort하고 retry함. `CLAUDE_ENABLE_STREAM_WATCHDOG=0`으로 비활성화할 수 있음
- Remote Control이 이제 `ANTHROPIC_BASE_URL`이 non-Anthropic host를 가리킬 때 비활성화됨 — `CLAUDE_CODE_USE_BEDROCK`/`_VERTEX`/`_FOUNDRY` 아래의 기존 동작과 일치
- foreground 세션에서 agent view를 열기가 두 번의 `←` 누름 대신 한 번의 누름이 필요하도록 변경 — 백그라운드 세션의 동작과 일치
