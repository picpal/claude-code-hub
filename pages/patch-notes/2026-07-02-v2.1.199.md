## 변경사항

- `/skill-a /skill-b do XYZ`와 같이 스택된 slash-skill 호출이 이제 첫 번째만이 아닌 모든 선행 skill(최대 5개)을 로드합니다
- SSL 인증서 오류(TLS 검사 프록시, 누락된 `NODE_EXTRA_CA_CERTS`, 만료된 인증서)가 더 이상 재시도를 소진한 후 조치 가능한 지침을 표시하지 않고, 이제 수정 힌트와 함께 즉시 실패합니다
- API가 부분 출력 후 스트림 중간에 오버로드/서버 오류를 발생시킬 때 스트리밍 응답이 폐기되는 문제가 수정되었으며, 이제 부분 응답이 불완전-응답 공지와 함께 유지됩니다
- 속도 제한 또는 서버 오류로 인해 subagent가 부분 작업을 부모에게 반환하지 않고 조용히 실패하는 문제가 수정되었습니다
- Subagent가 API 오류(예: 사용 한도 도달)를 성공적인 결과로 보고하는 문제가 수정되었으며, 이제 오류가 부모 agent에 보고됩니다
- Linux의 background-agent daemon이 정리되지 않은 종료로 인해 손상된 worker record를 남긴 후 ~50초마다 자신과 실행 중인 모든 agent를 종료하는 문제가 수정되었습니다
- macOS에서 SSH를 통해 background agent가 "Could not switch to audit session" 오류로 cold-start에 실패하는 문제가 수정되었습니다(2.1.196 버전의 회귀)
- `claude stop`이 background-agent respawn과 경합할 때 조용히 취소되는 문제가 수정되었으며, 이제 respawn이 stop을 인정합니다
- Background job 진행 표시기가 작업이 오래된 명령을 실행하는 동안 수 분간 정체되는 문제가 수정되었습니다
- 메모리 부족 장치의 background session이 일반 오류를 표시하는 문제가 수정되었으며, 이제 메모리 부족을 나타내고 리소스 확보를 제안합니다
- background agent가 완료될 때 agent view에서 Working과 Idle 사이에 잠깐 깜박거리는 원격 session의 문제가 수정되었습니다
- 다른 subagent가 계속 작동 중일 때 유휴 subagent가 agent panel에서 사라지는 문제가 수정되었으며, 이제 초과 유휴 agent가 확장 가능한 요약 행으로 축소됩니다
- Subagent를 보는 중 `/model` 또는 `/fast`를 입력할 때 조용히 lead의 model picker가 열리는 문제가 수정되었으며, 이제 공지에서 명령이 lead에 적용된다고 설명합니다
- `SessionStart`, `Setup`, 및 `SubagentStart` hook이 종료 코드 2로 종료할 때 조용히 stderr를 숨기는 문제가 수정되었으며, 이제 오류가 transcript에 표시됩니다
- `claude --dangerously-skip-permissions daemon <subcommand>`이 subcommand를 실행하는 대신 chat prompt로 처리되는 문제가 수정되었습니다
- Re-spawn된 agent가 이전 agent의 이름을 재사용할 때 `SendMessage`가 조용히 잘못 라우팅되는 문제가 수정되었으며, 이제 도구가 불일치를 감지하고 호출자에게 대상을 다시 지정하도록 요청합니다
- 새 메시지가 없는 session을 열거나 다시 시작할 때 불필요하게 transcript 파일이 증가하는 문제가 수정되었습니다
- `←` 또는 `/background`로 session을 background 처리할 때 agent view row에서 `/color`가 삭제되는 문제가 수정되었습니다
- 시작 복구 대화에서 손상된 config 파일을 재설정할 때 복구 불가능하게 파괴하는 문제가 수정되었으며, 이제 먼저 파일을 백업합니다
- Chrome의 Claude이 session이 다른 빌드 또는 config 디렉터리에서 실행될 때 재연결 페이지를 반복적으로 열었던 문제가 수정되었습니다
- Plan mode가 상태 변경 browser tool call에 대해 프롬프트하지 않던 문제가 수정되었으며, 읽기 전용 `browser_batch` 호출이 이제 올바르게 자동 허용됩니다
- 일시적 server 속도 제한 오류(사용 한도와 무관한 429)가 이제 구독자에게 backoff를 사용하여 자동으로 재시도되며 turn 실패로 인해 더 이상 실패하지 않습니다
- `CLAUDE_CODE_RETRY_WATCHDOG`이 비용량 일시적 오류에 대한 기본 재시도 수를 300으로 높이고 `CLAUDE_CODE_MAX_RETRIES`의 상한인 15를 제거합니다
- `claude agents` session row가 이제 pull-request link를 중복된 "PR" 레이블 없이 `#N`으로 표시합니다
