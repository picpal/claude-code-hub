## 변경사항

- `--plugin-url <url>` 플래그를 추가하여 현재 세션에서 URL로부터 plugin `.zip` 아카이브를 가져올 수 있도록 변경
- `CLAUDE_CODE_FORCE_SYNC_OUTPUT=1` env var을 추가하여 자동 감지가 누락된 터미널에서 synchronized output을 강제 활성화할 수 있도록 변경 (예: Emacs `eat`)
- `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE`를 추가했으며, Homebrew 또는 WinGet 설치 환경에서 설정 시 Claude Code가 백그라운드에서 업그레이드 명령을 실행하고 재시작을 알림
- Plugin manifest: `themes` 및 `monitors`는 이제 `"experimental": { ... }` 아래에 선언되어야 함. 최상위 선언도 여전히 작동하지만 `claude plugin validate`가 경고 표시
- Gateway `/v1/models` discovery for the `/model` picker은 이제 `CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1`을 통해 선택 사항으로 변경 (2.1.126–2.1.128에서는 자동)
- Ctrl+R history picker은 이제 기본적으로 모든 프로젝트 및 세션의 모든 prompt를 검색하도록 변경되었으며, 2.1.124 이전 동작과 일치. Ctrl+S를 눌러 현재 프로젝트 또는 세션으로 범위 축소
- Third-party deployment (Bedrock, Vertex, Foundry, 또는 `ANTHROPIC_BASE_URL` gateway)는 더 이상 first-party Anthropic surface를 가리키는 spinner tip을 표시하지 않음
- `skillOverrides` 설정이 이제 작동: `off`는 model 및 `/`에서 숨김, `user-invocable-only`는 model에서만 숨김, `name-only`는 설명 축소
- `claude_code.pull_request.count` OTel metric은 이제 shell command뿐만 아니라 MCP tool을 통해 생성된 PR/MR도 카운트
- Policy refusal error message에 더 쉬운 지원 디버깅을 위해 API Request ID 포함
- 인식되지 않은 400 status code의 API error가 기본 error message 대신 raw JSON을 표시하는 문제 수정
- `/clear`가 대화 후 terminal tab title을 리셋하지 않는 문제 수정
- `/rename`의 session title chip이 permission 또는 다른 dialog가 활성화 상태일 때 사라지는 문제 수정
- Subagent가 실행 중일 때 prompt 아래의 agent panel이 숨겨지는 문제 수정 (2.1.122의 regression)
- External-editor handoff (Ctrl+G)가 prompt 위의 conversation history를 지우는 문제 수정
- `/context`가 rendered ASCII visualization grid을 conversation에 dump하여 호출당 약 1.6k token을 낭비하는 문제 수정
- `/agents` Library list arrow-key navigation: 목록이 viewport를 초과할 때 highlighted agent이 이제 visible 상태로 유지되는 문제 수정
- `/branch` success message이 `/resume`을 위한 새 branch의 session id를 포함하지 않는 문제 수정
- Fullscreen mode에서 keycap/ZWJ/skin-tone emoji가 있는 bold header이 후행 문자를 손실하는 문제 수정
- `user:inference` scope이 부족한 stored OAuth credential을 가진 enterprise/team user의 경우 server-managed settings policy이 적용되지 않는 문제 수정
- Wake-from-sleep 후 OAuth refresh race가 모든 running session을 로그아웃하는 문제 수정
- 1-hour prompt cache TTL이 5분으로 자동 다운그레이드되는 문제 수정
- `/clear` 또는 `/effort` 또는 `/model` 변경 후 compaction 시 cache-miss warning이 불필요하게 나타나는 문제 수정
- `Bash(mkdir *)`, `Bash(touch *)` 등의 allow rule이 in-project path에서 honored되지 않는 문제 수정
- `*://` scheme wildcard가 있는 `deniedMcpServers` pattern이 mixed-case hostname과 일치하지 않는 문제 수정
- Voice mode에서 `--debug` 중 무해한 WebSocket warning이 error로 로깅되는 문제 수정
- [VSCode] `/clear`가 conversation context 및 displayed transcript를 지우지 않는 문제 수정
