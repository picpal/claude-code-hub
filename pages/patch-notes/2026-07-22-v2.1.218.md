## 변경사항

- `/code-review`을 background subagent로 실행하도록 변경하여 review 작업이 더 이상 대화를 채우지 않으며 stacked slash commands를 review 대상으로 유지합니다
- `--ax-screen-reader` 모드에서 단어 및 줄 삭제(`Option+Delete`, `Ctrl+W`, `Cmd+Backspace`, `Ctrl+U`, `Ctrl+K`)에 대한 screen-reader 공지사항을 추가했습니다
- Windows 경로에서 `\u` 접두사 세그먼트(예: `C:\Users\unicorn`)가 tool inputs에서 CJK 문자로 손상되어 해당 파일에 접근할 수 없던 문제를 수정했습니다
- 좌측 화살표 키가 undo 없이 대화를 폐기하는 문제를 수정했습니다: 편집 직후 누를 때 확인하라고 요청하며, agent view에서 Esc는 backgrounded된 대화로 돌아갑니다
- server 연결 실패 시 HTTP 상태 및 오류 텍스트를 `claude mcp list` 및 `/mcp`에 추가했으며, hidden leading 또는 trailing whitespace가 있는 MCP config 값에 대한 경고를 추가했습니다
- terminal에서 붙여 넣은 newlines를 Ctrl+J로 인코딩하는 경우 multi-line paste가 한 줄로 붕괴되며 `j`가 newlines 자리에 들어가는 문제를 수정했습니다
- message picker에서 compacting한 후 `/context`가 stale pre-compact token usage를 보고하는 문제를 수정했습니다
- `/ultrareview`가 "review my auth changes" 같은 descriptive arguments에서 실패하는 문제를 수정했습니다 — 이제 현재 branch의 review를 실행하며 텍스트를 findings에 대한 note로 적용합니다
- non-interactive sessions에서 `/code-review ultra`가 silent하게 local review를 실행하는 문제를 수정했습니다 — 이제 cloud review를 시작합니다
- gateway spend metering을 Bedrock application-inference-profile ARNs 및 기타 config-mapped upstream model IDs에 대해 configured model의 요율로 가격 책정하도록 수정했습니다
- long IDE selection이 emoji 중간에서 truncated되었을 때의 mojibake를 수정했으며, tool executor error가 silent하게 dropped될 수 있었던 경우를 수정했습니다
- engine teardown race가 phantom turn을 시작하고 중단할 수 있었던 문제를 수정했으며, close 후 pushed된 input을 consistently하게 거부하도록 만들었습니다
- interrupted tool calls 후 spurious "[Request interrupted by user]" 메시지를 수정했으며, tool이 mid-response에서 중단되었을 때 transcript에 남겨진 unpaired `tool_use` block을 수정했습니다
- `--ax-screen-reader` 모드에서 VoiceOver가 input 끝의 typed space를 echo 대신 "new line"으로 읽는 문제를 수정했습니다
- plugin 및 settings panels이 focused row로 terminal cursor를 이동하지 않아 screen readers 및 magnifiers가 arrow-key navigation을 따를 수 없던 문제를 수정했습니다
- deeply nested watched directory tree가 삭제되거나 이동되었을 때 및 deeply nested UI trees를 렌더링할 때 crashes(maximum call stack exceeded)를 수정했습니다
- session이 PR을 생성하거나 link한 직후 종료되었을 때 pull request events가 occasionally 손실되는 문제를 수정했습니다
- Bedrock setup wizard가 partitioned AWS regions 및 proxy-only networks에서 assume-role profiles에 대해 profile verification에 실패하는 문제를 수정했습니다
- system clock adjustment 후 rare negative 또는 incorrect turn duration measurements를 monotonic clock으로 timing turns하여 수정했습니다
- "N MCP servers need authentication" startup notice가 claude.ai에서 연결되지 않은 claude.ai connectors를 over-counting하는 문제를 수정했습니다
- history writes가 race되거나 실패했을 때 prompt history entries가 dropped 또는 duplicated되는 문제를 수정했습니다
- large thinking budget과 context-overflow error 후 retry loop가 identical doomed requests를 re-sent하는 문제를 수정했습니다; `Ctrl+B` backgrounding은 이제 다른 경로와 동일한 background-shell caps을 적용합니다
- untrusted folders에서 agent frontmatter hooks가 실행되는 문제를 수정했습니다: hooks는 이제 agent file의 자체 folder가 workspace trust를 수락하도록 요구합니다
- fork-session lineage가 headless 및 SDK sessions에서 compaction 후 손실되는 문제를 수정했습니다
- resumed session이 malformed delta attachment를 보유한 history를 가지고 있을 때 모든 turn에서 실패하거나 resume에서 crashing하는 문제를 수정했습니다
- `/ultrareview` error feedback을 개선하여 Claude가 invalid argument를 unchanged하게 retry하는 대신 correct할 수 있도록 했습니다
- auto mode를 개선했습니다: dangerous-rm, background-`&`, 및 suspicious-Windows-path checks가 더 이상 permission dialogs를 열지 않으며, auto-mode classifier가 이를 adjudicate합니다
- IDE interactions에 대한 sandbox command restrictions을 개선했습니다
- trust dialogs를 개선하여 grant이 covering하는 repository root을 name으로 표시합니다
- `/deep-research`를 manually 호출될 때만 시작하도록 변경했습니다; Claude가 더 이상 자동으로 시작하지 않습니다
- plan mode with auto를 변경하여 static analyzer가 read-only로 증명할 수 없는 Bash commands를 더 이상 prompt하지 않습니다; auto-mode classifier가 이를 judge합니다
- fast mode가 `/config model=<x>` 또는 Remote Control을 통해 models을 switching한 결과로 변경될 때 announcement를 추가했습니다
- server-managed settings를 변경하여 benign feature 및 cost toggles가 더 이상 settings-approval prompt를 trigger하지 않습니다
- agent markdown files을 변경하여 `:` 포함하는 agent names를 reject합니다(plugin namespacing에 예약됨)
- `context: fork`인 skills를 기본으로 background에서 실행하도록 변경했습니다; per skill로 `background: false`로 opt out합니다
- skill 및 plugin frontmatter booleans에 대해 `true`/`false`와 함께 `yes`/`no`/`on`/`off`/`1`/`0`(case-insensitive)을 수락되는 값으로 추가했습니다
- remote sessions가 worker를 replaced한 후에도 heartbeats를 계속 보내어 long-lived desktop 및 IDE processes가 rejected request를 매 몇 초마다 계속 retry하는 문제를 수정했습니다
