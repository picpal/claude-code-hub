## 변경된 사항
- 모델 선택기 항목을 사용자 정의 제공자 모델 ID(예: Bedrock 추론 프로필 ARN)에 매핑하기 위한 `modelOverrides` 설정을 추가했습니다.
- SSL 인증서 오류(기업용 프록시, `NODE_EXTRA_CA_CERTS`)로 인해 OAuth 로그인 또는 연결 확인이 실패할 경우, 구체적인 해결 방법을 안내하도록 개선했습니다.
- 복잡한 bash 명령어 실행 시 권한 요청 창으로 인해 발생하던 시스템 멈춤 및 CPU 사용률 100% 고착 현상을 수정했습니다.
- 많은 스킬 파일이 한꺼번에 변경될 때(예: `.claude/skills/` 디렉터리가 큰 저장소에서 `git pull`을 실행할 때) Claude Code가 멈출 수 있는 교착 상태를 수정했습니다.
- 동일한 프로젝트 디렉터리에서 여러 Claude Code 세션을 실행할 때 Bash 도구 출력이 누락되던 문제를 수정했습니다.
- Bedrock, Vertex 및 Microsoft Foundry 환경에서 `model: opus`/`sonnet`/`haiku`를 사용하는 서브에이전트가 아무런 경고 없이 이전 모델 버전으로 다운그레이드되던 문제를 수정했습니다.
- 에이전트가 종료될 때 서브에이전트가 생성한 백그라운드 bash 프로세스가 정리되지 않던 문제를 수정했습니다
- 선택기에서 `/resume` 명령어를 실행했을 때 현재 세션이 표시되던 문제를 수정했습니다
- 확장 프로그램을 자동 설치할 때 `/ide`가 `onInstall is not defined` 오류와 함께 강제 종료되는 문제를 수정했습니다.
- Bedrock/Vertex/Foundry 환경에서, 또는 텔레메트리 기능이 비활성화된 상태에서 `/loop` 명령어가 작동하지 않던 문제를 수정했습니다.
- `--resume` 또는 `--continue`를 통해 세션을 재개할 때 SessionStart 훅이 두 번 호출되던 문제를 수정했습니다.
- 매 턴마다 모델의 컨텍스트에 아무런 동작도 수행하지 않는 시스템 알림 메시지를 삽입하던 JSON 출력 훅 문제를 수정했습니다
- 연결 속도가 느린 상태에서 새로운 녹음이 중복될 때 발생하는 음성 모드 세션 오류 수정
- 네이티브 빌드 시 "ripgrep (rg) not found" 오류로 인해 Linux 샌드박스가 시작되지 않는 문제를 수정했습니다.
- Amazon Linux 2 및 기타 glibc 2.26 시스템에서 Linux 네이티브 모듈이 로드되지 않던 문제를 수정했습니다.
- 리모트 컨트롤을 통해 이미지를 수신할 때 발생하는 “media_type: 필드 필수” API 오류를 수정했습니다.
- 바탕화면 폴더가 이미 존재할 경우 Windows에서 `/heapdump` 명령어가 `EEXIST` 오류와 함께 실패하던 문제를 수정했습니다.
- 클로드의 대사를 중단한 후 위쪽 화살표 키를 누를 때의 동작이 개선되었습니다. 이제 중단된 대사를 복원하고 대화를 한 번에 되감아 줍니다.
- 시작 시 IDE 감지 속도 개선
- macOS에서 클립보드 이미지 붙여넣기 성능이 개선되었습니다
- 클로드가 응답하는 동안에도 `/effort`가 작동하도록 개선하여, `/model`의 동작과 일치하도록 했습니다
- 음성 모드를 개선하여, 푸시투톡(PTT) 버튼을 빠르게 다시 누를 때 발생하는 일시적인 연결 오류가 발생하면 자동으로 재시도하도록 했습니다.
- 상황에 더 잘 맞는 리모컨 스폰 모드 선택 안내 메시지를 개선했습니다
- Bedrock, Vertex 및 Microsoft Foundry의 기본 Opus 모델을 Opus 4.1에서 Opus 4.6으로 변경했습니다.
- 더 이상 사용되지 않는 `/output-style` 명령어 — 대신 `/config`를 사용하세요. 프롬프트 캐싱 성능을 개선하기 위해 출력 스타일은 이제 세션 시작 시 고정됩니다.
- VSCode: 프록시를 통해 접속하거나 Bedrock/Vertex 환경에서 Claude 4.5 모델을 사용하는 사용자의 HTTP 400 오류 문제를 수정했습니다.
