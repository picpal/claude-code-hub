> 원문: [Claude Code v2.1.172 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.172)

## 변경사항

- Sub-agent가 이제 자신의 sub-agent를 생성할 수 있습니다 (최대 5단계 깊이)
- Amazon Bedrock은 이제 `AWS_REGION`이 설정되지 않았을 때 `~/.aws` 설정 파일에서 AWS region을 읽으며, AWS SDK 우선순위와 일치합니다. `/status`에 region이 어디서 왔는지 표시됩니다
- `/plugin`에서 마켓플레이스의 플러그인을 탐색할 때 검색 표시줄을 추가했습니다
- `claude_code.lines_of_code.count` OTEL metric에 `model` 속성을 추가했습니다
- 1M context를 사용하면서 사용 크레딧이 없는 세션이 영구적으로 막히는 문제를 수정했습니다. 이제 세션이 자동으로 표준 context 제한 이하로 압축됩니다
- 대화에 여러 이미지가 포함되어 있을 때 반복되는 "an image in the conversation could not be processed and was removed" 오류를 수정했습니다
- worker가 응답한 후 최대 30초 동안 agent 뷰가 Working 상태에서 바쁜 스피너를 유지하는 문제를 수정했습니다
- background agent가 pre-warmed worker에 디스패치될 때 다른 디렉터리의 프로젝트 설정 (`.mcp.json` 승인, 신뢰)을 읽을 가능성이 있는 문제를 수정했습니다
- daemon이 자동으로 업데이트된 후 이전 버전에서 시작된 세션의 background-session attach가 EAUTH로 실패하는 문제를 수정했습니다
- nested agent가 중지된 후 background sub-agent가 agent 패널에서 "active" 상태로 계속 막혀 있는 문제를 수정했습니다
- `claude agents` dispatch input에서 `/model` 제안이 오해의 소지가 있는 슬래시 접두사로 렌더링되고 조직에서 비활성화된 모델을 표시하는 문제를 수정했습니다
- `availableModels` 제한이 subagent 모델 오버라이드, agent dispatch 모델 피커, advisor 모델에 적용되지 않는 문제를 수정했습니다
- `availableModels` allowlist가 항목이 `claude-opus-4-8`과 같은 버전별 ID를 사용할 때 `/model` 피커의 Opus 및 Sonnet 1M 행을 숨기는 문제를 수정했습니다
- Bedrock의 `/model` 피커가 provider가 제공하지 않는 모델을 제공하는 문제를 수정했습니다. 하나를 선택하면 세션 모델이 자동으로 전환되고 여러 행의 선택 마커가 켜졌습니다
- `ANTHROPIC_DEFAULT_OPUS_MODEL`이 이미 1M-context 접미사를 포함할 때 model ID가 doubled 1M-context 접미사를 받는 문제 (예: `[1M][1m]`)를 수정했습니다
- `opusplan` 모델 설정이 entitled user를 위한 plan mode에서 1M context를 제공하지 않는 문제를 수정했습니다. `opusplan[1m]` 임시 해결책은 이제 plan mode에서 Opus로 올바르게 전환됩니다
- `WebFetch(domain:*.example.com)` wildcard domain 규칙이 allow, deny, ask 위치의 subdomain과 일치하지 않는 문제와 mid-pattern wildcard가 포함된 파일 권한 규칙 (예: `Read(secrets-*/config.json)`)이 시작 시 거부되는 문제를 수정했습니다
- sub-agent의 chat 탭이 열려 있을 때 up-arrow prompt history가 main agent의 프롬프트를 표시하는 문제를 수정했습니다
- memory recall이 remote session에서 mounted team memory store (`CLAUDE_MEMORY_STORES`)를 찾지 못하는 문제를 수정했습니다
- workflow validation이 prompt 문자열이나 주석이 단순히 `Date.now()`/`Math.random()`을 언급하는 스크립트를 거부하는 문제를 수정했습니다
- Windows console에서 완전히 지원되지 않는 mouse tracking을 비활성화했습니다
- `/plugin` 마켓플레이스 목록이 긴 플러그인 목록에서 나온 후 커서를 잃는 문제와 플러그인 브라우저에서 Esc를 누르면 잘못된 탭으로 돌아가는 문제를 수정했습니다
- 중복된 message normalization을 제거하고 streaming tool-use state가 변경되지 않았을 때 전체 message-history transform을 피함으로써 긴 대화에서의 성능을 개선했습니다
- idle CPU 사용량 감소: `/goal` status chip이 유휴 상태에서 5 Hz로 terminal을 다시 렌더링하지 않으며, subagent가 병렬로 실행되는 동안 더 적은 UI 재렌더링이 발생합니다
- Claude in Chrome tool 로딩 개선: browser tool이 이제 tool별로 하나씩이 아닌 단일 배치 호출로 로드됩니다
- 비대화형 Usage Policy 거부 메시지를 개선하여 새 세션을 시작하거나 모델을 변경할 것을 제안합니다
- `/code-review`는 이제 claude.ai에 로그인하지 않았을 때 `ultra` 옵션을 표시로 유지하며, cloud review가 claude.ai 계정이 필요함을 설명합니다
- Remote Control footer indicator를 "/rc active"로 단축하고 좁은 terminal에서 숨겼습니다
- remote session에서 `/loop` 홍보를 중단했습니다. 여기서는 pending loop가 container를 활성 상태로 유지하지 않습니다
- [VSCode] PowerShell tool call이 원본 JSON 대신 적절한 command display 및 permission dialog로 렌더링되고 표시된 shell output에서 ANSI escape code를 제거했습니다
