## 변경사항

- `EnterWorktree` tool에 `path` parameter를 추가하여 현재 저장소의 기존 worktree로 전환할 수 있도록 함
- PreCompact hook 지원 추가: hook이 이제 종료 코드 2를 반환하거나 `{"decision":"block"}`을 반환하여 compaction을 차단할 수 있음
- 플러그인을 위한 background monitor 지원 추가: 상위 레벨의 `monitors` manifest key를 통해 세션 시작 또는 skill 호출 시 자동으로 활성화됨
- `/proactive`가 이제 `/loop`의 alias임
- stalled API stream 처리 개선: stream이 이제 5분간 데이터가 없으면 중단되고 무한 대기 대신 non-streaming으로 재시도됨
- 네트워크 오류 메시지 개선: 연결 오류가 이제 조용한 spinner 대신 즉시 재시도 메시지를 표시함
- 파일 쓰기 표시 개선: 긴 한 줄 쓰기(예: minified JSON)가 이제 여러 화면에 걸친 페이지 나누기 대신 UI에서 잘림
- `/doctor` layout 개선: 상태 아이콘 추가; `f`를 눌러 Claude가 보고된 문제를 수정하도록 함
- `/config` labels 및 descriptions를 명확성을 위해 개선
- skill description 처리 개선: listing cap을 250에서 1,536자로 높이고 descriptions이 잘릴 때 startup warning을 추가
- `WebFetch`를 개선하여 가져온 페이지에서 `<style>` 및 `<script>` contents를 제거하므로 CSS가 많은 페이지가 실제 텍스트에 도달하기 전에 content budget를 소진하지 않음
- stale agent worktree cleanup을 개선하여 squash-merge된 PR의 worktree를 제거하고 무한정 유지하지 않도록 함
- MCP 대규모 출력 잘림 prompt를 개선하여 format별 레시피(예: JSON의 경우 `jq`, text의 경우 computed Read chunk sizes) 제공
- queued messages(Claude가 작업 중일 때 전송된)에 첨부된 images가 dropped되는 문제 수정
- 긴 대화에서 prompt input이 두 번째 줄로 wrapping될 때 화면이 공백이 되는 문제 수정
- fullscreen mode에서 multi-line assistant responses를 선택할 때 leading whitespace가 복사되는 문제 수정
- assistant messages에서 leading whitespace가 잘려서 ASCII art 및 indented diagrams를 깨뜨리는 문제 수정
- commands가 clickable file links(예: Python `rich`/`loguru` logging)를 출력할 때 bash output이 garbled되는 문제 수정
- ESC-prefix alt encoding을 사용하는 terminals에서 alt+enter가 newline을 삽입하지 않고 Ctrl+J가 newline을 삽입하지 않는 문제 수정(2.1.100에서의 regression)
- EnterWorktree/ExitWorktree tool display에서 중복된 "Creating worktree" 텍스트 수정
- queued user prompts가 focus mode에서 사라지는 문제 수정
- file watcher가 post-fire cleanup을 놓쳤을 때 one-shot scheduled tasks가 반복적으로 re-fire되는 문제 수정
- inbound channel notifications이 Team/Enterprise users의 경우 첫 번째 메시지 후 silently dropped되는 문제 수정
- `package.json` 및 lockfile이 있는 marketplace plugins이 install/update 후 dependencies가 자동으로 설치되지 않는 문제 수정
- marketplace auto-update가 plugin process가 update 중에 파일을 열고 있을 때 official marketplace를 broken 상태로 남기는 문제 수정
- `/resume`, `--worktree`, 또는 `/branch` 후 exit 시 "Resume this session with..." hint가 출력되지 않는 문제 수정
- 더 긴 prompt의 끝에서 입력할 때 feedback survey shortcut keys가 firing되는 문제 수정
- stdio MCP server가 malformed(non-JSON) output을 emit할 때 세션이 hanging되는 대신 "Connection closed"로 빠르게 실패하는 문제 수정
- MCP tools이 MCP servers가 asynchronously 연결될 때 headless/remote-trigger sessions의 첫 번째 turn에서 missing되는 문제 수정
- AWS Bedrock의 non-US regions에서 inference profile discovery가 여전히 in-flight일 때 `/model` picker가 invalid `us.*` model IDs를 `settings.json`에 persist하는 문제 수정
- 429 rate-limit errors가 API-key, Bedrock, 및 Vertex users의 경우 clean message 대신 raw JSON dump를 표시하는 문제 수정
- session이 malformed text blocks를 포함할 때 resume 시 crash되는 문제 수정
- `/help`가 짧은 terminal heights에서 tab bar, Shortcuts heading, 및 footer를 dropping하는 문제 수정
- malformed keybinding entry values이 `keybindings.json`에서 clear error로 거부되지 않고 silently loaded되는 문제 수정
- 한 프로젝트의 settings에서 `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC`이 머신의 모든 프로젝트에 대해 permanently usage metrics를 비활성화하는 문제 수정
- Ghostty, Kitty, Alacritty, WezTerm, foot, rio, 또는 Contour를 SSH/mosh 위에서 사용할 때 washed-out 16-color palette 문제 수정
- Bash tool이 plan mode에서 exit할 때 더 높은 permission level에서 downgrade할 수 있으므로 `acceptEdits` permission mode를 suggest하는 문제 수정
