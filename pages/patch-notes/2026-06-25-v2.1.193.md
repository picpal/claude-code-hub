## 변경사항

- `autoMode.classifyAllShell` 설정을 추가하여 모든 Bash/PowerShell 명령어를 자동 모드 분류기를 통해 라우팅하도록 설정 (임의의 코드 실행 패턴만 라우팅하던 방식에서 변경)
- auto-mode 거부 사유를 transcript, 거부 toast 및 `/permissions` 최근 거부 목록에 추가
- `claude_code.assistant_response` OpenTelemetry log event를 추가하여 모델의 응답 텍스트 포함. `OTEL_LOG_ASSISTANT_RESPONSES=1`로 설정하지 않으면 redacted 상태; 변수가 설정되지 않은 경우 `OTEL_LOG_USER_PROMPTS`를 따르므로, 이미 prompt 내용을 로깅하는 배포는 업그레이드 시 응답 내용 수신을 시작함 — prompts만 유지하려면 `OTEL_LOG_ASSISTANT_RESPONSES=0`으로 설정
- bash mode (`!`)에 라이브 파일 경로 자동완성 추가
- MCP servers가 인증을 필요로 할 때 `/mcp`를 가리키는 시작 안내 추가
- idle 배경 shell 명령어에 대한 자동 메모리 압력 reaping 추가 (`CLAUDE_CODE_DISABLE_BG_SHELL_PRESSURE_REAP=1`로 비활성화 가능)
- `/login` 직후 `/model` 및 기타 client-data-gated UI가 stale/empty 상태를 보이는 문제 수정
- 모든 실행 중인 작업이 새 세션으로 이월될 때 backgrounding (←←)이 "N background tasks would be abandoned"로 인해 잘못 취소되는 문제 수정
- auto-update 후 매번 "Continue from where you left off" 재프롬프트를 받던 pinned background agents 수정
- 메인 turn을 backgrounding하면 메인 대화를 재실행하는 phantom "general-purpose (resumed)" subagent가 생성되는 문제 수정
- subagent 조회 시 agent panel이 sibling agents를 숨기는 문제 수정
- Background agents 개선: launch result가 더 이상 Claude에게 "end your response" 지시하지 않음 — agent가 실행되는 동안 다른 작업에 계속 작업
- MCP `headersHelper` auth 개선: helper는 tool call이 401/403을 반환할 때 자동으로 재실행 및 재연결
- Plugin auto-rename 개선: marketplace `renames` maps가 이제 자동으로 추적되어 설정을 새로운 이름으로 업데이트
- 디렉터리가 이미 working directory일 때의 `/add-dir` 메시지 개선
