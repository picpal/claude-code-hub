## 변경사항

- `Tool(param:value)` 구문을 permission rules에 추가하여 tool의 입력 매개변수를 일치시킬 수 있도록 함 (`*` 와일드카드 지원), 예: `Agent(model:opus)`로 Opus subagent 차단
- 중첩된 `.claude/skills` 디렉토리의 skill이 해당 파일을 작업할 때 로드되며, 이름 충돌 시 중첩된 skill은 `<dir>:<name>`으로 표시되어 둘 다 사용 가능
- 중첩된 `.claude/` 디렉토리: 이름 충돌 시 작업 디렉토리에 가장 가까운 agent, workflow, output-style이 우선 적용되며, project-scope workflow 저장은 이제 가장 가까운 기존 `.claude/workflows/`를 대상으로 함
- Auto mode 개선: subagent spawn이 이제 실행 전 classifier에 의해 평가되어, subagent가 차단된 작업을 검토 없이 요청할 수 있는 격차를 해소
- `/doctor` 개선: 모든 섹션에서 일관된 flat tree layout, 더 명확한 섹션 상태 아이콘, 강조된 command 이름
- Skill 목록 잘림 경고 개선: 설명이 영향을 받는 skill이 몇 개인지 표시
- Workflow prompt keyword를 보라색 shimmer highlight로 변경하고, 단어의 "run a workflow" 또는 "workflow:" 같은 명시적 구문에서만 트리거되도록 변경 (단어의 모든 언급에서 트리거되지 않음)
- Remote Control 오류 메시지 개선: 연결 실패는 이제 footer에 지속적인 빨간 "/rc failed" 표시기를 표시하며, "not yet enabled" 오류는 이제 gate인지, check 실패인지, 만료된 entitlement인지, 또는 org policy인지 설명
- `/bug`는 이제 제출 전 설명이 필수이며, 더 이상 model-refusal text를 GitHub issue title로 사용하지 않음
- 상위 프로세스에서 stale websocket/OAuth file-descriptor 환경 변수를 상속할 때 CLI가 crash(out-of-memory)되는 문제 수정
- OAuth token이 Claude Code login의 다른 계정에 속할 때 Chrome의 Claude가 조용히 연결 실패하는 문제 수정
- 중첩된 `.claude/skills` skill이 directory-qualified 이름으로 non-interactive run에서 permission prompt에 의해 차단되는 문제 수정
- 여러 subagent 문제 수정: subagent의 transcript 보기는 이제 tool result와 live progress를 표시하고, 해당 turn을 마치는 동안 전송된 메시지는 더 이상 삭제되지 않으며, 실행 중인 subagent를 background로 전환(ctrl+b)해도 처음부터 다시 시작되지 않음
- `claude agents` worker가 daemon이 `ANTHROPIC_BASE_URL` 및 `ANTHROPIC_AUTH_TOKEN`을 통해 custom API gateway가 있는 shell에서 시작되었을 때 `401 Invalid bearer token`으로 실패하는 문제 수정
- Compaction이 `--fallback-model`을 적용하지 않는 문제 수정: compaction은 이제 overload 또는 model-availability 오류 시 구성된 fallback model chain으로 fallback
- Credential이 세션 외부에서 새로고침된 후 stale cached request configuration으로 인해 model request가 계속 auth 오류로 실패하는 문제 수정
- turn이 완료된 후 `/bg` 또는 `←←`로 생성된 background session이 agents list에서 "Working"을 영구적으로 표시하는 문제 수정
- `CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE=1`이 fresh marketplace install이 clone되는 것을 방지하는 문제 수정
- Subagent `disallowedTools`의 MCP server-level spec (`mcp__server`, `mcp__server__*`, `mcp__*`)이 조용히 무시되는 문제 수정
- Vim mode undo: `u`가 이제 명령을 fast succession에서 하나의 undo step으로 병합하는 대신 NORMAL/VISUAL-mode command를 하나씩 단계별로 진행
- Custom URI scheme(`vscode://` 등)을 사용하는 statusline link가 `claude agents`에서 클릭해도 열리지 않는 문제 수정
- [VSCode] CJK IME candidate window를 dismiss하기 위해 Esc를 누를 때 실행 중인 Claude task가 취소되는 문제 수정
