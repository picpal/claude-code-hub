## 변경사항

- `claude agents --json`은 이제 waiting session이 어디에서 차단되고 있는지 보여주는 `waitingFor`를 포함합니다 (예: permission prompt)
- `--tools`: Grep/Glob을 명시적으로 나열하면 이제 embedded search가 있는 native builds에서 dedicated search tools을 제공합니다 (이전에는 이 이름들이 자동으로 무시되었습니다)
- `/effort`는 이제 선택한 레벨이 새 sessions의 기본값으로 유지될 것임을 확인합니다
- 자동완성 메뉴에서 slash command를 클릭하면 즉시 실행하는 대신 prompt에 채워집니다. Enter를 눌러 실행하세요
- Remote Control은 이제 startup message 대신 persistent footer pill로 표시됩니다 (session으로 가는 링크 포함)
- editor의 rebrand에 따라 `/ide` 메뉴, `/terminal-setup`, `/scroll-speed`에서 Windsurf의 이름을 Devin Desktop으로 변경했습니다
- config directory가 read-only이거나 쓸 수 없을 때 발생하던 silent startup hang을 수정했습니다 — Claude Code는 이제 in-memory config로 시작하고 blank screen 대신 startup errors를 표시합니다
- WebFetch permission rules이 built-in preapproved domains에 적용되지 않던 문제를 수정했습니다. 명시적인 `WebFetch(domain:...)` deny/ask/allow rules이 이제 preapproved-host auto-allow보다 우선합니다
- Windows permission rules이 backslashes (`~\`, `\\server\share`)나 case-variant paths로 작성될 때 매칭되지 않던 문제와 Read deny rules이 Glob/Grep 결과에서 파일을 숨기지 않던 문제를 수정했습니다
- stream-json/SDK sessions에서 turn의 매우 시작 부분에 전송된 interrupt (Esc)가 자동으로 삭제되어 "Interrupted" feedback 없이 turn이 실행되던 문제를 수정했습니다
- classifier side-queries와 truncation boundary 근처에 emoji가 포함된 MCP server descriptions에 대한 API 400 `no low surrogate in string` 오류를 수정했습니다
- MCP per-server `timeout` config values가 1000 ms 미만일 때 모든 tool call을 중단하는 1초 watchdog로 floored되던 문제를 수정했습니다. 이제 sub-1000 ms values는 무시되며 (`MCP_TOOL_TIMEOUT` 또는 기본값으로 fallback), `claude mcp get`이 이에 따라 annotation을 붙입니다
- LSP tool의 `workspaceSymbol` operation이 결과를 반환하지 않던 문제를 수정했습니다. 이제 `query` parameter를 허용하고 이를 language server에 전달합니다
- `claude agents`가 넓은 terminals에서 live status text (tool args, replies, prompts, exec output)를 60–120 columns에서 자르던 문제를 수정했습니다. 이제 status detail은 전체 terminal width를 사용합니다
- `claude agents`가 긴 session names를 40 columns에서 자르던 문제를 수정했습니다. 이제 name column은 terminal width와 함께 커집니다
- `claude agents` attach가 background-service restart 후 처음 시도에서 session list로 튕겨나가는 문제를 수정했습니다
- `claude agents` Ctrl+V image paste가 dispatch input과 session reply box에서 작동하지 않던 문제를 수정했습니다. 이미지 없이 붙여넣으면 이제 hint를 표시합니다
- ← 로 session을 backgrounding할 때 background service가 시작되지 못하면 대화가 자동으로 삭제되던 문제를 수정했습니다. 이제 session은 list에 실패한 행으로 남으며 Enter로 깨울 수 있습니다
- agents view에서 전송에 실패한 replies가 손실되던 문제를 수정했습니다. 이제 다음 session start 시 전달을 위해 queued됩니다
- cross-session messaging (`SendMessage`)가 `CLAUDE_CODE_TMPDIR` 또는 `$TMPDIR`이 깊은 directory를 가리킬 때 자동으로 손상되던 문제를 수정했습니다
- `claude agents`에서 실행 중인 background session을 열 때 attach 전에 5초 동안 stalling되던 문제를 수정했습니다
- Quieter startup: notices는 severity별로 그룹화되고, session info와 announcements는 launch당 한 줄을 공유합니다
- Startup warnings를 더 짧고 명확하게 다시 작성했으며, 각각 구체적인 fix를 포함합니다
- Launch-prompt warnings (deep link/pre-filled prompt)는 이제 action을 취할 때까지 input 아래에 pinned되어 있으며 스크롤되지 않습니다
- 실패한 turns는 이제 multi-line red error block 대신 compact warning line을 표시합니다
- Background service startup과 `claude update` verification을 개선하여 새 binaries의 endpoint-security scanning이 끝날 때까지 대기하므로 5초 후 실패하지 않습니다
- Background dispatch spawn failures는 이제 errno이 없을 때 error class name을 보고합니다
- "Claude in Chrome enabled" 및 "marketplace installed" startup messages를 제거했습니다. model auto-updates와 team-onboarding tip은 이제 logo 아래에 quiet notices로 표시됩니다
