## 변경사항

- `/config`에서 "Dynamic workflow size" 설정이 추가되었으며, 이는 Claude가 동적 workflow를 만드는 규모(small/medium/large agent count)를 제어합니다 — 강제되는 제한이 아닌 권장 지침입니다
- workflow가 생성한 agent에서 내보낸 telemetry에 `workflow.run_id` 및 `workflow.name` OpenTelemetry 속성이 추가되어 OTel 데이터에서 workflow 실행의 활동을 재구성할 수 있습니다
- 검색이 여전히 history 파일을 스캔하는 중 수락하거나 취소할 때 inline Ctrl+R history search에서 발생하는 충돌이 수정되었습니다
- background session에서 `/rename`이 job 재시작 시 되돌려지는 문제가 수정되었으며, 이로 인해 새 이름으로 session을 지정할 수 없었습니다
- in-place client certificate 회전 중 설정이 재적용될 때 발생하는 일시적인 mTLS handshake 실패가 수정되었습니다
- Remote Control(mobile/web)에서 interactive session으로 전송된 명령이 "Unknown command"로 실패하는 문제가 수정되었습니다
- Remote Control mobile 또는 web app에서 caption 없이 전송된 이미지 및 파일이 자동으로 삭제되는 문제가 수정되었습니다
- `claude auth login` 및 `claude mcp login --no-browser`에서 인쇄된 sign-in URL이 SSH를 통해 줄바꿈될 때 클릭 가능하지 않은 문제가 수정되었습니다 — 이제 단일 hyperlink로 내보내집니다
- `claude agents`에서 chat을 열 때 "currently running as a background agent"로 인해 실패하고 worker crash/respawn loop가 발생하는 문제가 수정되었습니다
- 문자열에 unicode quote escape가 있는 workflow script가 parsing 전에 손상되는 문제가 수정되었습니다. workflow parse 오류는 이제 항상 TypeScript를 지적하는 대신 문제가 되는 줄을 표시합니다
- microphone 또는 audio recorder에 오류가 발생할 때 voice dictation이 제한 없이 재시도하는 문제가 수정되었습니다 — 반복된 capture 실패는 이제 voice input을 일시 중지합니다
- `/remote-control` session이 mobile 및 web app에서 잘못된 permission mode를 표시하는 문제가 수정되었습니다
- 많은 git worktree가 있는 repository에서 session을 이름으로 재개하거나 resume picker를 열 때 수 분이 걸리고 많은 양의 메모리를 사용하는 문제가 수정되었습니다
- proxy 또는 network이 mid-download에서 연결을 끊을 때 installer 및 updater download가 "aborted"로 즉시 실패하는 문제가 수정되었습니다 — 일시적인 연결 끊김은 이제 재시도됩니다
- 이미 로드된 skill을 다시 호출할 때 context에 instruction의 중복 복사본이 추가되는 문제가 수정되었습니다
- `/workflows` agent 목록 레이아웃이 개선되었습니다: 더 넓은 제목, 전용 time 열, 짧은 model 이름, 행별 tool-call count 제거
- MCP 오류 메시지가 개선되었습니다: server config에 `url`은 있지만 `type`이 없을 때 더 명확한 오류 메시지로 "command: expected string" 대신 `"type": "http"`를 제안합니다
- `/review <pr>`이 빠른 single-pass review로 변경되었습니다. 선택한 effort level에서 multi-agent review를 원하시면 `/code-review <level> <pr#>`을 사용하세요
