> 원문: [Claude Code v2.1.132 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.132)

## 변경사항

- Bash tool subprocess 환경에 `CLAUDE_CODE_SESSION_ID` 환경 변수를 추가했으며, hooks에 전달되는 `session_id`와 일치하도록 변경했습니다
- fullscreen alternate-screen renderer를 옵트아웃하고 terminal의 native scrollback에서 대화를 유지하기 위해 `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1` env var를 추가했습니다
- Ctrl+V 이미지 paste가 clipboard에서 읽혀지는 동안 "Pasting…" footer hint를 추가했습니다
- 외부 SIGINT (예: IDE stop button, `kill -INT`)가 graceful shutdown을 실행하지 않는 문제를 수정했습니다 — terminal modes가 이제 복원되고 `--resume` hint가 출력되며 갑작스러운 종료가 아닙니다
- native build에서 session 중 terminal이 닫히거나 SSH가 끊길 때 발생하는 uncaught exception을 수정했습니다
- tool error truncation이 emoji를 분할할 때 `no low surrogate in string` 오류로 `--resume`이 실패하는 문제를 수정했습니다; 미리 손상된 sessions는 load 시 정제됩니다
- plan-mode session을 `-p --continue`/`--resume`으로 재개할 때 `--permission-mode` flag가 무시되는 문제를 수정했습니다. 또한 동일 session 내에서 `ExitPlanMode` 후 plan mode가 재적용되지 않는 문제도 수정했습니다
- laptop sleep/wake 또는 Ctrl+Z/`fg` 후 fullscreen mode가 다음 keystroke 또는 stream output까지 blank screen을 표시하는 문제를 수정했습니다
- Indic conjunct 또는 ZWJ emoji가 여러 줄에 걸쳐 있을 때 Ctrl+E/A/K/U/arrow keys에서 cursor가 mid-grapheme에 착지하는 문제를 수정했습니다
- vim operators가 decomposed (NFD) accented characters가 포함된 text를 손상시키는 문제를 수정했습니다
- `/`로 시작하는 text를 paste할 때 input이 조용히 무시되거나 unknown-command reply가 트리거되는 문제를 수정했습니다
- focus events 또는 mouse-tracking reports가 bracketed paste와 interleave될 때 stray escape sequences가 prompt에 dump되는 문제를 수정했습니다
- upstream xterm.js bug로 인해 Cursor 및 VS Code 1.92–1.104에서 mouse wheel scrolling이 너무 빠른 문제를 수정했습니다
- JetBrains IDE 2025.2 terminals에서 scroll-wheel handling 문제를 수정했습니다 (spurious arrow keys, wrong-direction events, runaway acceleration)
- Linux/X11에서 stats screenshot을 clipboard에 복사할 때 `/usage` Ctrl+S가 hanging되는 문제를 수정했습니다
- Windows Terminal에서 `/terminal-setup`이 모순된 오류를 표시하는 문제를 수정했습니다 — Shift+Enter는 그곳에서 natively 지원됩니다
- `/effort` picker가 `CLAUDE_CODE_EFFORT_LEVEL` env var override를 반영하지 않는 문제를 수정했습니다
- `/status`가 일부 사용자에게 잘못된 default model을 표시하는 문제를 수정했습니다
- slash command autocomplete popup이 약 3–5개의 visible commands로 제한되는 문제를 수정했습니다. 이제 terminal height에 따라 확장됩니다
- statusline `context_window` token counts가 current context usage 대신 cumulative session totals을 반영하는 문제를 수정했습니다
- macOS terminals에서 "Option as Meta"를 활성화하지 않으면 Alt+T (thinking toggle)가 작동하지 않는 문제를 수정했습니다 (iTerm2, Terminal.app defaults)
- `claude agents`에서 background session을 다시 열 때 Windows에서 dead keyboard input이 발생하는 문제를 수정했습니다
- stdio MCP server가 non-protocol data를 stdout에 쓸 때 unbounded memory growth (10GB+ RSS)가 발생하는 문제를 수정했습니다
- tools/list에서 연결되지만 실패하는 MCP servers가 `/mcp`에서 0개 tools를 조용히 표시하는 문제를 수정했습니다 — 이제 한 번 재시도하고 "connected · tools fetch failed"을 표시합니다
- unauthorized claude.ai MCP connectors가 "needs auth" 대신 "failed"로 표시되는 문제를 수정했습니다. headless `-p` mode는 이제 non-transient 4xx connection failures를 재시도하지 않습니다
- slash command dialogs 및 `/login`, `/upgrade`, `/extra-usage` dialog spacing의 visual consistency를 개선했습니다
- 추가 renderer benefits (lower memory usage, mouse support, auto-copy on select)를 설명하도록 `/tui fullscreen` startup banner를 업데이트했습니다
- `ENABLE_PROMPT_CACHING_1H`이 설정되어 있을 때 Bedrock 및 Vertex 400 오류를 수정했습니다
