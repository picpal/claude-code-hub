## 변경사항

- `sandbox.filesystem.disabled` 설정을 추가하여 네트워크 송신 제어를 유지하면서 파일시스템 격리를 건너뜀
- 메시지 정규화 비용이 턴 수에 따라 이차적으로 증가하여 여러 초의 정지 및 느린 재개를 유발하는 장기 세션의 속도 저하 수정
- OAuth 토큰이 만료되거나 세션 중에 회전된 후 "HTTP 401" 분류자 오류로 인해 자동 모드가 명령을 거부하는 현상 수정
- 귀하의 답변이 대기하거나 먼저 설명할 것을 요청했을 때도 AskUserQuestion이 Claude에게 계속하도록 지시하는 현상 수정 — 자유 텍스트 답변은 이제 중립적인 문구를 받음
- 웹의 Claude Code가 세션이 몇 분 동안 유휴 상태에 있은 후 동일한 질문을 다시 묻고 답변을 삭제하는 현상 수정
- @-멘션이 파일 수정 훅 후에 자동으로 아무것도 첨부하지 않음, vim `c`-연산자의 점 반복 및 붙여넣기, 재개 시 상태 표시줄이 두 번 실행되고 재개 선택기가 실패 시 응답하지 않는 현상 수정
- 재개된 백그라운드 에이전트 세션이 기본 에이전트로 되돌아가는 현상 수정 — 에이전트의 프롬프트 및 도구 제한이 이제 복원됨
- worktree 격리 서브에이전트가 `git -C`, `--git-dir` 또는 `GIT_DIR`/`GIT_WORK_TREE`를 통해 공유 checkout으로 git을 리디렉션하는 현상 수정
- 작업 디렉터리가 선택한 프로젝트와 일치하지 않을 때 worktree 세션이 다른 프로젝트의 남은 worktree에 들어가는 현상 수정
- git 저장소가 없는 worktree를 가진 백그라운드 세션을 삭제할 수 없는 현상 수정
- `claude daemon stop --any`가 오래된 레거시 daemon 잠금 파일을 통해 관련 없는 프로세스를 종료할 수 있는 현상 수정
- 백그라운드 작업이 있는 장기 실행 세션에서 유휴 프롬프트에서 Esc-Esc를 누르면 되감기 선택기가 열리지 않는 현상 수정
- 리다이렉션이 포함된 복합 명령문에 대한 Bash 명령 권한 확인 수정 — `&&` 목록 또는 부정 내에서
- 에이전트 목록에서 Ctrl+X를 두 번 누르면 세션을 삭제하지 못하고, 백그라운드 워커가 죽었을 때 삭제된 세션이 다시 나타나는 현상 수정
- 백그라운드 서브에이전트가 시작 창 중에 우선순위가 높은 메시지가 도착할 때 취소되는 현상 수정
- GUI 편집기가 `/memory`, `/plan`, `/keybindings` 또는 Ctrl+G에서 열려 있을 때 터미널의 마우스 및 포커스 오류 수정 — `/memory`는 더 이상 편집기가 닫힐 때까지 기다리지 않음
- 세션의 OAuth 토큰에 필수 범위가 없을 때 Chrome의 Claude가 재연결 시 403 루프에 빠지는 현상 수정
- 워크플로우 저장 및 예약된 작업 쓰기가 `.claude`의 심볼릭 링크를 따라가서 프로젝트 외부로 쓰기를 리디렉션할 수 있는 현상 수정
- MCP 재인증이 새로운 로그인이 성공하기 전에 작동 중인 자격 증명을 취소하고, 백그라운드 세션의 재연결 필요-인증 메시지가 사용할 수 없는 명령을 가리키는 현상 수정
- Windows의 읽기 전용 명령이 권한 프롬프트 없이 네트워크 경로에 액세스하는 현상 수정
- 실제 셸 단어 경계와 일치하도록 비-ASCII 문자의 Bash 명령 구문 분석 수정
- 보이지 않는 유니코드 문자를 포함하는 명령의 PowerShell 도구 권한 검증 수정
- 전체화면 모드에서 대화상자가 패널의 오른쪽 가장자리를 넘어 늘어나는 현상 수정
- 전체화면 모드에서 `/config` 설정 목록이 키보드 힌트 바닥글을 자르는 현상 수정
- 104개 열보다 좁은 터미널에서 트랜스크립트 모드(Ctrl+O) 바닥글 힌트가 줄바꿈되는 현상 수정
- Prometheus 메트릭 엔드포인트(`OTEL_METRICS_EXPORTER=prometheus`)가 유효하지 않은 `# UNIT` 줄을 내보내는 현상 수정
- 세션 중에 변경된 스킬 및 명령이 재시작할 때까지 슬래시 메뉴에 나타나지 않는 현상 수정
- `name` frontmatter 필드가 있는 플러그인 스킬이 슬래시 명령 자동 완성에서 플러그인 접두사를 잃는 현상 수정
- 텔레메트리가 권한 거부를 잘못 보고하는 현상 수정 — 실패한 권한 프롬프트 요청은 더 이상 사용자 거부로 계산되지 않으며, 사용자 중단은 이제 거부 대신 사용자 중단으로 보고됨
- 새 세션의 이름, `claude attach` id 및 사본이 checkout을 공유할 때의 참고 사항이 있는 한 줄로 `/fork` 확인 개선
- PowerShell 도구의 `git` 및 `gh` 명령 인수 검증 개선
- 구성된 제한, 측정된 diff 크기 및 가장 큰 기여 파일을 표시하도록 `/ultrareview` diff-too-large 오류 개선
- `/code-review ultra` 빈-diff 메시지를 정확한 base ref를 이름 붙이도록 개선하고 명시적 base를 전달할 것을 제안
- 소비 한도 조정 프롬프트를 개선하여 소비 한도 변경이 거부될 때 서버의 이유를 표시
- `/context`는 이제 대화가 context window를 초과할 때 명시적 경고를 표시하며, 실패한 `/compact`는 오류로 표시됨
- `/rewind`는 더 이상 추적되는 경로의 심볼릭 링크 또는 하드 링크를 통해 파일을 복원하거나 삭제하지 않으며 건너뜬 경로 수를 보고
- 백그라운드 세션: `/mcp` 및 `/install-github-app`은 이제 클라이언트가 연결되지 않았을 때 에이전트 보기에서 "needs input" 요청을 예약
- 번들 dataviz 스킬 업데이트 — 기본 차트 팔레트를 재정렬하고 4개 시리즈 차트에 대한 직접 레이블을 제안한 지침 수정
- [VSCode] 영어 또는 코드와 혼합될 때 오른쪽에서 왼쪽으로 텍스트(아랍어, 히브리어, 페르시아어)가 잘못된 순서로 렌더링되는 현상 수정
- 클라우드 세션이 세션의 컨테이너가 턴 중에 재시작될 때 진행 중인 메시지를 삭제하는 현상 수정 — 중단된 턴은 이제 세션이 응답하지 않은 채로 남겨두는 대신 재개 시 다시 실행됨
