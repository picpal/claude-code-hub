## 변경사항

- `requiredMinimumVersion` 및 `requiredMaximumVersion` managed settings을 추가했습니다 — Claude Code는 버전이 허용 범위를 벗어나면 시작을 거부하고 승인된 버전으로 이동하도록 사용자에게 안내합니다
- `/plugin list` 명령어를 추가했습니다. 설치된 플러그인을 나열하며 `--enabled`/`--disabled` 필터를 지원합니다
- `/btw`에 "c to copy" 바로가기를 추가했습니다. 원본 markdown 답변을 클립보드에 복사하여 다른 곳에 붙여넣을 때 형식이 보존됩니다
- Hooks: Stop 및 SubagentStop hooks는 이제 `hookSpecificOutput.additionalContext`를 반환하여 Claude에 피드백을 제공하고 hook 오류로 표시되지 않으면서 턴을 계속할 수 있습니다
- Skills: 명령어 본문에서 숫자 앞에 리터럴 `$`를 포함하기 위해 `\$` escape 구문을 추가했습니다
- stdio MCP servers는 이제 `--resume`에서 hooks/Bash와 동일한 `CLAUDE_CODE_SESSION_ID`을 수신합니다
- `claude -p`가 배경 명령어가 종료되지 않을 때 최종 결과 후 영구적으로 중단되는 문제를 수정했습니다 — stdin이 닫힌 후 ~5초 후에 배경 shell이 중지됩니다
- `CI=true`이고 Anthropic API key가 설정되지 않았을 때 Bedrock/Vertex/Foundry에서 `claude -p`가 "ANTHROPIC_API_KEY required"로 실패하는 문제를 수정했습니다
- bazel 및 EDR-protected Go workflows에서 bash 명령어가 실패하는 문제를 수정했습니다: `$TMPDIR`이 sandboxed 명령어에만이 아니라 모든 명령어에 대해 `/tmp/claude-{uid}`로 재정의되었습니다(2.1.154에서 회귀)
- 세션-env 디렉토리에 read-only 속성이 있거나 OneDrive 내부에 있을 때 Windows에서 bash 명령어가 "EEXIST: file already exists" 오류로 실패하는 문제를 수정했습니다
- fresh config 디렉토리에서 시작 중에 managed settings fetch가 완료될 때 org-managed 권한 규칙이 전체 세션에 적용되지 않는 문제를 수정했습니다
- `claude agents`의 배경 세션이 Claude Code 업데이트 후 재연결될 때 실행 중인 배경 작업을 잃는 문제를 수정했습니다
- Esc를 눌러 agent view를 종료할 때 terminal 정렬 불일치 및 여러 초의 중단이 발생하는 문제를 수정했습니다
- desktop app의 배경 작업 chip에서 Stop을 클릭할 때 기본 프로세스가 이미 사라졌을 때 chip이 지워지지 않는 문제를 수정했습니다
- terminal이 end marker를 삭제하는 paste 작업 후 키보드 입력이 영구적으로 응답하지 않게 되는 문제를 수정했습니다
- hook `if: "Bash(...)"` 조건이 `$()` 또는 `$VAR`를 포함하는 모든 bash 명령어에서 실행되는 문제를 수정했습니다; 패턴은 이제 subshells 및 backticks 내의 명령어에 대해서도 매칭됩니다
- home-directory 경로의 deny 규칙(예: `Read(~/Desktop/**)`)이 경로를 `$HOME`을 통해 참조하는 bash 명령어를 차단하지 않는 문제를 수정했습니다
- `/mcp` 및 `/plugins`와 같은 panel 대화를 닫은 후 transcript에 남겨진 stray "(no content)" 라인을 수정했습니다
- 배경 agent 세션은 이제 백그라운드에서 새로운 Claude Code 버전으로 업데이트되므로 업데이트 후 세션을 열 때 더 이상 cold restart를 기다리지 않습니다
- / 메뉴의 built-in 명령어 및 skills에 대한 명확한 설명을 제공합니다
- subscription-switch 제안이 이제 toast 대신 startup announcement 슬롯에 표시됩니다
- `claude agents`를 state-grouped view에서 dispatching할 때 이제 agent view가 열린 디렉토리에서 세션을 시작합니다
