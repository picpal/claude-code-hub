> 원문: [Claude Code v2.1.119 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.119)

## 변경사항

- `/config` 설정(theme, editor mode, verbose 등)이 이제 `~/.claude/settings.json`에 유지되며, project/local/policy 오버라이드 우선순위에 참여합니다
- footer PR 배지를 github.com 대신 custom code-review URL로 가리키도록 `prUrlTemplate` 설정이 추가되었습니다
- startup 로고에서 working directory를 숨기기 위해 `CLAUDE_CODE_HIDE_CWD` environment variable이 추가되었습니다
- `--from-pr`이 이제 GitLab merge-request, Bitbucket pull-request, GitHub Enterprise PR URL을 수용합니다
- `--print` mode가 이제 agent의 `tools:` 및 `disallowedTools:` frontmatter을 준수하여 interactive-mode 동작과 일치합니다
- `--agent <name>`이 이제 built-in agent의 agent definition `permissionMode`를 준수합니다
- PowerShell tool commands가 이제 permission mode에서 자동 승인될 수 있으며, Bash 동작과 일치합니다
- Hooks: `PostToolUse` 및 `PostToolUseFailure` hook inputs이 이제 `duration_ms`(tool 실행 시간, permission prompts 및 PreToolUse hooks 제외)를 포함합니다
- Subagent 및 SDK MCP server 재구성이 이제 순차적 대신 병렬로 서버를 연결합니다
- 다른 plugin의 버전 제약으로 고정된 Plugin이 이제 가장 높은 만족하는 git tag로 자동 업데이트됩니다
- Vim mode: INSERT의 Esc가 이제 대기 중인 메시지를 input으로 끌어오지 않습니다. interrupt하려면 Esc를 다시 누릅니다
- Slash command 제안이 이제 쿼리와 일치하는 문자를 강조합니다
- Slash command picker가 이제 long description을 truncate하지 않고 두 번째 줄로 wrapping합니다
- Output의 `owner/repo#N` shorthand link가 이제 항상 github.com을 가리키는 대신 git remote의 host를 사용합니다
- Security: `blockedMarketplaces`가 이제 `hostPattern` 및 `pathPattern` entries를 올바르게 시행합니다
- OpenTelemetry: `tool_result` 및 `tool_decision` events가 이제 `tool_use_id`를 포함합니다. `tool_result`는 또한 `tool_input_size_bytes`를 포함합니다
- Status line: stdin JSON이 이제 `effort.level` 및 `thinking.enabled`를 포함합니다
- CRLF content(Windows clipboards, Xcode console) 붙여넣기가 더 이상 모든 줄 사이에 extra blank line을 삽입하지 않도록 수정되었습니다
- multi-line paste가 bracketed paste 내부의 kitty keyboard protocol sequence를 사용하는 terminal에서 newline을 잃지 않도록 수정되었습니다
- Bash tool이 permission을 통해 거부될 때 native macOS/Linux builds에서 Glob 및 Grep tools가 사라지도록 수정되었습니다
- fullscreen mode에서 스크롤링이 tool 완료될 때마다 더 이상 bottom으로 snapping되지 않도록 수정되었습니다
- MCP HTTP connection이 server가 OAuth discovery requests에 대해 non-JSON bodies를 반환할 때 "Invalid OAuth error response" 실패하도록 수정되었습니다
- Rewind overlay가 image attachments가 있는 message에 대해 "(no prompt)"을 표시하지 않도록 수정되었습니다
- auto mode가 conflicting "Execute immediately" instructions로 plan mode을 override하지 않도록 수정되었습니다
- async `PostToolUse` hooks가 response payload를 emit하지 않을 때 session transcript에 empty entries를 쓰지 않도록 수정되었습니다
- spinner가 subagent task notification이 queue에서 orphaned될 때 더 이상 켜져 있지 않도록 수정되었습니다
- Tool search가 이제 unsupported beta header error를 피하기 위해 Vertex AI에서 기본적으로 비활성화됩니다(`ENABLE_TOOL_SEARCH`로 opt in)
- `@`-file Tab completion이 absolute path가 있는 slash command 내부에서 사용될 때 더 이상 전체 prompt를 바꾸지 않도록 수정되었습니다
- macOS Terminal.app의 Docker 또는 SSH를 통해 startup 시 prompt에 stray `p` character가 나타나지 않도록 수정되었습니다
- HTTP/SSE/WebSocket MCP server의 `headers`의 `${ENV_VAR}` placeholders가 이제 request 전에 substituted되도록 수정되었습니다
- `--client-secret`을 통해 저장된 MCP OAuth client secret이 이제 `client_secret_post`를 필요로 하는 server에 대해 token exchange 중에 전송되도록 수정되었습니다
- `/skills` Enter key가 dialog를 닫는 대신 prompt에서 `/<skill-name>`을 pre-fill하지 않도록 수정되었습니다
- `/agents` detail view가 subagent에 사용 불가능한 built-in tools를 더 이상 "Unrecognized"로 mislabeling하지 않도록 수정되었습니다
- Plugin의 MCP server가 plugin cache가 incomplete할 때 Windows에서 spawn하지 않도록 수정되었습니다
- `/export`가 현재 default model 대신 conversation이 실제로 사용한 model을 표시하도록 수정되었습니다
- verbose output setting이 restart 후 persist하지 않도록 수정되었습니다
- `/usage` progress bar가 더 이상 "Resets …" labels과 겹치지 않도록 수정되었습니다
- Plugin MCP server가 optional field를 blank로 남겨진 `${user_config.*}` references일 때 실패하지 않도록 수정되었습니다
- List items가 sentence-final number를 포함할 때 더 이상 number를 자신의 줄로 wrapping하지 않도록 수정되었습니다
- `/plan` 및 `/plan open`이 plan mode에 진입할 때 existing plan에 act하지 않도록 수정되었습니다
- Auto-compaction 전에 invoked된 skill이 다음 user message에 대해 re-executed되지 않도록 수정되었습니다
- `/reload-plugins` 및 `/doctor`가 disabled plugin에 대해 load errors를 report하지 않도록 수정되었습니다
- `isolation: "worktree"`를 가진 Agent tool이 이전 session의 stale worktree를 재사용하지 않도록 수정되었습니다
- Disabled MCP server가 `/status`에서 "failed"로 표시되지 않도록 수정되었습니다
- `TaskList`가 임의의 filesystem order 대신 ID별 sorted로 tasks를 반환하도록 수정되었습니다
- `gh` output이 "rate limit"을 언급하는 PR titles를 포함할 때 spurious "GitHub API rate limit exceeded" hints가 표시되지 않도록 수정되었습니다
- SDK/bridge `read_file`이 growing files에 대해 size cap을 올바르게 시행하지 않도록 수정되었습니다
- PR이 git worktree에서 작업할 때 session에 linked되지 않도록 수정되었습니다
- `/doctor`가 higher-precedence scope에 의해 override된 MCP server entries에 대해 경고하지 않도록 수정되었습니다
- Windows: 'cmd /c' wrapper를 필요로 하는 MCP config에 대한 false-positive "Windows requires" 경고가 제거되었습니다
- [VSCode] Voice dictation의 첫 번째 recording이 macOS에서 microphone permission prompt를 표시하는 동안 아무것도 생성하지 않도록 수정되었습니다
