## 변경된 사항
- 프롬프트 명령어나 슬래시 명령어를 주기적으로 실행할 수 있는 `/loop` 명령어를 추가했습니다(예: `/loop 5m check the deploy`)
- 세션 내에서 반복적으로 알림을 표시하기 위한 cron 예약 도구를 추가했습니다
- `keybindings.json`에서 음성 활성화 키를 재설정할 수 있도록 `voice:pushToTalk` 키 바인딩을 추가했습니다(기본값: 스페이스). `meta+k`와 같은 수정키+문자 조합은 타이핑에 전혀 방해가 되지 않습니다.
- bash 자동 승인 허용 목록에 `fmt`, `comm`, `cmp`, `numfmt`, `expr`, `test`, `printf`, `getconf`, `seq`, `tsort` 및 `pr`을 추가했습니다
- 장시간 실행되는 세션에서 키 입력이 처리되지 않는데도 프로세스가 종료되지 않아 표준 입력(stdin)이 멈추는 현상을 수정했습니다.
- 시스템에서 깨어난 후 CoreAudio 초기화로 인해 메인 스레드가 차단되어 음성 모드가 활성화된 사용자의 경우 발생하던 5~8초간의 시작 시 멈춤 현상을 수정했습니다.
- 다수의 claude.ai 프록시 커넥터가 만료된 OAuth 토큰을 동시에 새로 고칠 때 발생하는 시작 화면 멈춤 현상 수정
- 동일한 플랜 파일을 공유하는 포크된 대화(`/fork`)에서, 한 포크의 플랜을 수정하면 다른 포크의 내용이 덮어쓰이는 문제를 수정했습니다.
- 이미지 처리 실패 시 ‘읽기’ 도구가 크기가 너무 큰 이미지를 컨텍스트에 삽입하여, 이미지가 많은 긴 세션에서 이후 턴이 중단되던 문제를 수정했습니다.
- heredoc 커밋 메시지가 포함된 복합 bash 명령어에 대해 발생하던 오탐지 권한 요청 문제를 수정했습니다.
- 여러 Claude Code 인스턴스를 실행할 때 플러그인 설치 정보가 사라지는 문제를 수정했습니다
- OAuth 토큰 갱신 후 claude.ai 커넥터가 재연결되지 않던 문제를 수정했습니다
- 이전에 연결된 커넥터에만 표시되던 claude.ai MCP 커넥터 시작 알림이, 조직에서 구성된 모든 커넥터에 대해 표시되던 문제를 수정했습니다.
- 백그라운드 에이전트 완료 알림에서 출력 파일 경로가 누락되어, 컨텍스트 압축 후 상위 에이전트가 에이전트 결과를 복구하기 어려웠던 문제를 수정했습니다.
- 명령어가 0이 아닌 종료 상태로 종료될 때 Bash 도구 오류 메시지에 중복 출력되는 문제를 수정했습니다.
- 로컬 Chrome이 설치되지 않은 컴퓨터에서 실행한 후 Chrome 확장 프로그램 자동 감지 기능이 “설치되지 않음” 상태에서 영구적으로 멈춰버리는 문제를 수정했습니다.
- 마켓플레이스가 브랜치/태그 참조에 고정된 경우 `/plugin marketplace update` 명령어가 병합 충돌로 인해 실패하는 문제를 수정했습니다.
- `/plugin marketplace add owner/repo@ref` 명령어가 `@`를 잘못 해석하는 문제를 수정했습니다. 이전에는 `#`만 참조 구분자로 작동하여 `strictKnownMarketplaces`에서 원인을 파악할 수 없는 오류가 발생했습니다.
- 슬래시(/)를 붙이거나 붙이지 않고 동일한 디렉터리를 추가했을 때 `/permissions`의 ‘작업 공간’ 탭에 중복 항목이 표시되던 문제를 수정했습니다.
- 팀 에이전트가 구성된 상태에서 `--print` 옵션이 무한 대기하는 문제를 수정했습니다. 이제 종료 루프가 오래 지속되는 `in_process_teammate` 작업을 더 이상 대기하지 않습니다.
- `ToolSearch`를 호출할 때마다 REPL에 "❯ Tool loaded."가 표시되던 문제를 수정했습니다.
- 모델이 mingw 스타일 경로를 사용할 때 Windows에서 `cd <cwd> && git ...` 명령을 입력하라는 메시지가 표시되던 문제를 수정했습니다.
- 네이티브 이미지 프로세서의 로딩을 첫 사용 시점으로 미루어 시작 시간을 단축했습니다
- 노트북이 절전 모드에서 깨어난 후 최대 10분까지 기다리지 않고, 몇 초 이내에 브리지 세션 재연결이 완료되도록 개선되었습니다.
- `/plugin uninstall` 명령어를 개선하여, `.claude/settings.json` 파일을 수정하는 대신 `.claude/settings.local.json`에서 프로젝트 범위 내 플러그인을 비활성화하도록 했으므로, 변경 사항이 팀원들에게 영향을 미치지 않습니다.
- 플러그인에서 제공하는 MCP 서버 중복 제거 기능이 개선되었습니다. 수동으로 구성된 서버(동일한 명령어/URL)와 중복되는 서버는 이제 건너뛰어, 중복 연결 및 도구 세트 생성을 방지합니다. 제외된 항목은 `/plugin` 메뉴에서 확인할 수 있습니다.
- 디버그 로그가 더 이상 기본적으로 기록되지 않으므로, 세션 도중에도 디버그 로깅을 켜고 끌 수 있도록 `/debug`를 업데이트했습니다.
- 인증되지 않은 조직 등록형 claude.ai 커넥터에 대한 시작 알림 소리가 제거되었습니다.
