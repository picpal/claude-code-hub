## 변경사항

- `ANTHROPIC_BEDROCK_SERVICE_TIER` environment variable를 추가하여 Bedrock service tier (`default`, `flex`, 또는 `priority`)를 선택할 수 있도록 하였으며, `X-Amzn-Bedrock-Service-Tier` header로 전송됩니다
- `/resume` 검색 상자에 PR URL을 붙여넣으면 해당 PR을 생성한 session을 찾을 수 있도록 하였습니다 (GitHub, GitHub Enterprise, GitLab, Bitbucket 지원)
- `/mcp`에서 동일한 URL의 manually-added server에 의해 숨겨진 claude.ai connectors를 표시하며, 중복 제거에 대한 힌트를 제공합니다
- Browser sign-in flow 후 MCP server가 여전히 unauthorized일 때 표시되는 `/mcp` 메시지를 명확히 하였습니다
- OpenTelemetry: `api_request`/`api_error` log events의 numeric attributes를 이제 문자열이 아닌 숫자로 emit합니다
- OpenTelemetry: `@`-mention resolution을 위한 `claude_code.at_mention` log event를 추가하였습니다
- `/branch`가 rewound timelines의 entries를 포함한 source session에서 "tool_use ids were found without tool_result blocks" 오류로 실패하는 forks를 생성하는 문제를 수정하였습니다
- `/model`이 Bedrock application inference profile ARNs에 대해 Effort 옵션을 표시하지 않고, 해당 ARNs이 `output_config.effort`를 수신하지 않는 문제를 수정하였습니다
- Vertex AI / Bedrock이 session-title generation 및 기타 structured-output queries에서 `invalid_request_error: output_config: Extra inputs are not permitted` 오류를 반환하는 문제를 수정하였습니다
- Vertex AI `count_tokens` endpoint가 proxy gateways 뒤의 사용자에게 400 오류를 반환하는 문제를 수정하였습니다
- `spinnerTipsOverride.excludeDefault`가 time-based spinner tips를 억제하지 않는 문제를 수정하였습니다
- ToolSearch가 nonblocking mode에서 session 시작 후 연결된 MCP tools를 놓치는 문제를 수정하였습니다
- Bash mode에서 `!exit` / `!quit`이 shell command로 실행되지 않고 CLI를 종료하는 문제를 수정하였습니다
- 최신 models로 전송된 이미지가 올바른 2000px maximum 대신 side당 2576px로 resize되는 문제를 수정하였습니다
- Remote control session의 idle status가 초당 두 번씩 redraw되어 `tmux -CC` control pipes를 flood하고 terminal을 일시 중지할 수 있었던 문제를 수정하였습니다
- Assistant messages가 일부 sessions에서 stale view preference로 인해 blank로 나타나는 문제를 수정하였습니다
- `settings.json`의 malformed hooks entry가 더 이상 전체 파일을 invalidate하지 않도록 수정하였습니다
- Voice mode: Caps Lock에 바인딩된 keybindings는 이제 오류를 표시합니다. 이는 terminals이 Caps Lock을 key event로 전달하지 않기 때문입니다
