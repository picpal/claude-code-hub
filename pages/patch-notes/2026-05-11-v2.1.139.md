> 원문: [Claude Code v2.1.139 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.139)

## 변경사항

- Agent view(연구 미리보기) 추가: 모든 Claude Code 세션(실행 중, 당신의 답변을 기다리는 중, 완료됨)의 단일 목록입니다. `claude agents`를 실행하여 시작하세요. https://code.claude.com/docs/en/agent-view 참조
- `/goal` 명령어 추가: 완료 조건을 설정하면 Claude가 조건이 충족될 때까지 턴을 거쳐 계속 작업합니다. interactive, `-p`, Remote Control에서 작동합니다. 오버레이 패널로 실시간 경과 시간/턴/토큰을 표시합니다
- `/scroll-speed` 명령어 추가: 마우스 휠 스크롤 속도를 조정하고 실시간 미리보기를 제공합니다
- `claude plugin details <name>` 추가: 플러그인의 구성요소 인벤토리 및 예상 세션당 토큰 비용을 표시합니다
- Transcript view 네비게이션 추가: `?`는 키보드 단축키, `{`/`}`는 사용자 프롬프트 사이를 이동, `v`는 단축키 패널을 토글합니다
- Hook `args: string[]` 필드(exec form) 추가: 쉘 없이 직접 명령어를 실행하므로 경로 플레이스홀더에 따옴표를 붙일 필요가 없습니다
- `PostToolUse`에 hook `continueOnBlock` 구성 옵션 추가 — `true`로 설정하면 hook의 거부 사유를 Claude에 피드백하고 턴을 계속합니다
- MCP stdio 서버가 이제 환경에서 `CLAUDE_PROJECT_DIR`을 받습니다(hook과 일치). 플러그인 구성은 명령어에서 `${CLAUDE_PROJECT_DIR}`을 참조할 수 있습니다
- Compaction 프롬프트가 이제 모델에 민감한 사용자 지침을 보존하도록 요청합니다
- `/mcp` Reconnect는 이제 재시작 없이 `.mcp.json` 편집을 적용하고 재연결 실패 시 HTTP 상태 및 URL을 표시합니다
- `/context all` 스킬당 토큰 예상치는 이제 모델의 tokenizer를 고려하고 반올림된 값을 표시합니다
- `claude plugin install <name>@<marketplace>`는 이제 marketplace를 자동으로 새로고침하고 플러그인을 찾을 수 없다고 보고하기 전에 재시도합니다
- `/plugin` 설치된 플러그인 세부 정보가 이제 hook 이벤트 이름과 MCP 서버 이름을 깔끔하게 표시합니다
- `/context`는 이제 플러그인 소스의 스킬에 대해 제공 플러그인의 이름을 표시합니다
- Remote MCP 서버 재연결 재시도(일시적 오류)는 이제 모든 사용자에게 활성화됩니다
- Subagent의 API 요청은 이제 `x-claude-code-agent-id` / `x-claude-code-parent-agent-id` 헤더를 전달하고, `claude_code.llm_request` OTEL span에 `agent_id` / `parent_agent_id` 속성을 포함합니다
- Remote Control, `/schedule`, claude.ai MCP connector, 그리고 알림 기본 설정은 `ANTHROPIC_API_KEY` / `apiKeyHelper` / `ANTHROPIC_AUTH_TOKEN`이 설정되면 비활성화됩니다(Claude.ai 로그인이 있어도). 이 기능들을 사용하려면 API 키를 해제하세요
- 만료된 자격 증명과 `forceRemoteSettingsRefresh` 정책 설정이 `claude auth login`/`logout`/`status`를 차단하는 교착 상태를 수정했습니다(복구 방법이 없었음)
- `autoAllowBashIfSandboxed`가 `$VAR` 및 `$(cmd)`와 같은 셸 전개가 있는 명령어를 자동으로 승인하지 않는 버그를 수정했습니다
- Hook이 터미널에 쓸 때 화면의 대화형 프롬프트를 손상시킬 수 있는 버그를 수정했습니다. 이제 hook은 터미널 접근 없이 실행됩니다
- HTTP/SSE MCP 서버가 비프로토콜 데이터를 스트리밍할 때 무제한 메모리 증가를 수정했습니다. 응답 본문이 이제 SSE 프레임당 16MB로 제한됩니다
- `Skill(name *)` 권한 규칙을 수정했습니다. 와일드카드 형식이 이제 접두사 일치로 작동하여 `Bash(ls *)` 동작과 일치합니다
- 설정 핫 리로드가 심볼릭 링크된 `~/.claude/settings.json`에 대한 편집을 감지하지 못하는 문제를 수정했습니다
- Marketplace 키가 manifest 이름과 다를 때 플러그인 세부 정보 로드 실패를 수정했습니다
- `/model` 선택 도구의 "Default" 행이 `ANTHROPIC_DEFAULT_OPUS_MODEL`/`ANTHROPIC_DEFAULT_SONNET_MODEL` 재정의를 반영하지 않는 문제를 수정했습니다
- 응답 완료 후 5분 후에 발생하는 거짓 "stream idle timeout"을 수정했습니다(스트림 취소 시 watchdog 타이머가 지워지지 않은 원인)
- 10개 이상의 MCP 서버가 구성되고 캐시 디렉토리를 쓸 수 없을 때 자동 `exit 1`을 수정했습니다. 이제 오류 메시지에 근본 원인이 포함됩니다
- 대화 상자의 탭 이름, 목록 포인터, select 행에서 깜빡이는 입력 커서를 수정했습니다
- Transcript view의 문자 단축키가 마우스 클릭 후 작동하지 않는 문제를 수정했습니다
- Bash 모드 위쪽 화살표 기록이 첫 번째 항목을 반복하고 진행 중인 초안을 덮어쓰는 문제를 수정했습니다
- 여러 이미지 붙여넣기 또는 드롭이 마지막 이미지만 삽입하는 문제를 수정했습니다
- 어두운 테마에서 하이퍼링크가 읽을 수 없는 짙은 남색을 사용하는 문제를 수정했습니다. 이제 활성 테마에 맞춰집니다
- 모델이 `opus` alias로 설정된 타사 사용자에 대해 모델 선택 도구가 중복된 "Current model" 행을 표시하는 문제를 수정했습니다
- PAYG 3P 공급자의 레거시 Opus 선택 도구 항목이 기본 항목과 같은 모델로 확인되는 문제를 수정했습니다
- Cursor 및 VS Code 1.92–1.104에서 마우스 휠 스크롤 속도를 수정했습니다. 트랙패드는 이제 일정한 속도로 스크롤하고 마우스 휠은 한 노치당 약 3줄을 유지합니다
- Windows Terminal 및 VS Code에서 백그라운드 세션에 연결할 때의 스크롤 동작을 수정했습니다
- 분리된 서버의 MCP 리소스가 `@server:` 자동완성에 남아 있는 문제를 수정했습니다
- 두 파일 diff 스니펫이 잘린 줄 수를 1줄로 초과 보고하는 문제를 수정했습니다
- Grep 결과가 Windows 드라이브 문자 경로를 상대화하지 않고 단일 파일 경로에 대해 개수 모드가 잘못된 합계를 보고하는 문제를 수정했습니다
- 테두리에 포함된 텍스트가 CJK/이모지로 인해 시각적 셀 너비 오계산으로 인해 오버플로우되는 문제를 수정했습니다
- Fuzzy 매칭 강조가 이모지와 astral plane 문자를 쌍 중간에서 분할하는 문제를 수정했습니다
- 정규식 메타문자를 포함하는 스킬 인수 이름이 인수 대체를 깨뜨리는 문제를 수정했습니다
- ProgressBar가 거의 가득 찬 소수 셀에 대해 전체 블록을 렌더링하는 문제를 수정했습니다
- 마지막 구독자가 가는 동안 fetch가 진행 중일 때 task polling 및 `fs.watch`가 부활하는 문제를 수정했습니다
- Manifest 이름이 소스 식별자와 다를 때 플러그인 종속성 해결이 stale 개수를 남기는 문제를 수정했습니다
- Insights Time-of-Day 차트가 세션에 구문 분석할 수 없는 타임스탬프가 있을 때 왜곡되는 문제를 수정했습니다
- cmd/super/win modifier만 사용하는 keybinding이 구문 분석할 수 없는 것으로 표시되는 문제를 수정했습니다
- `claude_code.active_time.total` OpenTelemetry 메트릭이 `--print` 모드에서 내보내지지 않는 문제를 수정했습니다
- `claude plugin update`가 marketplace 내 교차 플러그인 symlink를 보존하지 않는 문제를 수정했습니다
- [VSCode] Cmd/Ctrl+Shift+T를 눌러 가장 최근에 닫은 세션 탭을 다시 열 수 있습니다. `claudeCode.enableReopenClosedSessionShortcut`을 통해 구성 가능합니다
