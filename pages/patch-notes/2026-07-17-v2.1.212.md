> 원문: [Claude Code v2.1.212 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.212)

## 변경사항

- `/fork` 명령어가 이제 대화를 새로운 백그라운드 세션(`claude agents`의 별도 행)으로 복사하면서 작업을 계속할 수 있으며, 이전에 실행하던 인세션 subagent는 이제 `/subtask`로 변경되었습니다
- `claude auto-mode reset` 명령어를 추가하여 기본 auto-mode 구성을 복원할 수 있으며, 확인 프롬프트가 표시됩니다(`--yes` 옵션으로 건너뛸 수 있음)
- WebSearch tool 호출에 대한 세션 전체 제한(기본값 200, `CLAUDE_CODE_MAX_WEB_SEARCHES_PER_SESSION`으로 조정 가능)을 추가하여 검색 루프 폭발을 방지합니다
- subagent 생성에 대한 세션별 상한선(기본값 200, `CLAUDE_CODE_MAX_SUBAGENTS_PER_SESSION`으로 재정의 가능)을 추가하여 위임 루프 폭발을 방지하며, `/clear`로 예산을 초기화할 수 있습니다
- 2분 이상 실행되는 MCP tool 호출이 이제 자동으로 백그라운드로 이동하여 세션이 계속 사용 가능하게 유지되며, `CLAUDE_CODE_MCP_AUTO_BACKGROUND_MS`로 임계값을 설정하거나 비활성화할 수 있습니다
- agent 뷰에서 `/resume`을 입력하면 이제 과거 세션(목록에서 삭제된 세션 포함)의 선택기가 열리고 선택한 세션을 백그라운드 세션으로 재개할 수 있습니다
- plan mode가 파일 수정 Bash 명령어(`touch`, `rm` 등)를 권한 프롬프트나 SDK `canUseTool` 콜백 없이 자동 실행하는 문제를 수정했습니다
- `.claude/worktrees`의 저장소 커밋된 심볼릭 링크 이후의 worktree 생성으로 인해 저장소 외부에 파일이 생성될 수 있는 문제를 수정했습니다
- `continue:false` hook의 중단이 tool이 실패하거나 스트림 중간에 완료될 때 삭제되고, hook 인프라 오류가 사용자 거부로 잘못 보고되는 문제를 수정했습니다
- Bash tool 실행 중 SIGTERM으로 인해 print/SDK mode에서 명령어의 프로세스 트리가 고아 상태가 되는 문제를 수정했으며, CLI는 이제 턴을 중단하고 트리를 종료한 후 143으로 종료합니다
- Group Policy가 PowerShell 5.1을 차단할 때 Windows에서 `/background` 및 `claude --bg` 명령어가 "EUNKNOWN: unknown error, uv_spawn"으로 실패하는 문제를 수정했으며, daemon은 이제 PowerShell 7을 선호합니다
- shell mode(`!`)가 경로 자동 완성 팝업이 열려 있는 동안 파일 경로를 포함하는 명령어를 실행하지 않는 문제를 수정했습니다
- auto-mode 거부 알림이 긴 거부 사유가 이모지 중간에 잘릴 때 깨진 문자를 렌더링하는 문제를 수정했습니다
- Ctrl+J가 extended key reporting을 지원하는 터미널에서 agent 뷰 dispatch input에 줄바꿈을 삽입하지 않는 문제를 수정했으며, 줄바꿈 단축키를 `?` 도움말 오버레이에서 표시합니다
- `/ultrareview`가 `#123`, `PR 123`, 붙여넣은 PR URL과 같은 PR 참조를 거부하는 문제를 수정했으며, 오류 힌트는 이제 실제로 입력한 명령어 이름을 표시합니다
- `/ultrareview <branch>` 명령어가 원격으로 존재할 때 origin에서 branch를 가져오지 않는 문제를 수정했으며, 오타 발생 시 가장 유사한 branch 이름을 제안합니다
- `/ultrareview`가 `/clear` 이후 새로운 대화에서 청구 확인을 건너뛰는 문제를 수정했습니다
- Claude Desktop의 `/ultrareview`에서 "not a git repository" 오류가 터미널 명령어 대신 프로젝트의 저장소 폴더를 제안하지 않는 문제를 수정했습니다
- 호스팅(host-managed) 세션이 저장소 설정에서 mTLS 인증서, 추가 CA 번들 또는 OAuth 범위를 구성했을 때 시작 시 실패하는 문제를 수정했으며, 이러한 transport 설정은 경고와 함께 무시됩니다
- offset/limit으로 읽은 후 세션을 재개했을 때 파일을 편집할 때 거짓 "File has not been read yet" 오류가 발생하는 문제를 수정했습니다
- print/SDK mode에서 `--continue`/`--resume`을 사용하여 세션을 재개한 후 `ExitWorktree`가 "no active EnterWorktree session" 오류로 실패하는 문제를 수정했습니다
- Remote Control 클라이언트가 세션 실행 중간에 참여했을 때 workflow agent grid가 비어 있는 채로 유지되는 문제를 수정했습니다
- streaming-mode control 요청이 핸들러가 완료되기 전에 완료로 표시되어 세션 재시작 시 요청이 손실될 수 있는 문제를 수정했습니다
- `/fork`로 생성된 백그라운드 세션이 상태 쓰기 실패 후 live-parent 보호를 잃는 문제를 수정했습니다
- agent 뷰에서 중지된 백그라운드 세션을 다시 열 때 조용히 실패하는 문제를 수정했으며, 이제 세션을 재개하거나 실패 이유를 표시하고 강제 재시작을 허용합니다
- agent teams: 중지되는 teammate가 팀 초기화가 세션 내에서 다시 실행될 때 리더에게 중복된 idle 알림을 보낼 수 있는 문제를 수정했습니다
- plan-approval 다이얼로그 바닥글이 파일 경로가 길 때 "ctrl+g to edit in <editor>"를 분리하는 문제를 수정했습니다
- 전체화면 모드에서 너비+높이 터미널 크기 조정 후 welcome banner가 이전 패널 너비를 유지하는 문제를 수정했습니다
- diff 미리보기가 좁은 레이아웃에서 줄 번호와 +/- 마커를 잃는 문제를 수정했습니다
- @-mention이 부분 파일 읽기 후 아무것도 첨부하지 않으며, plugin 제거가 잘못된 마켓플레이스를 대상으로 하고, 종료 코드 143에서 거짓 "Command timed out"이 발생하는 문제를 수정했습니다
- OpenTelemetry HTTP 내보내기가 chunked transfer encoding을 허용하지 않는 Azure Monitor 및 기타 엔드포인트에서 411/400으로 거부되는 문제를 수정했습니다
- OTLP 이벤트 로그 레코드가 SDK/headless mode에서 `TRACEPARENT`가 설정되었을 때 `trace_id`/`span_id`가 누락되는 문제를 수정했습니다
- 많은 이미지가 있는 대화가 잘못되어 "Request too large" 오류로 실패하는 문제를 수정했으며, 실제 원인을 설명하도록 오류 메시지를 개선했습니다
- web search 및 web fetch가 API가 과부하일 때 "API Error" 텍스트를 검색 결과 또는 페이지 콘텐츠로 반환하는 문제를 수정했습니다
- 제한된 백오프로 529 오류 및 rate-limited 요청을 재시도하여 web search 및 web fetch 신뢰성을 개선했습니다
- prompt caching 개선: mid-conversation system 블록이 이제 LLM gateway 및 custom base URL(Bedrock, Vertex, 1P) 뒤에서 작동합니다
- 백그라운드 agent attach 개선: cold-attach가 이제 세션 부팅 중 빈 대기 대신 형식화된 transcript를 즉시 표시합니다
- inter-agent messaging에서 token 사용량 감소: `SendMessage` 본문이 더 이상 재생 기록 및 tool 결과에 복제되지 않습니다
- `/fork`를 변경하여 세션에 제목이 없을 때 프롬프트 이후에 복사본을 이름 지정하므로 agent 뷰의 행이 인식 가능합니다
- bare `/btw`를 변경하여 가장 최근 교환에서 side-question 패널을 다시 열어 이전 답변을 찾아볼 수 있습니다
- `←` 바닥글 힌트를 변경하여 백그라운드 agent가 완료되고 입력이 필요하지 않을 때 `N done`을 잠시 펄스합니다
- Task tool의 `mode` parameter를 폐기(이제 무시됨); subagent는 기본적으로 부모 세션의 권한 mode를 상속합니다
- Enterprise `forceLoginMethod`를 변경하여 터미널뿐 아니라 VS Code extension, SDK, `setup-token`, `install-github-app` 로그인에 적용합니다
- 세션 transcript를 변경하여 각 assistant 메시지에 대해 reasoning effort 수준을 기록합니다
- headless/SDK 세션을 변경하여 mid-turn에 `set_model` control 요청을 적용하며, 다음 model round-trip은 다음 턴을 기다리지 않고 새로운 model을 사용합니다
- agent 뷰 / `claude agents --json`를 변경하여 sandbox, MCP-input 또는 managed-settings 프롬프트에서 대기 중인 세션이 "Working" 대신 "Needs input"으로 표시됩니다
- auth 상태 패널 제목을 "Cloud authentication"에서 "Authentication"으로 업데이트했습니다
- 이전 릴리스 노트(2.1.200) 수정: tmux 3.6 시리즈까지는 synchronized output이 부족하며, 지원이 있는 최신 tmux가 자동으로 감지됩니다
