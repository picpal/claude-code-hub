## 변경사항

- auto mode 안전성 개선: 파괴적인 git 명령어(`git reset --hard`, `git checkout -- .`, `git clean -fd`, `git stash drop`)는 로컬 작업을 삭제하도록 요청하지 않았을 때 차단되며, `git commit --amend`는 이번 세션에서 agent가 만들지 않은 커밋에 대해 차단되고, `terraform destroy`/`pulumi destroy`/`cdk destroy`는 특정 스택을 삭제하도록 요청하지 않았을 때 차단됩니다.
- 요청한 model이 deprecated되었거나 더 최신 버전으로 자동 업데이트될 때 경고를 추가했습니다. 이는 print mode(`-p`)의 stderr에 표시되며 이제 agent frontmatter에서 설정한 모델도 포함합니다.
- web 및 Remote Control 세션에서 커밋 및 PR로부터 claude.ai 세션 링크를 생략하도록 `attribution.sessionUrl` 설정을 추가했습니다.
- `/config` 명령어의 모든 사용 가능한 shorthand 키를 나열하기 위해 `/config --help`를 추가했습니다.
- `/config` toggle 동작 변경: Enter와 Space 모두 선택된 설정을 변경하며, Esc는 이제 되돌리는 대신 저장하고 닫습니다.
- 로고 아래 시작 "setup issues" 줄을 제거했습니다. 구성 문제를 확인하려면 `/doctor`를 실행하거나 `--debug`를 사용하세요.
- subagent spawns 및 영향을 받는 구성의 session-title generation에 대해 `thinking.disabled.display: Extra inputs are not permitted` 400 오류를 수정했습니다.
- subagent에서 WebSearch가 빈 결과를 반환하는 문제를 수정했습니다.
- native cursor가 활성화된 상태에서 vim mode로 히스토리를 탐색한 후 터미널 커서가 prompt 위에 정체되는 문제를 수정했습니다.
- 무거운 중첩된 subagent load 상황에서 Windows Terminal의 fullscreen TUI 손상(statusline이 화면 중간에 위치, spinner 행 중복, 텍스트 병합)을 수정했습니다.
- model이 thinking block만 반환했을 때 visible output 없이 조용히 turn이 완료되는 문제를 수정했습니다. Claude는 이제 한 번 다시 prompt를 표시합니다.
- 여러 plugin이 활성화되었을 때 사용자 수준의 skill이 slash-command autocomplete에 여러 번 나타나는 문제를 수정했습니다.
- authentication이 필요한 MCP server가 headless/SDK mode에서 auth-stub tool을 model에 노출하는 문제를 수정했습니다.
- shell이 느린 rc-file initialization을 가질 때 tmux teammate pane 시작에 실패하는 문제를 수정했고, agent spawn 중 입력한 keystroke가 새로운 tmux pane으로 누수되지 않고 leader prompt로 이동하도록 했습니다.
- teammate에 의해 시작된 background task가 teammate의 turn 종료 시 kill되는 문제를 수정했습니다.
- scheduled task 및 webhook trigger 전달이 keyboard input으로 처리되는 문제를 수정했습니다. 이제 task notification으로 분류되며 pending action을 승인하거나 auto mode에서 session title을 설정할 수 없습니다.
- focus mode가 각 response 아래 "Ran N PostToolUse hooks" timing 줄을 표시하는 문제를 수정했습니다.
