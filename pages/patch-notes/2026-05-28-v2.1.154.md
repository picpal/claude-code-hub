> 원문: [Claude Code v2.1.154 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.154)

## 변경사항

- Opus 4.8이 출시되었습니다! 이제 가장 어려운 작업을 위해 기본적으로 높은 노력 · /effort xhigh를 제공합니다
- 동적 workflow 도입: Claude에게 workflow를 만들도록 요청하면 백그라운드에서 수십 개에서 수백 개의 agent에 걸쳐 작업을 조율하므로 더 크고 복잡한 작업을 수행할 수 있습니다. `/workflows`를 실행하여 실행 내역을 확인하세요
- Opus 4.8의 Fast mode는 이제 이전 비용의 일부로 제공됩니다: 표준 요금의 2배로 2.5배 빠른 속도
- Lean system prompt는 이제 Haiku, Sonnet, Opus 4.7 및 이전 버전을 제외한 모든 모델의 기본값입니다
- Claude는 이제 객관식 질문 prompt를 자신이 진정으로 결정할 수 없는 경우에만 사용하며, 이미 진행할 충분한 컨텍스트가 있을 때는 묻지 않습니다
- `/simplify`는 이제 full `/code-review --fix` bug-hunting review를 실행하는 대신 cleanup-only review(재사용, 단순화, 효율성, 높이)를 실행하고 수정사항을 적용합니다
- `/effort` slider 레이블을 명확하게 하기 위해 "Speed"/"Intelligence"에서 "Faster"/"Smarter"로 변경했습니다
- `claude agents`: `! <command>`를 입력하여 attach/detach할 수 있는 background session으로 shell command를 실행합니다. `claude --bg --exec '<command>'`로도 사용할 수 있습니다
- `claude agents`: `/logout`은 이제 background session으로 전송되는 대신 로그아웃합니다
- `←←`로 agents view를 열기는 이제 Bedrock, Vertex, Foundry에서 그리고 telemetry가 비활성화되었을 때 작동합니다
- Claude in Chrome: `/chrome` → "Select browser…"를 통해 사용할 connected browser를 선택하거나, 여러 브라우저가 연결된 상태에서 browser action이 실행될 때 in-chat에서 선택합니다
- Plugin은 이제 `plugin.json`이나 marketplace entry에서 `defaultEnabled: false`를 선언할 수 있으며, `/plugin` 또는 `claude plugin enable`로 활성화합니다. 활성화된 plugin의 dependencies는 여전히 자동으로 활성화됩니다
- `/plugin` Discover 탭은 이제 현재 디렉터리와 관련성 신호가 일치하는 plugin을 "suggested for this directory" annotation과 함께 고정합니다
- Streaming tool execution은 이제 항상 활성화되며, telemetry가 비활성화되었거나 Bedrock/Vertex/Foundry에 있을 때도 포함됩니다(이전에는 feature flag 뒤에 있었음)
- Stdio MCP server subprocess는 이제 환경에서 `CLAUDE_CODE_SESSION_ID`와 `CLAUDECODE=1`을 받습니다
- `claude mcp list`/`get`은 이제 unapproved `.mcp.json` server를 auto-approving하고 output이 piped될 때 연결하는 대신 `⏸ Pending approval`으로 표시합니다
- `/remote-control` autocomplete는 이제 Remote Control이 이미 활성화되어 있을 때 "Disconnect Remote Control"을 표시합니다
- `/claude-api` skill에 Claude Opus 4.8 지원 및 4.7 → 4.8 migration guidance를 추가했습니다
- `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE` 폐기됨(06/01에 제거될 예정). Opus 4.6에서 fast mode를 사용하려면 `/model claude-opus-4-6[1m]`으로 전환한 후 `/fast on`을 입력하세요
- Auto-mode classifier의 data exfiltration 감지, 특히 repository contents의 bulk transfer를 개선했습니다
- `HOME`이 trailing slash를 가질 때 `rm -rf $HOME`이 위험한 경로로 차단되지 않는 문제를 수정했습니다
- 같은 session 내에서 sandboxed vs unsandboxed Bash command에서 `$TMPDIR`이 다른 디렉터리로 해석되는 문제를 수정했습니다
- Claude Code theme이 terminal background와 일치하지 않을 때 `claude agents`에서 highlighted-row text가 읽을 수 없는 문제를 수정했습니다
- Background-agent completion notifications가 일부 1M-context model에서 조기 "out of context" behavior를 트리거하는 문제를 수정했습니다
- Background-session classifier가 scheduled `/command`가 실행될 때 사용자의 goal을 잃는 문제를 수정했습니다
- Pinned background session이 Claude Code update 이후 매분 다시 생성되어 반복된 agent-start notifications 및 idle 상태에서 process churn을 유발하는 문제를 수정했습니다
- Background session이 "blocked", "running", 또는 "working"에서 stuck되었을 때 idle grace period 이후 retire되지 않는 문제를 수정했습니다
- Background session의 subagent가 worktree-isolation guard를 bypass하고 shared checkout에 write하는 문제를 수정했습니다
- macOS에서 daemon이 exit한 이후 orphaned `claude --bg-pty-host` process가 100% CPU에서 spinning되는 문제를 수정했습니다
- Number key shortcuts이 option dialog에서 divider 아래에 표시된 option에서 작동하지 않는 문제를 수정했습니다
- `worktree.baseRef: "head"`가 linked worktree 내에서 subagent를 spawning하거나 `EnterWorktree`를 호출할 때 main checkout의 HEAD 대신 current worktree의 HEAD로 해석되는 문제를 수정했습니다
- Previous line이 terminal width에서 정확히 끝날 때 wrapped line에 trailing space가 나타나는 문제를 수정했습니다
- Thinking spinner가 생성하는 distinct color의 개수를 capping하여 VS Code에서 intermittent terminal rendering corruption을 수정했습니다
- Plan-mode prompt가 pasted image 또는 text로 시작할 때 plan file name에 `[Image #N]` / `[Pasted text #N]` placeholder가 포함되는 문제를 수정했습니다
- Colored tool output에서 phantom expand/click affordance를 수정했습니다: screen에 맞는 short ANSI-colored line은 이제 "ctrl+o to expand" hint를 표시하지 않습니다
- Managed settings의 single invalid `allowedMcpServers`/`deniedMcpServers` entry가 모든 managed-settings policy를 discard하는 문제를 수정했습니다; bad entry는 이제 `claude doctor` warning과 함께 dropped됩니다
- `CLAUDE_CODE_ALWAYS_ENABLE_EFFORT`가 설정되었을 때 effort parameter를 지원하지 않는 model에서 API 400 error를 발생시키는 문제를 수정했습니다
- Windows: `claude.exe`가 사용 중일 때 인한 update failure가 generic error 대신 다른 session을 닫고 retry하라고 알려주는 문제를 수정했습니다
- Shortcuts help panel에서 stale "& for background" hint를 제거했습니다
- [VSCode] Auto mode는 이제 mode picker에 나타나기 위해 bypass-permissions setting이 필요하지 않으며, new-session screen의 dismissable notice는 auto mode가 처음 활성화될 때 설명합니다
- Workflow만 실행 중일 때 prompt 아래의 task panel이 stray unselectable "main" row를 표시하는 문제를 수정했습니다
- MCP server가 long 또는 multi-line tool name 또는 long description을 가질 때 /mcp tools list 및 tool detail rendering을 수정했습니다
- Fast mode가 on일 때 API(pay-as-you-go) user의 Default option에서 /model picker가 fast mode pricing을 표시하지 않는 문제를 수정했습니다
- Safety classifier가 reasoning하는 동안 output token이 부족할 때 auto mode가 "could not evaluate this action"으로 action을 부정확하게 차단하는 문제를 수정했습니다
