> 원문: [Claude Code v2.1.153 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.153)

## 변경사항

- `github`/`git` plugin marketplace sources에 `skipLfs` 옵션을 추가하여 clone 및 update 중에 Git LFS 다운로드를 건너뛸 수 있도록 함
- Claude Code가 npm 전역 설치가 자동으로 업데이트되지 않을 때 일회성 알림을 표시하며, `/doctor` 명령어가 수정 사항을 나열함
- Status line 명령어가 이제 `COLUMNS` 및 `LINES` 환경 변수를 수신하므로 스크립트가 터미널 너비에 맞게 출력을 크기 조정할 수 있음
- `claude agents`: dispatch 입력의 자동완성이 이제 project skills뿐만 아니라 native slash commands 및 bundled skills도 제안함
- `claude agents`: PR column이 단일 PR의 경우 `PR #N`을 표시하거나 여러 PR의 경우 `N PRs`를 표시함
- `claude doctor`가 마지막 업데이트 시도 결과를 표시함
- MCP servers 및 connectors의 분리된 "needs authentication" 시작 알림을 단일 메시지로 통합함
- macOS: background agents가 이제 Privacy & Security에서 "Claude Code"로 표시되며 업그레이드 간에 권한을 유지함
- 옵션인 GET SSE stream을 지원하지 않는 stateful MCP servers가 `tools/list`에서 재연결 루프가 되는 현상을 수정함 (v2.1.147의 회귀)
- custom API gateway가 gateway의 자체 token 대신 사용자의 Anthropic OAuth credential을 수신할 수 있는 회귀를 수정함
- subagent (Agent tool) frontmatter MCP servers가 `--strict-mcp-config`, `--bare`, remote mode, enterprise managed MCP config 및 managed-settings MCP server allow/deny policies를 무시하는 현상을 수정함
- `--strict-mcp-config`이 더 이상 명시적으로 전달된 agent 정의 (`--agents` / SDK `agents`)에서 inline `mcpServers`를 제거하지 않으며, 차단된 subagent MCP servers가 표시되는 경고를 표시함
- Windows PowerShell installer가 설치 실패 시에도 "Installation complete!"를 보고하는 현상을 수정함
- npm 설치의 경우 `claude update`가 configured release channel의 버전 대신 최신 버전을 설치하는 현상을 수정함
- 많은 저장된 sessions을 가진 machines에서 transcript 파일 경로로 session을 재개할 때 과도한 메모리 사용량 (여러 GB)을 수정함
- binary-takeover support 이전에 시작된 stale daemon에서 실행되는 `claude agents` 및 `claude --bg`를 업그레이드 후에도 수정함
- stdin이 stream-json mode에서 EOF 없이 종료될 때 CLI가 종료되지 못하는 hang을 수정하여 stale session marker를 남기지 않도록 함
- Claude의 responses에서 malformed `file://` links이 터미널에서 클릭 가능하지 않은 현상을 수정함
- `claude --help`가 92열보다 좁은 터미널에서 wrapped되지 않은 출력을 렌더링하는 현상을 수정함
- MCP tool progress notifications이 collapsed tool view에서 렌더링되지 않는 현상을 수정함
- `subagent_type: 'claude'`를 가진 `Agent` tool이 documented되지 않은 temporary worktree에서 실행되어 gitignored paths에 기록된 출력을 자동으로 버리는 현상을 수정함
- Claude가 응답 중일 때 `/bg`를 입력하면 이제 response를 버리는 대신 background session에서 계속함
- `/btw` keyboard shortcuts이 task 실행 중인 background sessions에서 응답하지 않는 현상을 수정함
- background sessions이 `$CLAUDE_JOB_DIR`에 temp files을 기록할 때 "sensitive file" permission prompt를 트리거하는 현상을 수정함
- working directory가 삭제된 background agent를 복구할 때 명확한 오류 메시지 대신 truncated stack trace를 표시하는 현상을 수정함
- `EnterWorktree`이 background sessions에서 즉시 사용 가능하지 않은 현상을 수정함 (이전에는 `ToolSearch`가 필요함)
- iTerm2/Terminal.app에서 `cmd+k`가 attached background sessions을 repaint하지 않는 현상을 수정함
- IME candidate window가 Windows의 attached background sessions에서 input caret 옆이 아닌 화면 하단에 나타나는 현상을 수정함
- agent가 file diffs를 렌더링한 후 256-color-only terminals에서 background agent에 attach할 때 background-color bleed가 발생하는 현상을 수정함
- `/copy` 및 copy-on-select가 tmux 내부의 background session에 attached될 때 system clipboard를 업데이트하지 못하는 현상을 수정함
- Remote Control이 활성화된 상태로 `claude agents`를 열 때 Code tab에 zombie session entries가 남는 현상을 수정함
- background sessions에서 `/rename`이 session banner를 즉시 업데이트하지 않는 현상을 수정함
- Windows update rollback: Windows update가 실패하면 Claude Code가 copy로 원본 executable을 복구하고 복구 방법을 알려줌
- [VSCode] VS Code가 Windows에서 종료될 때 Claude Code processes가 cleanly하게 shut down되지 않아 false "unclean exit" reports 및 orphaned MCP servers가 발생하는 현상을 수정함
- `/model`이 이제 새로운 sessions의 기본값으로 선택 항목을 저장함 (IDE와 일치). picker에서 `s`를 눌러 현재 session에서만 models을 전환함
- `modelPicker:setAsDefault` keybinding을 customized한 경우 keybindings.json에서 `modelPicker:thisSessionOnly`로 이름을 바꿈 (`d` action이 `s`로 대체됨)
