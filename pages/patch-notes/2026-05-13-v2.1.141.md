## 변경사항

- hook JSON output에 `terminalSequence` 필드를 추가하여 제어 terminal 없이도 hook이 desktop notifications, window titles, bell을 emit할 수 있도록 지원
- `CLAUDE_CODE_PLUGIN_PREFER_HTTPS`를 추가하여 GitHub SSH key가 없는 환경에서 GitHub plugin sources를 SSH 대신 HTTPS로 clone하도록 지원
- workload identity federation을 위해 `ANTHROPIC_WORKSPACE_ID` environment variable을 추가 — federation rule이 여러 workspace를 포함할 때 minted token을 특정 workspace로 제한
- `claude agents --cwd <path>`를 추가하여 session list의 범위를 directory로 제한
- `/feedback`에서 이제 최근 sessions (지난 24시간 또는 7일)를 포함할 수 있어 현재 session보다 길게 걸치는 issues를 다룰 수 있음
- Rewind menu에 "Summarize up to here"를 추가하여 최근 turns를 유지하면서 이전 context를 압축
- Auto mode permission dialog에서 이제 `permissions.ask` rule이 prompt를 유발한 시점을 설명
- IDE가 연결된 경우 file-edit permission prompts에서 "view diff in your IDE" 옵션을 복원
- `/bg` 또는 `←←`를 통해 실행된 background agents가 이제 기본값으로 되돌리는 대신 현재 permission mode를 유지
- `claude agents`: 작업을 완료했으나 background shell을 실행 중인 agents는 이제 Working 상태로 유지되지 않고 Completed로 이동
- 긴 thinking 기간 중 spinner feedback을 개선 — spinner는 이제 10초 후 amber로 변하여 Claude가 계속 작업 중임을 나타냄
- plugin menu navigation을 개선: `→`/Tab로 tabs를 전환하고, `↑`로 tab strip으로 이동하며, fullscreen mode에서 tab headers와 search box를 클릭 가능
- `ANTHROPIC_SMALL_FAST_MODEL` override가 설정되지 않은 경우 Bedrock/Vertex/Foundry/gateway에서 background side-queries가 사용 불가능한 Haiku model ID를 보내는 문제를 수정 — 이제 main-loop model로 fallback
- Windows에서 `claude daemon status` 및 `/doctor`가 daemon pipe key file이 locked 또는 unreadable일 때 exception을 던지는 문제를 수정 — 이제 불명확한 failure 대신 underlying error를 표시
- flag를 추가하는 wrapper를 통해 실행된 경우 `claude agents`가 agent-type list 대신 dashboard를 표시하는 문제를 수정
- 작업 directory가 삭제된 경우 crashed session을 여는 `claude agents`가 redundant dispatches를 실행하는 문제를 수정
- custom `ANTHROPIC_BASE_URL` gateway의 background jobs이 auto-named되지 않는 문제를 수정 — namer는 이제 Haiku model이 구성되지 않았을 때 main model을 사용
- 한 session의 `/model`이 다른 concurrent sessions의 autocompact threshold를 silent하게 변경하는 문제를 수정
- tool-permission prompt가 열려있는 상태에서 permission mode를 전환할 때 새 setting이 tool을 permit하는 경우 prompt를 auto-dismiss하지 않는 문제를 수정
- permission/dialog prompt가 열려있는 상태에서 Enter를 누르면 input box의 text도 함께 submit되는 문제를 수정
- `EnterWorktree`가 작업 directory를 전환한 후 hooks가 non-existent `transcript_path`를 받는 문제를 수정
- cell wrapping이 있는 markdown tables가 bordered grid로 렌더링되지 않고 vertical key-value layout으로 fallback되는 문제를 수정 (2.1.136 버전의 regression)
- cancelled prompts가 input box에 auto-restored될 때 Up-arrow history에서 제거되어 중복 entries를 방지하는 문제를 수정
- response가 시작되기 전에 Ctrl+C/Esc로 cancelled된 prompts가 Up-arrow history에서 제거되는 문제를 수정
- vim INSERT/VISUAL mode에서 Ctrl+C가 running turn을 interrupt하지 않는 문제를 수정
- alternative `chat:submit` keybindings (예: `meta+enter`, `ctrl+enter`)가 `enter`를 `chat:newline`으로 rebind한 경우 작동하지 않는 문제를 수정
- output style이 구성된 경우 prompt suggestions가 silent하게 disabled되는 문제를 수정
- `spinnerVerbs` setting이 turn-completion messages에서 honored되지 않는 문제를 수정
- AskUserQuestion popup이 선행 chat content의 마지막 줄을 숨기는 문제를 수정
- Web Search status가 searches에서 errors를 반환할 때 "Did 0 searches"를 표시하는 문제를 수정
- 여러 줄 statusline output이 어떤 줄이 terminal width를 초과할 때 rows가 드롭되거나 손상되는 문제를 수정
- light-ansi theme이 light backgrounds에서 diff context lines에 invisible white를 사용하는 문제를 수정 — 이제 black을 사용
- error overlay가 original error message를 숨기는 minified bundle source를 dump하는 문제를 수정
- feedback survey rating digit을 입력한 후 Enter를 누르면 rating 대신 chat message로 submit되는 문제를 수정
- agent panel에서 선택된 subagent에 `x`를 누르면 agent를 stopping하는 대신 prompt에 typing되는 문제를 수정
- session title이 user의 첫 prompt 이전에 plugin monitor notifications에서 derived되는 문제를 수정
- collapsed read/search group 아래에서 tool call당 한 번 "Allowed by PermissionRequest hook"이 반복되는 문제를 수정
- `/tui`가 running background shells과 subagents를 silent하게 drop하는 문제를 수정 — 이제 거절하고 완료될 때까지 기다리도록 요청
- welcome banner가 Bedrock, Vertex, Foundry 및 다른 third-party providers에서 "API Usage Billing"을 표시하는 문제를 수정 — 이제 provider name을 표시
- `/mcp` server list가 short terminals의 fullscreen mode에서 focused server를 visible하게 유지하지 않는 문제를 수정
- `/feedback` bundles의 redaction이 session IDs와 같은 quoted values에 대해 invalid JSON을 생성하는 문제를 수정
- desktop 및 third-party provider sessions가 host managed-settings에서 `apiKeyHelper`/`ANTHROPIC_AUTH_TOKEN`을 incorrectly inherit하는 문제를 수정
- logger initialization 전에 fired된 early analytics events가 silent하게 drop되는 문제를 수정
- marketplace `ref`가 `sha`도 pinned된 경우 upstream에 더 이상 존재하지 않는 plugins에 대해 `claude plugin install`이 실패하는 문제를 수정
- plugin details pane이 `.mcp.json`을 통해 선언된 plugins에 대해 0 MCP servers를 표시하는 문제를 수정
- unset config variables를 가진 plugin MCP servers가 generic connection failure 대신 "config issue" message with a fix-it hint를 표시하지 않는 문제를 수정; malformed `.mcp.json` entries가 더 이상 다른 MCP servers를 drop하지 않음
- POSIX shell parameter expansions (예: `${var%pattern}`)를 사용하는 MCP server configs가 missing environment variables로 incorrectly flagged되는 문제를 수정
- 연결 시 403을 반환하는 MCP HTTP/SSE servers가 "failed" 대신 "needs auth"로 표시되는 문제를 수정
- optional server-events stream이 reconnect하지 못했을 때 remote MCP servers가 unnecessarily disconnect되는 문제를 수정 — tool calls는 POST를 통해 계속됨
- worker session token이 mid-session에 rotate될 때 모든 Remote Control MCP connectors가 401로 실패하는 문제를 수정
- Remote Control이 server가 stale token을 거절할 때 `/login`을 통해 looping하는 대신 trusted device를 automatically re-enroll하는 문제를 수정
- beta tracing이 enabled된 SDK/headless mode에서 early OTel spans이 silent하게 drop될 수 있는 race condition을 수정
- custom `voice:pushToTalk` keybindings 및 `"space": null` unbinds가 silent하게 ignored되는 문제를 수정
- Windows에서 clipboard에 screenshot이 포함된 경우 Alt+V image paste가 "no image found"를 report하는 문제를 수정
- glibc 및 musl platform packages가 모두 설치된 경우 Linux에서 SDK "Claude Code native binary not found"를 수정
- Bedrock: `awsCredentialExport`는 이제 configured될 때 항상 실행되며, ambient AWS credentials이 resolve될 때 skipped되지 않아 cross-account access를 위한 auth를 수정
- [VSCode] 채팅 중 mic이 microphone이 오직 silence만 생성했을 때 feedback을 보이지 않는 문제를 수정 — 이제 "No audio detected"를 표시
- [VSCode] Voice mode: WSL error는 이제 WSLg users에게 `sox libsox-fmt-pulse` 설치를 제안
- `claude agents`: pre-warmed background worker가 unhealthy할 때 session 실행이 실패하지 않음 — 이제 fresh launch로 fallback
- `claude agents`는 이제 background를 한 fresh REPL에서 남겨진 empty placeholder sessions을 표시하지 않으며, no other agents가 있는 ← 를 통해 입력될 때 onboarding text를 표시
- `←`에서 남겨진 empty idle background sessions는 이제 daemon에 의해 5분 후 자동으로 retired됨
