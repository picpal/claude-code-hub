## 변경사항

- Claude Opus 4.7 xhigh를 이제 사용할 수 있습니다! /effort를 사용하여 속도와 지능을 조정하세요
- Max 구독자가 Opus 4.7을 사용할 때 Auto mode를 이제 사용할 수 있습니다
- Opus 4.7에 `high`와 `max` 사이의 `xhigh` effort level을 추가했습니다. `/effort`, `--effort` 및 모델 선택기를 통해 사용 가능하며, 다른 모델은 `high`로 폴백됩니다
- `/effort`는 이제 인수 없이 호출할 때 대화형 슬라이더를 열며, 화살표 키로 레벨을 탐색하고 Enter로 확인합니다
- "Auto (match terminal)" 테마 옵션을 추가했습니다. 터미널의 밝음/어두움 모드와 일치하며, `/theme`에서 선택합니다
- `/less-permission-prompts` skill을 추가했습니다. 트랜스크립트에서 일반적인 읽기 전용 Bash 및 MCP tool 호출을 스캔하고 `.claude/settings.json`에 대한 우선순위가 지정된 allowlist를 제안합니다
- `/ultrareview`를 추가했습니다. 병렬 다중 agent 분석 및 평가를 사용하여 클라우드에서 포괄적인 코드 review를 실행합니다. 현재 branch를 검토하려면 인수 없이 호출하거나 특정 GitHub PR을 가져와서 검토하려면 `/ultrareview <PR#>`를 사용하세요
- Auto mode는 더 이상 `--enable-auto-mode`를 필요로 하지 않습니다
- Windows: PowerShell tool이 점진적으로 출시되고 있습니다. `CLAUDE_CODE_USE_POWERSHELL_TOOL`을 사용하여 opt in 또는 opt out하세요. Linux 및 macOS에서는 `CLAUDE_CODE_USE_POWERSHELL_TOOL=1`을 사용하여 활성화하세요 (PATH에 `pwsh` 필요)
- glob 패턴이 있는 읽기 전용 bash 명령 (예: `ls *.ts`)과 `cd <project-dir> &&`로 시작하는 명령은 더 이상 permission prompt를 트리거하지 않습니다
- `claude <word>`가 오타로 호출될 때 가장 가까운 일치 subcommand를 제안합니다 (예: `claude udpate` → "Did you mean `claude update`?")
- Plan file은 이제 순수 무작위 단어 대신 prompt의 이름을 따라 지정됩니다 (예: `fix-auth-race-snug-otter.md`)
- `/setup-vertex` 및 `/setup-bedrock`을 개선했습니다. `CLAUDE_CONFIG_DIR`이 설정되면 실제 `settings.json` 경로를 표시하고, 재실행 시 기존 pins에서 모델 후보를 시드하며, 지원되는 모델에 대해 "with 1M context" 옵션을 제공합니다
- `/skills` 메뉴는 이제 예상 token count로 정렬을 지원합니다. `t`를 눌러 전환하세요
- `Ctrl+U`는 이제 전체 입력 버퍼를 지웁니다 (이전: 라인 시작까지 삭제). `Ctrl+Y`를 눌러 복원하세요
- `Ctrl+L`은 이제 prompt 입력을 지우는 것 외에도 전체 화면 재드로우를 강제합니다
- Transcript view footer는 이제 `[` (scrollback으로 dump) 및 `v` (editor에서 열기) 단축키를 표시합니다
- 잘린 긴 pastes의 "+N lines" marker는 이제 스캔하기 쉬운 전체 너비 rule입니다
- Headless `--output-format stream-json`은 이제 plugins이 충족되지 않은 dependencies로 인해 강등될 때 init event에 `plugin_errors`를 포함합니다
- 디버깅을 위해 전체 API request 및 response bodies를 OpenTelemetry log events로 내보내는 `OTEL_LOG_RAW_API_BODIES` 환경 변수를 추가했습니다
- 정상 작동 중 TUI에 나타날 수 있는 거짓 decompression, network 및 transient error messages를 억제했습니다
- v2.1.110의 non-streaming fallback retries 제한을 되돌렸습니다. API 과부하 중에 더 많은 outright failures를 위해 긴 대기를 교환했습니다
- iTerm2 + tmux 설정에서 터미널 notifications이 전송될 때 terminal display tearing (무작위 문자, drifting input)을 수정했습니다
- non-git 작업 디렉토리에서 `@` file suggestions이 매 turn마다 전체 프로젝트를 재스캔하고, tracked files이 없는 새로 초기화된 git repos에서 config files만 표시하는 문제를 수정했습니다
- 편집 전의 LSP diagnostics이 편집 후에 나타나 모델이 방금 편집한 파일을 다시 읽으도록 하는 문제를 수정했습니다
- tab-completing `/resume`이 session picker를 표시하는 대신 임의의 titled session을 즉시 재개하는 문제를 수정했습니다
- `/context` grid 렌더링이 행 사이에 추가 빈 라인이 있는 문제를 수정했습니다
- `/clear`가 `/rename`으로 설정된 session name을 드롭하여 statusline output이 `session_name`을 잃는 문제를 수정했습니다
- Plugin error 처리를 개선했습니다. dependency errors는 이제 conflicting, invalid 및 overly complex version requirements를 구분합니다. `plugin update` 후 stale resolved versions를 수정했습니다. `plugin install`은 이제 interrupted prior installs에서 복구됩니다
- Claude가 존재하지 않는 `commit` skill을 호출하고 custom `/commit` command가 없는 사용자에게 "Unknown skill: commit"을 표시하는 문제를 수정했습니다
- Bedrock/Vertex/Foundry의 429 rate-limit errors이 status.claude.com을 참조하는 문제를 수정했습니다 (Anthropic이 운영하는 providers만 포함)
- 하나를 해제한 후 feedback surveys가 연속해서 나타나는 문제를 수정했습니다
- bash/PowerShell/MCP tool output의 bare URLs이 터미널이 여러 라인에 걸쳐 래핑될 때 클릭할 수 없는 문제를 수정했습니다
- Windows: `CLAUDE_ENV_FILE` 및 SessionStart hook environment files이 이제 적용됩니다 (이전: no-op)
- Windows: drive-letter path를 사용한 permission rules은 이제 올바르게 root-anchored되며, drive-letter case만 다른 paths는 같은 path로 인식됩니다
