## 변경된 사항
- 타사 게이트웨이에서 `ANTHROPIC_BASE_URL`을 사용할 때 발생하던 API 400 오류를 수정했습니다. 이제 도구 검색 기능이 프록시 엔드포인트를 올바르게 감지하고 `tool_reference` 블록을 비활성화합니다.
- 사용자 정의 Bedrock 추론 프로필이나 표준 Claude 명명 규칙과 일치하지 않는 기타 모델 식별자를 사용할 때 발생하는 `API 오류: 400 이 모델은 effort 매개변수를 지원하지 않습니다` 오류를 수정했습니다.
- `ToolSearch` 직후 모델 응답이 비어 있는 문제를 수정했습니다. 서버가 프롬프트 끝에 시스템 프롬프트 스타일 태그를 포함한 도구 스키마를 렌더링하는 바람에, 모델이 이를 오인하여 조기에 중단하는 경우가 있었습니다.
- 첫 번째 턴이 끝난 후 `instructions`를 가진 MCP 서버가 연결될 때 발생하는 프롬프트 캐시 오류 문제를 수정했습니다.
- 느린 SSH 연결 상태에서 입력 시 Enter 키를 누르면 전송 대신 새 줄이 삽입되던 문제를 수정했습니다.
- Windows/WSL 환경에서 PowerShell의 `Set-Clipboard` 명령어를 사용할 때 비-ASCII 문자(CJK, 이모지)가 클립보드에서 손상되는 문제를 수정했습니다.
- VS Code 통합 터미널에서 실행할 때 Windows에서 시작 시 불필요한 VS Code 창이 추가로 열리는 문제를 수정했습니다.
- Windows 네이티브 바이너리에서 음성 모드가 "네이티브 오디오 모듈을 로드할 수 없음" 오류로 인해 작동하지 않던 문제를 수정했습니다.
- 설정에서 `voiceEnabled: true`로 설정된 경우 세션 시작 시 푸시투톡이 활성화되지 않던 문제를 수정했습니다.
- `#NNN` 참조가 포함된 마크다운 링크가 연결된 URL 대신 현재 저장소로 잘못 연결되던 문제를 수정했습니다.
- 프로젝트의 `.claude/settings.json` 파일에 레거시 Opus 모델 문자열이 고정되어 있을 때 반복적으로 표시되던 “모델이 Opus 4.6으로 업데이트되었습니다” 알림 문제를 수정했습니다
- `/plugin` 디렉터리에서 플러그인이 잘못 설치된 것으로 표시되던 문제를 수정했습니다.
- 마켓플레이스 설치 후 자동 새로고침 기능으로 인해, 프로그램을 처음 실행할 때 플러그인이 “마켓플레이스에서 찾을 수 없음” 오류를 표시하던 문제를 수정했습니다.
- 구버전 Git에서 `/security-review` 명령어가 `unknown option merge-base` 오류로 실패하던 문제를 수정했습니다.
- `/color` 명령어로 기본 색상으로 되돌릴 수 없던 문제를 수정했습니다. 이제 `/color default`, `/color gray`, `/color reset`, `/color none` 명령어를 사용하면 기본 색상으로 복원됩니다.
- 메모 입력창에서 키를 누를 때마다 마크다운 렌더링이 다시 실행되어 발생하던 `AskUserQuestion` 미리보기 대화 상자의 성능 저하 문제를 수정했습니다.
- 초기 시작 단계에서 읽은 기능 플래그의 디스크 캐시가 절대 갱신되지 않아, 세션 간에 오래된 값이 지속되던 문제를 수정했습니다.
- Claude Code Remote 환경에서 `acceptEdits` 또는 `plan` 이외의 `permissions.defaultMode` 설정 값이 적용되던 문제를 수정했습니다. 이제 해당 값들은 무시됩니다.
- `--resume` 명령어를 실행할 때마다 스킬 목록이 다시 불러오는 문제를 수정했습니다(재시작 시 약 600개의 토큰 절약).
- VS Code의 순간 이동 세션에서 순간 이동 마커가 표시되지 않던 문제를 수정했습니다.
- 마이크에서 소리가 감지되지 않을 때 표시되는 오류 메시지를 개선하여 “음성 감지되지 않음”과 명확히 구분할 수 있도록 했습니다.
- 요약기 요청 내 이미지를 보존하도록 압축 기능을 개선하여, 프롬프트 캐시를 재사용함으로써 더 빠르고 비용 효율적인 압축이 가능해졌습니다.
- `/rename` 명령어가 더 이상 아무런 알림 없이 대기열에 쌓이지 않고, Claude가 처리 중인 동안에도 작동하도록 개선되었습니다.
- 턴 진행 중 프롬프트 입력에 따른 화면 재렌더링이 약 74% 감소했습니다
- 사용자 지정 CA 인증서가 없는 사용자의 경우 시작 시 메모리 사용량을 약 426KB 줄였습니다
- 연결 상태에서는 원격 제어 `/poll` 요청 빈도를 10분에 한 번으로 줄였으며(기존 1~2초), 이를 통해 서버 부하를 약 300배 감소시켰습니다. 재연결에는 영향을 미치지 않으며, 전송 오류가 발생하면 즉시 빠른 폴링이 재개됩니다.
- [VSCode] VS Code 활동 표시줄에 모든 Claude Code 세션을 나열하는 스파크 아이콘을 추가했으며, 해당 세션을 클릭하면 전체 화면 편집기로 열립니다.
- [VSCode] VS Code에서 플랜을 전체 마크다운 문서 형식으로 볼 수 있는 기능을 추가했으며, 피드백을 남길 수 있도록 댓글 작성 기능도 지원합니다.
- [VSCode] 기본 제공 MCP 서버 관리 대화 상자가 추가되었습니다. 채팅 패널에서 `/mcp` 명령어를 사용하여 터미널로 전환하지 않고도 서버를 활성화/비활성화하고, 다시 연결하며, OAuth 인증을 관리할 수 있습니다.
