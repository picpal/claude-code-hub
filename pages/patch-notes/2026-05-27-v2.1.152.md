## 변경사항

- `/code-review --fix`는 이제 리뷰 후 리뷰 결과를 작업 트리에 적용하며, 재사용, 단순화 및 효율성 제안을 표시합니다. `/simplify`는 이제 `/code-review --fix`를 호출합니다.
- Skills 및 slash commands는 이제 frontmatter에서 `disallowed-tools`을 설정하여 skill이 활성화되는 동안 모델에서 도구를 제거할 수 있습니다.
- `/reload-skills` 명령을 추가하여 세션을 다시 시작하지 않고도 skill 디렉토리를 다시 스캔합니다.
- `SessionStart` hooks는 이제 `reloadSkills: true`를 반환하여 skill 디렉토리를 다시 스캔할 수 있으며, hook에서 설치된 skills을 동일 세션에서 사용 가능하게 합니다.
- `SessionStart` hooks는 이제 시작 및 재개 시 `hookSpecificOutput.sessionTitle`을 통해 세션 제목을 설정할 수 있습니다.
- 관리자가 컨텍스트 인식 팁을 통해 제안될 수 있는 org marketplaces을 허용 목록에 추가할 수 있는 `MessageDisplay` hook 이벤트를 추가했습니다.
- `pluginSuggestionMarketplaces` managed setting을 추가했습니다.
- `claude plugin marketplace remove`는 이제 `marketplace add`, `install`, `uninstall`과의 대칭성을 위해 `--scope user|project|local`을 허용합니다.
- Claude Code는 이제 primary model이 없는 경우 모든 요청이 실패하는 대신 세션의 나머지 부분에 대해 구성된 `--fallback-model`로 전환합니다.
- Auto mode는 더 이상 opt-in 동의가 필요하지 않습니다.
- Vim mode: NORMAL 모드의 `/`는 이제 역방향 히스토리 검색을 열며, bash/zsh vi-mode와 일치합니다(Ctrl+R 같음).
- `/usage` 분석에는 이제 대용량 세션 파일이 포함되며, 파일은 streaming read로 스캔되어 메모리 사용량이 일정합니다.
- Collapsed group의 thinking 요약은 이제 최소 3초간 읽을 수 있으며, markdown으로 렌더링되고 10줄로 제한됩니다(`Ctrl+O`는 전체 thinking을 표시합니다).
- 전체 화면 모드에서 "Thinking for Ns" 표시기는 이제 모델이 생각하는 동안 실시간으로 계산되며, 중간 생각을 중단하면 값을 유지합니다.
- Workflow tool의 인라인 progress 표시를 단순화했습니다. 실시간 agent 수는 이제 프롬프트 아래의 persistent workflow 상태 행에만 표시됩니다.
- post-response timer는 이제 backgrounded agents 또는 workflows가 여전히 실행 중일 때 "Waiting for N background agents/workflows to finish"를 표시하며, 결과 처리 후 누적 시간을 보고합니다.
- session entrypoint를 OpenTelemetry metric 속성으로 추가했습니다(`app.entrypoint`, `OTEL_METRICS_INCLUDE_ENTRYPOINT=true`를 통해 opt-in).
- 매우 긴 세션에서 renderer의 style pool을 재활용하여 terminal styling 저하를 수정했습니다.
- condensed startup mode에서 sandbox 활성화 경고가 표시되지 않는 문제를 수정했습니다. 이제 모든 레이아웃에 표시됩니다.
- tool이 실행 중일 때 loading spinner가 "still thinking"/"almost done thinking"을 표시하는 문제를 수정했으며, 각 tool 후에 thinking 상태를 "thinking"으로 리셋합니다.
- focus mode가 숨겨진 활동이 없는 턴에서 허위 "N messages hidden" 수를 표시하는 문제를 수정했습니다.
- expanded tool 결과 내 링크를 클릭할 때 링크를 열지 않고 섹션을 축소하는 문제를 수정했습니다.
- markdown 테이블 셀 borders가 inline code의 색상을 상속하는 문제, 줄 바꿈 continuation 줄이 스타일을 잃는 문제, 그리고 narrow-terminal stacked layout에서 빈 header 셀이 레이블을 표시하는 문제를 수정했습니다.
- 동일한 command이지만 다른 환경 변수를 가진 plugin MCP servers이 잘못되게 중복 제거되는 문제를 수정했습니다.
- `/doctor`가 제거된 marketplaces를 참조하는 stale `enabledPlugins` 항목에 대해 "marketplace not found" 또는 "plugin not found"를 보고하는 문제를 수정했습니다.
- git branch를 추적하는 plugins이 plugin registry가 다시 빌드된 후 자동으로 업데이트를 받지 못하는 문제를 수정했습니다.
- remote MCP servers이 egress proxy가 활성화되었을 때 Claude Code Remote 세션에서 연결을 실패하는 문제를 수정했습니다.
- effort-change 확인 대화 상자가 대화에 메시지가 없거나 동일한 underlying value로 해결되는 effort 레벨 간에 전환할 때 나타나는 문제를 수정했습니다.
- Agent tool 설명이 `--bare`로 실행하거나 attachments를 비활성화했을 때 전달되지 않는 agent 목록을 참조하는 문제를 수정했습니다.
- subagent가 취소된 후 stale permission prompt를 수락할 때 `claude agents`에서 background worker crash가 발생하는 문제를 수정했습니다.
- `cache_creation_input_tokens`이 API가 nested `cache_creation` breakdown을 통해서만 cache writes를 보고할 때 transcript 및 result usage에서 0으로 보고되는 문제를 수정했습니다.
- PushNotification tool이 Remote Control이 활성화되었을 때 SDK-hosted 세션에서 "Mobile push not sent (Remote Control inactive)"을 잘못 보고하는 문제를 수정했습니다.
- model 또는 login 전환이 history에 stale thinking-block signature를 남겨 세션이 고착되는 문제를 수정했습니다. 이제 retry safety-net으로 사전에 제거됩니다.
