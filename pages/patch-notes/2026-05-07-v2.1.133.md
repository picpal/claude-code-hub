> 원문: [Claude Code v2.1.133 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.133)

## 변경사항

- `worktree.baseRef` 설정(`fresh` | `head`)을 추가하여 `--worktree`, `EnterWorktree`, agent-isolation worktree가 `origin/<default>` 또는 로컬 `HEAD`에서 분기할지 선택할 수 있게 했습니다. **참고:** 기본값 `fresh`는 `EnterWorktree`의 기본 위치를 `origin/<default>`으로 변경합니다(2.1.128 이후로는 로컬 `HEAD`였음) — 새 worktree에서 푸시되지 않은 커밋을 유지하려면 `worktree.baseRef: "head"`로 설정하세요
- `sandbox.bwrapPath` 및 `sandbox.socatPath` 관리 설정(Linux/WSL)을 추가하여 사용자 정의 bubblewrap 및 socat 바이너리 위치를 지정할 수 있게 했습니다
- `parentSettingsBehavior` 관리자 계층 키(`'first-wins' | 'merge'`)를 추가하여 관리자가 SDK `managedSettings`(부모 계층)을 정책 병합에 포함시킬 수 있게 했습니다
- Hook은 이제 `effort.level` JSON 입력 필드 및 `$CLAUDE_EFFORT` 환경 변수를 통해 활성 노력 수준을 수신하며, Bash 도구 명령은 `$CLAUDE_EFFORT`을 읽을 수 있습니다
- 포커스 모드 동작을 개선했습니다
- 메모리 압박 상황에서 웜-스페어 백그라운드 워커를 해제하여 메모리 사용량을 개선했습니다
- 새로고침 토큰 레이스로 인해 공유 자격 증명이 초기화되어 모든 병렬 세션이 401 상태에서 중단되는 문제를 수정했습니다
- `Edit`/`Write` 허용 규칙이 드라이브 루트(`C:\`) 또는 POSIX `/`로 스코프되었을 때 잘못 일치하여 항상 프롬프트를 표시하는 문제를 수정했습니다
- 클록 스큐 또는 느린 디스크로 인해 history 또는 session-log 파일 잠금이 손상될 때 처리되지 않은 거부(`ECOMPROMISED`)를 수정했습니다
- 대화 압축 중 Esc 키를 누르면 거짓 "Error compacting conversation" 알림이 표시되는 문제를 수정했습니다
- `HTTP(S)_PROXY` / `NO_PROXY` / mTLS이 discovery, 동적 클라이언트 등록, 토큰 교환, 토큰 새로고침을 포함한 전체 MCP OAuth 플로우에서 존중되지 않는 문제를 수정했습니다
- Read/Write/Edit이 `--add-dir` / SDK `additionalDirectories`를 통해 전달된 매핑된 네트워크 드라이브에서 거부되는 문제를 수정했습니다
- Remote Control이 claude.ai에서 중지/중단할 때 로컬 Esc처럼 CLI 세션을 완전히 취소하지 않아 중단된 도구 또는 프롬프트 후에 대기 중인 메시지가 진행되지 않는 문제를 수정했습니다
- 한 세션의 `/effort`이 다른 동시 세션의 노력 수준을 예기치 않게 변경하는 문제와 IDE 노력 변경이 자동으로 삭제될 수 있는 관련 문제를 수정했습니다
- subagent가 Skill 도구를 통해 프로젝트, 사용자 또는 플러그인 skill을 발견하지 못하는 문제를 수정했습니다
- `claude --help`에서 이제 `--remote-control-session-name-prefix` 옆에 `--remote-control`을 나열합니다
- [VSCode] 확장 빌드가 Claude 바이너리를 번들로 포함하지 않을 때 `claudeCode.claudeProcessWrapper`가 "Unsupported platform" 오류로 실패하는 문제를 수정했습니다
