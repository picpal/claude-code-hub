## 변경사항

- 로그인이 곧 만료될 때 경고를 추가하여 백그라운드 세션이 중단되기 전에 다시 인증할 수 있도록 함
- 수동 권한 모드에서 회색 ⏸ 배지를 푸터에 추가하여 활성 모드를 항상 표시하도록 함
- MCP `roots/list`에 세션의 추가 작업 디렉토리를 추가하고, 집합이 변경될 때 `notifications/roots/list_changed`를 전송하도록 함
- macOS에서 거짓 메모리 부족 감지로 인해 백그라운드 에이전트 세션을 열거나 전환할 때 15-20초 동안 멈추는 문제 수정(2.1.196 버전의 회귀)
- daemon의 세션 토큰이 만료되었을 때 백그라운드 세션이 attach, 회신 및 중지 요청에 영구적으로 응답하지 않는 문제 수정 — 세션이 이제 자동으로 복구됨
- `claude agents`로 돌아갈 때 실행 중인 subagent를 조용히 중지하고 프롬프트를 처음부터 다시 실행하는 문제 수정 — 이제 작업이 유지됨
- 대화형 세션에서의 메모리 및 턴당 CPU 회귀 수정: context-usage 표시기가 더 이상 매 턴 후 전체 transcript를 다시 분석하지 않음
- 백그라운드 에이전트가 daemon의 stale한 `PATH`를 상속하는 문제 수정(dispatching shell 대신) — Windows에서 누락된 도구 유발
- 백그라운드 및 agent-view 세션이 shell로 내보낸 `ANTHROPIC_BASE_URL`을 제거하는 문제 수정 — API 키를 기본 endpoint로 전송하고 401로 실패
- 많은 git worktree가 있는 repository에서 Bash가 "argument list too long"으로 실패하는 문제 수정
- Worktree로 격리된 subagent가 자신의 worktree 대신 부모 checkout에서 shell 명령을 실행하는 문제 수정
- Worktree 생성 시 multi-repo workspace의 중첩 repository를 거부하여 백그라운드 세션이 격리 및 편집을 할 수 없는 문제 수정
- 작업 디렉토리가 삭제되거나 파일로 교체되었거나 유효하지 않은 경로가 되었을 때 백그라운드 에이전트가 crash-loop하는 문제 수정 — 이제 명확한 오류 메시지와 함께 한 번 실패
- 백그라운드 daemon auto-upgrade 실패로 실행 중인 모든 백그라운드 세션이 조용히 중단되는 문제 수정
- `TaskStop` 및 `TaskOutput`이 다른 에이전트에 의해 생성된 백그라운드 에이전트를 찾지 못하는 문제 수정 — 이제 오류가 id 및 description으로 실행 중인 에이전트를 나열
- `claude agents` composer가 slash command를 사용할 수 없는 위치에서 입력한 메시지를 버리는 문제 수정
- 이미 다른 세션에서 열려 있는 대화의 중단된 세션을 열 때 에이전트 목록이 crash하는 문제 수정
- 질문이 이미 답변된 후에도 백그라운드 세션이 에이전트 목록에 "Needs input"으로 표시되는 문제 수정
- 백그라운드 에이전트 시작 실패 시 실제 오류 대신 "exit_with_message"만 표시되는 문제 수정
- 백그라운드 세션이 daemon을 통해 fork될 때 settings.json의 `effortLevel` 변경을 무시하는 문제 수정
- 연결된 백그라운드 세션이 `CLAUDE_CODE_DISABLE_MOUSE` 및 `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` 옵트아웃을 무시하는 문제 수정
- 모든 명명된 에이전트가 완료된 후 `/exit`이 실행 중인 백그라운드 에이전트에 대해 잘못된 경고를 표시하는 문제 수정
- non-git 디렉토리에서 시작된 백그라운드 세션이 `WorktreeCreate` hook이 구성되었을 때 파일을 편집할 수 없는 문제 수정
- `claude agents`의 `@` 디렉토리 선택기가 등록된 git worktree를 표시하지 않는 문제 수정
- Windows에서 `/clear` 후 백그라운드 작업 출력이 빈 파일로 영구적으로 교체되는 문제 수정
- 긴 transcript 기록을 스크롤할 때 콘텐츠가 점프하는 문제 수정
- bash 모드에서 shell-history 제안이 표시될 때 입력 중 terminal이 깜박이고 점프하는 문제 수정
- 백그라운드 세션에 다시 연결할 때 문자 그대로의 `^[[I` / `^[[O` escape code가 출력되는 문제 수정
- LSP 전용 plugin이 해당 language server가 diagnostics를 제공하거나 navigation 요청에 응답할 때 잘못된 미사용으로 표시되는 문제 수정
- 긴 응답이 스트리밍되는 동안 응답성 개선: live-preview 업데이트가 더 이상 전체 화면을 다시 렌더링하지 않음
- Subagent 동작 개선: 에이전트가 전체 작업을 다른 subagent에 다시 위임할 가능성이 감소
- 크기가 큰 번들 dependency를 inlining하는 대신 lazy하게 로드하여 binary 크기를 ~7 MB, 시작 메모리를 ~7 MB 감소
- 왼쪽 화살표가 더 이상 백그라운드 작업, diff 및 workflow 상세 보기를 닫지 않도록 변경 — 대신 Esc를 누름
- 빈 `claude agents` 보기가 항상 구성된 섹션(Needs input / Working / Completed)을 설명과 함께 표시하도록 변경
- 시작 시 "claude command missing or broken" 경고 제거 — 이제 `/doctor` 및 `/status`에 표시됨
- `claude agents` 푸터에서 중복 navigation hint 제거
- [VSCode] "Enable Remote Control for all sessions" 설정 토글 추가
