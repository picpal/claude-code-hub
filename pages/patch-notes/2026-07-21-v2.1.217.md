## 변경사항

- prompt input에서 emoji shortcode 자동완성 추가: `:heart:`를 입력하여 ❤️를 삽입하거나, `:hea`를 입력하여 제안을 받을 수 있습니다 — `emojiCompletionEnabled` 설정으로 비활성화할 수 있습니다
- transcript 쓰기가 실패하거나 (예: 디스크 부족) 상속된 환경 변수로 인해 session 저장이 꺼져 있을 때 경고 추가: 이제 transcript가 자동으로 손실되지 않습니다
- 잘린 MCP tool 출력이 전체 잘리지 않은 결과를 session 동안 메모리에 유지하는 메모리 누수 수정
- Windows auto-update 실패로 인해 `claude.exe`가 누락될 수 있는 문제 수정: 실패한 업데이트는 이제 보존된 실행 파일을 자동으로 복원합니다
- background session 격리가 symlinked working directories를 정규화하지 않아 session이 workspace folder를 벗어날 수 있는 문제 수정
- auto-compact가 Bedrock의 Claude Opus 4.8에 대해 트리거되지 않고 `/compact`가 제한을 초과하면 실패하는 문제 수정
- corporate mTLS, TLS-verify, OAuth scope, proxy 설정이 Claude Desktop session에서 무시되는 문제 수정
- screen reader mode의 시작 공지가 첫 번째 prompt 렌더링으로 잘리는 문제와 thinking 상태 행이 경과 시간 및 token 개수를 업데이트하기 위해 몇 초마다 다시 렌더링되는 문제 수정
- `OTEL_EXPORTER_OTLP_ENDPOINT`를 설정하는 managed settings가 모든 signal을 관리하지 않는 문제 수정 — 낮은 범위의 signal 특정 재정의가 더 이상 telemetry를 관리되는 endpoint에서 리디렉션하지 않습니다
- transcript에 형식이 잘못된 attachment entry가 있을 때 `--resume`/`--continue` 및 `/resume`이 TypeError로 실패하는 문제 수정
- Remote Control session이 permission prompt나 dialog가 나타난 후 연결된 viewer에게 보류 중인 상태를 표시하지 않는 문제 수정
- background session으로 전송되거나 (`/background` 또는 `←`) session이 종료될 때 background shell이 중지 불가능해지는 경우가 있는 문제 수정 (부하가 높은 시스템에서 가장 명확하고, Windows에서 특히 눈에 띕니다)
- 많은 brace group이 있는 `CLAUDE.md` 또는 `SKILL.md` paths frontmatter 값으로 인해 CLI가 시작 시 OOM-kill되거나 정지하는 문제 수정 — brace expansion이 이제 예산 제한됩니다
- starting background session에 연결할 때 transcript preview가 input 영역에 바싹 붙는 문제 수정: 이제 live layout과 같은 한 줄 간격을 남기므로, session이 인계받을 때 transcript가 더 이상 이동하지 않습니다
- footer PR badge 링크가 terminal 지원을 감지할 수 없을 때도 (예: ssh/tmux를 통해) 클릭 가능한 하이퍼링크가 되도록 개선; `FORCE_HYPERLINK=0`으로 거부할 수 있습니다
- login 만료 경고가 만료 5일 전이 아니라 3일 전에 표시되도록 변경
- frontend-design plugin 제안 팁을 무기한 반복하는 대신 평생 최대 3회 노출로 제한
- 동시 실행 중인 subagent의 상한선 추가 (기본값 20, `CLAUDE_CODE_MAX_CONCURRENT_SUBAGENTS`로 재정의) — 하나의 message가 무제한 background agent를 생성하지 않도록 합니다
- subagent가 기본적으로 nested subagent를 더 이상 생성하지 않도록 변경; `CLAUDE_CODE_MAX_SUBAGENT_SPAWN_DEPTH`를 설정하여 더 깊은 nesting을 허용할 수 있습니다
- `--max-budget-usd`가 background subagent를 중지하지 않는 문제 수정: 상한선에 도달하면 새로운 생성이 거부되고 실행 중인 background agent가 중지됩니다
