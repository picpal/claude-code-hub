> 원문: [Claude Code v2.1.97 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.97)

## 변경사항

- `NO_FLICKER` 모드에서 포커스 뷰 토글(`Ctrl+O`)을 추가했습니다. 포커스 뷰는 prompt, 한 줄의 tool 요약(edit diffstats 포함), 최종 응답을 표시합니다
- status line 설정에 `refreshInterval`을 추가했습니다. N초마다 status line 명령을 재실행합니다
- status line JSON 입력에 `workspace.git_worktree`을 추가했습니다. 현재 디렉터리가 연결된 git worktree 내부에 있을 때 설정됩니다
- `/agents`에서 `● N running` 지표를 추가했습니다. live subagent 인스턴스가 있는 agent 타입 옆에 표시됩니다
- Cedar policy 파일(`.cedar`, `.cedarpolicy`)에 대한 구문 강조를 추가했습니다
- 보호된 경로에 write를 승인한 후 `--dangerously-skip-permissions`이 자동으로 accept-edits 모드로 다운그레이드되던 문제를 수정했습니다
- Bash tool 권한을 수정 및 강화했습니다. env-var 프리픽스와 network 리다이렉트 관련 검사를 강화하고, 일반적인 명령어에서의 false 프롬프트를 감소시켰습니다
- JavaScript prototype 속성과 이름이 일치하는 권한 규칙(예: `toString`)이 `settings.json`을 자동으로 무시하도록 만들던 문제를 수정했습니다
- 관리자가 제거한 후에도 managed-settings 허용 규칙이 프로세스 재시작까지 활성 상태를 유지하던 문제를 수정했습니다
- `permissions.additionalDirectories` 변경 사항이 mid-session에 적용되지 않던 문제를 수정했습니다
- `settings.permissions.additionalDirectories`에서 디렉터리를 제거할 때 `--add-dir`을 통해 전달된 동일한 디렉터리의 액세스가 취소되던 문제를 수정했습니다
- MCP HTTP/SSE 연결이 서버 재연결 시 약 50 MB/hr의 해제되지 않은 버퍼를 누적하던 문제를 수정했습니다
- MCP OAuth `oauth.authServerMetadataUrl`이 재시작 후 토큰 갱신 시 적용되지 않던 문제를 수정했습니다. ADFS 및 유사 IdP를 수정했습니다
- 서버가 작은 `Retry-After`를 반환할 때 429 재시도가 ~13초 내에 모든 시도를 소진하던 문제를 수정했습니다. 지수 백오프가 이제 최소값으로 적용됩니다
- context 압축 후 rate-limit 업그레이드 옵션이 사라지던 문제를 수정했습니다
- `/resume` picker의 여러 문제를 수정했습니다: `--resume <name>`이 편집 불가능하게 열리던 문제, Ctrl+A 재로드가 검색을 초기화하던 문제, 빈 목록이 네비게이션을 삼키던 문제, task-status 텍스트가 대화 요약을 대체하던 문제, cross-project 오래된 상태
- `--resume` 시 편집된 파일이 10KB보다 클 때 file-edit diff가 사라지던 문제를 수정했습니다
- `--resume` cache 미스 및 attachment 메시지로부터의 손실된 mid-turn 입력이 transcript에 저장되지 않던 문제를 수정했습니다
- Claude가 작업 중일 때 입력한 메시지가 transcript에 유지되지 않던 문제를 수정했습니다
- 긴 세션에서 prompt-type `Stop`/`SubagentStop` hook이 실패하고, hook 평가자 API 오류가 실제 메시지 대신 "JSON validation failed"를 표시하던 문제를 수정했습니다
- worktree 격리 또는 `cwd:` 재정의가 있는 subagent가 작업 디렉터리를 부모 세션의 Bash tool로 유출하던 문제를 수정했습니다
- prompt-too-long 재시도 시 compaction이 중복 multi-MB subagent transcript 파일을 쓰던 문제를 수정했습니다
- `claude plugin update`가 원격에 최신 커밋이 있을 때 git 기반 marketplace 플러그인에 대해 "already at the latest version"이라고 보고하던 문제를 수정했습니다
- 플러그인의 frontmatter `name`이 YAML 부울 키워드일 때 slash 명령어 picker가 중단되던 문제를 수정했습니다
- `NO_FLICKER` 모드에서 래핑된 URL을 복사할 때 줄 바꿈에 공백을 삽입하던 문제를 수정했습니다
- `NO_FLICKER` 모드에서 zellij 내에서 실행할 때 scroll 렌더링 아티팩트가 발생하던 문제를 수정했습니다
- `NO_FLICKER` 모드에서 MCP tool 결과 위에 마우스를 올릴 때 발생하던 crash를 수정했습니다
- `NO_FLICKER` 모드에서 API 재시도가 stale streaming 상태를 남기던 메모리 누수를 수정했습니다
- Windows Terminal에서 `NO_FLICKER` 모드의 slow 마우스 휠 스크롤을 수정했습니다
- 터미널이 24행보다 짧을 때 `NO_FLICKER` 모드에서 custom status line이 표시되지 않던 문제를 수정했습니다
- Warp에서 `NO_FLICKER` 모드일 때 Shift+Enter 및 Alt/Cmd+arrow 단축키가 작동하지 않던 문제를 수정했습니다
- Windows의 no-flicker 모드에서 복사할 때 한국어/일본어/Unicode 텍스트가 손상되던 문제를 수정했습니다
- `AWS_BEARER_TOKEN_BEDROCK` 또는 `ANTHROPIC_BEDROCK_BASE_URL`이 빈 문자열로 설정되어 있을 때(GitHub Actions가 설정되지 않은 입력에 대해 수행하는 경우) Bedrock SigV4 인증이 실패하던 문제를 수정했습니다
- Accept Edits 모드를 개선하여 safe env var 또는 프로세스 wrapper가 프리픽스된 filesystem 명령어를 자동으로 승인하도록 했습니다(예: `LANG=C rm foo`, `timeout 5 mkdir out`)
- auto 모드 및 bypass-permissions 모드를 개선하여 sandbox network 액세스 프롬프트를 자동으로 승인하도록 했습니다
- sandbox를 개선했습니다: `sandbox.network.allowMachLookup`이 이제 macOS에서 적용됩니다
- 이미지 처리를 개선했습니다: 붙여넣기 및 첨부된 이미지가 이제 Read tool을 통해 읽은 이미지와 동일한 token 예산으로 압축됩니다
- slash 명령어 및 `@`-mention 완료를 개선하여 CJK 문장 구두점 후에 트리거되도록 했으므로, 일본어/중국어 입력이 더 이상 `/` 또는 `@` 앞의 공백을 필요로 하지 않습니다
- Bridge 세션을 개선하여 claude.ai 세션 카드에 로컬 git repo, 분기 및 작업 디렉터리를 표시하도록 했습니다
- footer 레이아웃을 개선했습니다: 지표(Focus, notifications)가 이제 아래로 래핑되지 않고 mode-indicator 행에 유지됩니다
- context-low 경고를 persistent 행 대신 transient footer 알림으로 표시하도록 개선했습니다
- markdown blockquote를 개선하여 래핑된 줄 전체에 걸쳐 연속 왼쪽 막대를 표시하도록 했습니다
- session transcript 크기를 개선했습니다: 빈 hook 항목을 건너뛰고 저장된 pre-edit 파일 복사본을 제한합니다
- transcript 정확도를 개선했습니다: per-block 항목이 이제 streaming placeholder 대신 최종 token 사용량을 전달합니다
- Bash tool OTEL tracing을 개선했습니다: tracing이 활성화되면 subprocesses가 이제 W3C `TRACEPARENT` env var을 상속합니다
- `/claude-api` 스킬을 업데이트하여 Claude API와 함께 Managed Agents를 포함하도록 했습니다
