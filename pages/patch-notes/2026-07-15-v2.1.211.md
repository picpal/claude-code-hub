> 원문: [Claude Code v2.1.211 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.211)

## 변경사항

- `--forward-subagent-text` flag 및 `CLAUDE_CODE_FORWARD_SUBAGENT_TEXT` 환경 변수를 추가하여 stream-json 출력에 subagent 텍스트 및 thinking을 포함하도록 개선
- 채팅 채널에 relayed되는 permission preview가 bidirectional-override, zero-width, 및 look-alike quote 문자를 중립화하지 않아 tool 입력이 승인 메시지를 시각적으로 변경할 수 없도록 수정
- auto mode가 sandboxed되지 않은 Bash에 대한 PreToolUse hook의 `ask` 결정을 override하는 문제를 수정 — 이제 hook `ask`는 프롬프트에서 결정을 하한값으로 설정
- 많은 세션이 하나의 credential store를 공유할 때 수면에서 깨어난 후 parallel Claude Code 세션이 동시에 모두 로그아웃되는 문제를 수정
- plugin MCP server가 유휴 웹 세션이 깨어난 후 재연결되지 않아 다음 메시지까지 MCP 호출이 실패하는 문제를 수정
- Claude Code on Vertex 및 Bedrock이 시작 시 기본 Opus 모델을 시도하고 모델이 명시적으로 구성될 때 spurious fallback notice를 인쇄하는 문제를 수정
- 명시적 모델 override로 spawned된 subagent가 resumed되거나 follow-up 메시지를 받을 때 부모의 모델로 revert되는 문제를 수정
- 프로젝트 설정을 제외하도록 설정된 경우에도 nested `.claude/rules/*.md` 파일이 로드되는 문제를 수정
- 파일 업로드 검증 수정: DOS device suffix(`.prn`)로 끝나거나 trailing dot이 있는 파일명은 이제 허용되며, 여러 hard link를 가진 파일은 거부됨
- Chrome에서 Claude로 파일 업로드가 remote 및 CLI 세션에서 작동하지 않는 문제를 수정
- input을 "?"로 남기는 edit이 silently swallowed되고 shortcuts panel을 toggle하는 문제를 수정
- Claude in Chrome extension이 활성화되었지만 Chrome이 실행 중이지 않을 때 startup hang이 발생하는 문제를 수정
- async content(Settings tabs, Stats, diff view 및 기타 loading state)를 나타내는 데 300ms 지연이 발생하는 문제를 수정
- 방금 중지된 background 세션을 agents view에서 다시 열 때 동일한 session id로 blank conversation을 시작하는 문제를 수정
- `/loop`이 한 번 사용한 후 `/resume`에서 세션을 숨기는 문제를 수정
- screen reader 사용자가 `/terminal-setup` 또는 onboarding terminal setup 후 audible terminal bell을 잃는 문제를 수정
- LLM gateway auth(`ANTHROPIC_AUTH_TOKEN` + `ANTHROPIC_BASE_URL`)의 background job이 daemon이 respawn한 후 "Not logged in"으로 돌아오는 문제를 수정
- `claude agents` job이 git이 더 이상 worktree를 인식하지 않을 때 영구적으로 삭제 불가능해지는 문제를 수정 — 이제 row가 삭제가 거부된 이유를 표시하며 silently reappearing하지 않음
- `/clear`이 session cost counter를 reset하지 않는 문제를 수정 — 이제 `/clear` 후 statusline의 cost는 $0에서 시작
- Claude in Chrome 설정 페이지가 Windows의 브라우저에서 열리지 않는 문제를 수정
- Windows의 headless print-mode 세션이 stdin이 readable하지 않을 때 crash되거나 silently exit되는 문제를 수정
- background 세션 제목이 agents view에 표시될 때 프롬프트에 link가 포함되어 있으면 naming 모델의 refusal 텍스트를 표시하는 문제를 수정
- 사용자가 background agent를 kill했을 때 auto-respawning되고 revived agent가 old 세션의 stale 프롬프트를 다시 실행하는 문제를 수정
- schedule이 없는 routine이 year 1에서 next run time을 보고하는 문제를 수정
- Windows에서 synced skill/plugin directory naming을 강화했으며 `/clear` 후 CCR web fetch/search proxy가 계속 작동하도록 유지
- terminal layout 및 rendering performance 개선
- background agent 결과 보고 개선 — 이제 Claude가 still-running agent의 상태를 보고하고 결과를 fabricate하는 대신 real completion을 대기
- memory index over-limit warning을 개선하여 frontmatter 및 HTML comment를 제외한 loaded content만 측정
- integer 환경 변수(timeout, token budget, retry count)를 `1e6` 및 `64_000` 같은 scientific notation 및 digit-separator spelling을 수용하도록 업데이트
- 문서 link를 current docs site로 업데이트
- "always allow" permission rule을 저장소 root에 저장하도록 변경하여 git worktree에서 승인된 approval이 세션 및 worktree 전체에서 지속되도록 개선
- `/usage-credits`를 organization admin에게 요청을 보내기 전에 confirmation을 요청하도록 변경
- Vim mode `s` 및 `S`(substitute char/line)을 NORMAL mode에서 작동하도록 변경하여 vim 동작과 일치
- [VSCode] Remote Control banner를 업데이트하여 동작을 설명
- Claude in Chrome: file-upload path validation 강화
- Claude in Chrome: screenshot action의 `save_to_disk`가 이제 이미지를 disk에 쓰고 path를 반환하므로, 이전에는 아무 작업도 수행하지 않음
- Bedrock, Vertex, Mantle 및 Foundry의 prompt-caching regression을 수정하여 모든 요청에서 trailing system context block을 fresh input token으로 청구하지 않도록 개선
