## 변경사항

- `ENABLE_PROMPT_CACHING_1H` env var를 추가하여 API key, Bedrock, Vertex, Foundry에서 1시간 prompt cache TTL로 선택할 수 있도록 하였습니다(`ENABLE_PROMPT_CACHING_1H_BEDROCK`는 폐기되었으나 여전히 적용됨). 또한 5분 TTL을 강제하는 `FORCE_PROMPT_CACHING_5M`을 추가했습니다.
- session으로 돌아올 때 context를 제공하는 recap 기능을 추가했습니다. /config에서 설정할 수 있으며 /recap으로 수동으로 호출할 수 있습니다. telemetry가 비활성화된 경우 `CLAUDE_CODE_ENABLE_AWAY_SUMMARY`로 강제할 수 있습니다.
- Model이 이제 Skill tool을 통해 `/init`, `/review`, `/security-review`와 같은 built-in slash command를 발견하고 호출할 수 있습니다.
- `/undo`는 이제 `/rewind`의 alias입니다.
- 다음 응답이 전체 history를 cache되지 않은 상태로 다시 읽으므로, `/model`을 개선하여 conversation 중간에 model을 전환하기 전에 경고를 표시합니다.
- `/resume` picker를 개선하여 현재 directory의 session으로 기본 설정되도록 하였습니다. `Ctrl+A`를 눌러 모든 project를 표시할 수 있습니다.
- 개선된 error message: server rate limit이 이제 plan usage limit과 구분되며, 5xx/529 error는 status.claude.com의 링크를 표시합니다. 알 수 없는 slash command는 가장 유사한 명령어를 제안합니다.
- file read, edit, syntax highlighting의 memory footprint을 감소시켰습니다. 이제 language grammar를 on demand로 로드합니다.
- detailed transcript를 볼 때(`Ctrl+O`) "verbose" indicator를 추가했습니다.
- `DISABLE_PROMPT_CACHING*` environment variable로 prompt caching이 비활성화될 때 시작 시 경고를 추가했습니다.
- `/login` code prompt에서 paste가 작동하지 않는 문제를 수정했습니다(2.1.105에서의 regression).
- `DISABLE_TELEMETRY`를 설정한 subscriber가 1시간 대신 5분 prompt cache TTL로 fallback하는 문제를 수정했습니다.
- safety classifier의 transcript가 context window를 초과했을 때 Agent tool이 auto mode에서 permission을 요청하는 문제를 수정했습니다.
- `CLAUDE_ENV_FILE`(예: `~/.zprofile`)이 `#` comment line으로 끝날 때 Bash tool이 output을 생성하지 않는 문제를 수정했습니다.
- `claude --resume <session-id>`가 `/rename`을 통해 설정한 session의 custom name과 color를 잃는 문제를 수정했습니다.
- 첫 번째 message가 짧은 greeting일 때 session title이 placeholder example text를 표시하는 문제를 수정했습니다.
- `--teleport` 이후 prompt input에서 terminal escape code가 garbage text로 나타나는 문제를 수정했습니다.
- `/feedback` retry: failure 후 먼저 description을 편집하지 않고 Enter를 눌러 resubmit하는 기능이 작동하지 않는 문제를 수정했습니다.
- `--teleport`와 `--resume <id>` precondition error(예: dirty git tree, session not found)가 error message를 표시하지 않고 silently exit하는 문제를 수정했습니다.
- web UI에서 설정한 Remote Control session title이 세 번째 message 이후 auto-generated title로 덮어씌워지는 문제를 수정했습니다.
- `--resume`이 self-referencing message를 포함한 transcript를 truncate하는 문제를 수정했습니다.
- transcript write failure(예: disk full)가 logged되지 않고 silently drop되는 문제를 수정했습니다.
- `language` setting이 구성되었을 때 response에서 diacritical mark(accents, umlauts, cedillas)가 drop되는 문제를 수정했습니다.
- 처음 설치한 project와 다른 project에서 실행할 때 policy-managed plugin이 auto-update되지 않는 문제를 수정했습니다.
