> 원문: [Claude Code v2.1.157 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.157)

## 변경사항

- `.claude/skills` 디렉터리의 플러그인이 이제 자동으로 로드되므로 marketplace가 필요하지 않습니다
- `claude plugin init <name>`을 추가하여 `.claude/skills`에서 새 플러그인을 스캐폴딩할 수 있습니다
- `/plugin` 인수에 대한 자동 완성을 추가했습니다: 하위 명령, 설치된 플러그인 이름 및 알려진 marketplace의 플러그인
- `claude agents`: `settings.json`의 `agent` 필드는 이제 dispatched 세션에서 존중되며, `--agent <name>`으로 재정의할 수 있습니다
- `EnterWorktree`는 이제 세션 중에 Claude 관리 worktree 간을 전환할 수 있습니다
- `tool_decision` telemetry 이벤트는 `OTEL_LOG_TOOL_DETAILS=1`일 때 이제 `tool_parameters`(bash 명령, MCP/skill 이름)를 포함합니다
- Claude가 관리하는 Worktree는 이제 agent가 완료될 때 잠금 해제 상태로 남겨져 `git worktree remove`/`prune`이 정리할 수 있습니다
- 붙여넣기, MCP 또는 대화상자를 통해 첨부된 처리 불가능한 이미지(0바이트, 손상됨)가 요청을 충돌시키는 대신 텍스트 자리 표시자가 되도록 수정했습니다
- desktop app, IDE extensions 또는 SDK를 사용할 때 auto 및 bypass-permissions 모드에서 sandbox network 권한 프롬프트가 표시되도록 수정했습니다
- `claude agents` completed 세션이 유휴 subagent가 여전히 parked 상태이거나 backgrounded shell을 누출했을 때 retire되지 않도록 수정했습니다
- `claude agents`에서 Esc를 누르면 느린 "opening…"을 취소하지 않아 목록이 반응하지 않는 상태로 남겨지도록 수정했습니다
- `.claude/worktrees/` 아래의 background agent worktree가 30일 job 보존 sweep 후 고아 상태가 되도록 수정했습니다
- sleep/wake 후 재연결된 background 세션이 model에 올바른 날짜를 알려주지 않도록 수정했습니다
- `claude agents`에서 copy-on-select이 `set-clipboard on`으로 tmux 내의 시스템 clipboard에 도달하지 않도록 수정했습니다(2.1.153 회귀)
- `--resume`이 이전 Claude Code 프로세스가 종료될 때 실행 중이던 background subagent를 보고하지 않도록 수정했습니다
- `--resume` 세션 선택기가 fullscreen 모드에서 종료할 때 터미널에 내용을 남겨두도록 수정했습니다
- `--worktree` 및 `--worktree --tmux`가 현재 linked worktree 대신 canonical repo root로 돌아가도록 수정했습니다
- `/model` 선택기가 선택된 모델이 이미 해당 family의 최신 버전일 때 잘못된 "Newer version available" 힌트를 표시하도록 수정했습니다; pinned-model 행은 이제 raw ID 대신 model의 설명을 표시합니다
- fullscreen 모드에서 진행 중인 메시지 텍스트에 리터럴 markdown 마커(backtick, asterisk)가 나타나도록 수정했습니다
- 시작 시 managed-settings 보안 대화상자를 승인한 후 터미널이 frozen되도록 수정했습니다
- terminal UI가 다시 그려진 후 scrollback에 드물게 중복 줄이 나타나도록 수정했습니다
- VS Code, Cursor 및 Windsurf integrated terminal에서 마우스 오른쪽 버튼 붙여넣기가 clipboard를 복제하도록 수정했습니다
- WSL: 이미지 붙여넣기(`alt+v` keybinding), Windows 11 screenshot 붙여넣기를 수정했으며, Windows Explorer에서 이미지 드래그 지원을 추가했습니다
- 중복된 메시지 렌더링 재계산을 제거하여 길고 resumed된 대화의 성능을 개선했습니다
- `/terminal-setup`은 이제 VS Code/Cursor/Windsurf integrated terminal에서 GPU 가속을 비활성화하여 garbled-text 렌더링을 방지합니다
- Feature of the Week 크레딧 청구 상태는 이제 prompt 위의 줄 대신 상태 영역의 notification으로 나타납니다
- `claude agents`: dispatch 입력의 slash-command 자동 완성은 이제 substring을 일치시킵니다
- "bash commands will be sandboxed" 시작 배너를 제거했습니다 — sandbox 상태는 여전히 `/status`에 표시되고 명령이 차단될 때 나타납니다
- "/ide for …" 시작 힌트 toast를 제거했습니다
- [IDE] background subagent가 실행 중일 때 Stop을 클릭해도 실제로 중지되지 않도록 수정했습니다
- [VSCode] Opus 4.8에서 fast mode 지표가 나타나지 않도록 수정했습니다
- workflow trigger 키워드 바로 다음에 backspace를 누르면 이제 문자를 삭제하는 대신 workflow 요청을 닫습니다(alt+w와 동일)
- `/config`에 "Workflow keyword trigger" 설정을 추가하여 prompt의 "workflow"라는 단어가 동적 workflow를 trigger하지 않도록 합니다
