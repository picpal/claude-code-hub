## 변경사항

- 셸 시작 파일(`.zshenv`, `.zlogin`, `.bash_login`)과 `~/.config/git/`에 쓰기 전에 프롬프트를 추가했습니다. 이전에는 의도하지 않은 명령 실행으로 이어질 수 있었습니다
- `acceptEdits` 모드는 이제 코드 실행 권한을 부여하는 빌드 도구 설정 파일(`.npmrc`, `.yarnrc*`, `bunfig.toml`, `.bazelrc`, `.pre-commit-config.yaml`, `.devcontainer/` 등)에 쓰기 전에 프롬프트를 표시합니다
- Edit는 더 이상 `grep`으로 파일을 본 후 별도의 Read가 필요하지 않습니다. 단일 파일 `grep`/`egrep`/`fgrep` 명령어는 이제 편집 전 읽기 확인을 만족합니다
- Windows의 WSL에서 선택 시 복사가 클립보드에 쓰지 못하는 문제를 수정했습니다. 이제 MobaXterm과 같은 터미널에서 지원하지 않는 OSC 52 대신 PowerShell interop을 사용합니다
- `claude agents`에서 완료된 세션을 복원할 때 채팅 기록이 손실되고 원래 프롬프트가 재실행되는 문제를 수정했습니다
- 밤새 retire 후 재연결된 백그라운드 세션이 대화를 잃고 원래 프롬프트를 재실행하는 문제를 수정했습니다
- 로드된 머신에서 백그라운드 daemon이 콜드 스타트할 때 `claude --bg`가 "socket missing" 오류로 실패하는 경우를 수정했습니다
- `claude rm` 후 백그라운드 daemon이 종료될 때까지 백그라운드 세션이 시작된 디렉터리를 삭제할 수 없는 Windows의 문제를 수정했습니다
- 작업을 재개한 백그라운드 agent가 agents 목록에서 완료됨으로 표시되는 문제를 수정했습니다
- auto-updater가 매번 종료 시마다 다시 확인하여 세션 목록으로 돌아갈 때 `claude agents`가 몇 초 동안 멈추는 문제를 수정했습니다
- 호스트가 높은 CPU 부하를 받을 때 백그라운드 세션에 연결되거나 agent 보기에 있을 때 Esc, 화살표 키 및 입력이 응답하지 않게 되는 Windows의 문제를 수정했습니다
- 백그라운드 agent가 이를 지원하지 않는 터미널(Apple Terminal, tmux)에 terminal sync-output 마커를 내보내 실행 중인 agent에 들어갈 때 렌더링 아티팩트가 발생하는 문제를 수정했습니다
- agents 목록에서 세션을 열자마자 마우스 휠이 스크립트 대신 프롬프트 기록을 스크롤하는 문제를 수정했습니다
- `claude agents` 보기에서 CJK IME composition이 입력 캐럿이 아닌 화면 왼쪽 하단에 나타나는 문제를 수정했습니다
- hyperlink를 지원하는 Windows 터미널에서 유효한 `file:///C:/...` 링크가 손상된 경로로 다시 쓰여지는 문제를 수정했습니다
- 프로젝트 디렉터리 또는 브랜치 이름에 ASCII가 아닌 문자 또는 특수 문자가 포함되어 있을 때 voice mode가 연결되지 못하는 문제를 수정했습니다
- 서드파티 제공자(Bedrock/Vertex/Foundry)에서 auto mode 사용 불가 메시지가 모델을 잘못 비난하는 대신 `CLAUDE_CODE_ENABLE_AUTO_MODE` opt-in을 가리키도록 수정했습니다
- `/effort ultracode`가 모델이 xhigh를 실행할 수 없을 때 dynamic workflows 설정을 잘못 비난하는 문제를 수정했습니다. ultracode는 이제 이를 지원하지 않는 모델에는 더 이상 제공되지 않습니다
- CLI 플래그가 적용되지 않는 SDK 또는 기타 호스트를 통해 실행할 때 모델을 찾을 수 없는 오류가 `--model`을 제안하는 문제를 수정했습니다
- brief mode를 끈 상태로 brief mode 세션을 재개할 때 Claude의 과거 응답이 scrollback에서 사라지는 문제를 수정했습니다
- vim mode `p`가 레지스터를 `v$`로 yanked할 때 커서 위치가 아닌 아래 줄에 붙여넣기하는 문제를 수정했습니다
- `claude agents`에서 최근에 비활성화된 백그라운드 agent 세션을 열 때의 성능을 개선했습니다
- 루틴 작업에 대한 추론을 줄여 auto mode classifier 레이턴시를 개선했으므로 "could not evaluate this action" 블록의 가능성이 낮아졌습니다
- 백그라운드 세션 teardown(`claude rm`/`stop`, idle reap)을 개선하여 SIGKILL 이전에 실행 중인 셸 서브프로세스에 SIGTERM을 보내므로 cleanup 핸들러가 실행됩니다
- `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE`를 제거했습니다. 환경 변수는 이제 no-op입니다
- 시작 시 JetBrains 플러그인 설치 제안을 제거했습니다
- dynamic-workflow 트리거 키워드의 이름을 `workflow`에서 `ultracode`로 변경했습니다. 이제 "workflow"라는 단어는 더 이상 실행을 트리거하지 않습니다. 자신의 말로 요청하는 것은 여전히 작동합니다. 트리거 키워드는 프롬프트 입력에서 보라색으로 강조 표시됩니다
