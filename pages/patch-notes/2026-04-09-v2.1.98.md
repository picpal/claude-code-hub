> 원문: [Claude Code v2.1.98 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.98)

## 변경사항

- 로그인 화면에서 "3rd-party platform"을 선택할 때 접근 가능한 대화형 Google Vertex AI 설정 마법사를 추가했습니다. GCP 인증, 프로젝트 및 지역 구성, 자격증명 검증 및 모델 고정을 안내합니다.
- `CLAUDE_CODE_PERFORCE_MODE` env var를 추가했습니다. 설정되면 Edit/Write/NotebookEdit이 읽기 전용 파일에서 실패하고 자동으로 덮어쓰는 대신 `p4 edit` 힌트를 표시합니다.
- 백그라운드 스크립트의 스트리밍 이벤트를 위한 Monitor 도구를 추가했습니다.
- `CLAUDE_CODE_SUBPROCESS_ENV_SCRUB`이 설정되어 있을 때 Linux에서 PID namespace 격리를 사용한 subprocess 샌드박싱과 세션당 스크립트 호출을 제한하는 `CLAUDE_CODE_SCRIPT_CAPS` env var를 추가했습니다.
- 교차 사용자 prompt 캐싱을 개선하기 위해 print 모드에 `--exclude-dynamic-system-prompt-sections` 플래그를 추가했습니다.
- 현재 디렉토리가 연결된 git worktree 내부에 있을 때마다 설정되는 `workspace.git_worktree`를 상태 라인 JSON 입력에 추가했습니다.
- OTEL 트레이싱이 활성화되어 있을 때 Bash 도구 subprocess에 W3C `TRACEPARENT` env var를 추가했으므로 자식 프로세스 span이 Claude Code의 트레이스 트리에 올바르게 상위로 설정됩니다.
- LSP: Claude Code는 이제 initialize 요청의 `clientInfo`를 통해 자신을 언어 서버에 식별합니다.
- 백슬래시로 이스케이프된 플래그가 읽기 전용으로 자동 허용되고 임의의 코드 실행으로 이어질 수 있는 Bash 도구 권한 우회를 수정했습니다.
- 복합 Bash 명령이 auto 및 bypass-permissions 모드에서 강제 권한 프롬프트, 안전 검사 및 명시적 ask 규칙을 우회하는 것을 수정했습니다.
- env-var 접두사가 있는 읽기 전용 명령이 변수가 알려진 안전 변수(`LANG`, `TZ`, `NO_COLOR` 등)가 아닌 한 프롬프트되지 않는 것을 수정했습니다.
- `/dev/tcp/...` 또는 `/dev/udp/...`로의 리다이렉트가 자동 허용 대신 프롬프트되지 않는 것을 수정했습니다.
- 정지된 스트리밍 응답이 비스트리밍 모드로 폴백되는 대신 타임아웃되는 것을 수정했습니다.
- 서버가 작은 `Retry-After`를 반환할 때 429 재시도가 ~13초 내에 모든 시도를 소진하는 것을 수정했습니다. 지수 백오프가 이제 최소값으로 적용됩니다.
- MCP OAuth `oauth.authServerMetadataUrl` 구성 재정의가 재시작 후 토큰 새로고침에서 인정되지 않는 것을 수정했습니다. ADFS 및 유사 IdP에 영향을 미칩니다.
- kitty 키보드 프로토콜이 활성화되어 있을 때 xterm 및 VS Code 통합 터미널에서 대문자가 소문자로 삭제되는 것을 수정했습니다.
- macOS 텍스트 대체가 치환을 삽입하는 대신 트리거 단어를 삭제하는 것을 수정했습니다.
- `--dangerously-skip-permissions`가 Bash를 통해 보호된 경로에 대한 쓰기를 승인한 후 수동으로 accept-edits 모드로 다운그레이드되는 것을 수정했습니다.
- 관리자가 제거한 후 프로세스 재시작까지 관리형 설정 허용 규칙이 활성 상태로 유지되는 것을 수정했습니다.
- `permissions.additionalDirectories` 변경이 중간 세션에 적용되지 않는 것을 수정했습니다. 제거된 디렉토리는 즉시 액세스를 잃고 추가된 디렉토리는 재시작 없이 작동합니다.
- `additionalDirectories`에서 디렉토리를 제거하면 `--add-dir`을 통해 전달된 동일한 디렉토리에 대한 액세스가 취소되는 것을 수정했습니다.
- `Bash(cmd:*)` 및 `Bash(git commit *)` 와일드카드 권한 규칙이 추가 공백이나 탭이 있는 명령과 일치하지 못하는 것을 수정했습니다.
- `cd`와 다른 세그먼트를 혼합하는 파이프된 명령에 대해 `Bash(...)` 거부 규칙이 프롬프트로 다운그레이드되는 것을 수정했습니다.
- `cut -d /`, `paste -d /`, `column -s /`, `awk '{print $1}' file` 및 `%`를 포함하는 파일명에 대한 거짓 Bash 권한 프롬프트를 수정했습니다.
- JavaScript 프로토타입 속성과 일치하는 이름의 권한 규칙(예: `toString`)이 `settings.json`을 자동으로 무시하게 하는 것을 수정했습니다.
- 에이전트 팀 멤버가 `--dangerously-skip-permissions`를 사용할 때 리더의 권한 모드를 상속하지 않는 것을 수정했습니다.
- fullscreen 모드에서 MCP 도구 결과 위에 마우스를 올릴 때 발생하는 충돌을 수정했습니다.
- fullscreen 모드에서 래핑된 URL을 복사할 때 줄 바꿈에 공백이 삽입되는 것을 수정했습니다.
- `--resume`에서 편집된 파일이 10KB보다 클 때 파일 편집 diff가 UI에서 사라지는 것을 수정했습니다.
- 여러 `/resume` picker 문제를 수정했습니다: `--resume <name>`이 편집 불가능하게 열기, 필터 재로드가 검색 상태 초기화, 빈 목록이 화살표 키 삼키기, 교차 프로젝트 낡음 및 임시 작업 상태 텍스트가 대화 요약 대체.
- `/export`가 절대 경로 및 `~`를 인정하지 않고 사용자 제공 확장을 무음으로 `.txt`로 다시 작성하는 것을 수정했습니다.
- `/effort max`가 알 수 없거나 향후 model ID에 대해 거부되는 것을 수정했습니다.
- plugin의 frontmatter `name`이 YAML boolean 키워드일 때 slash command picker가 중단되는 것을 수정했습니다.
- 메시지 remount 후 rate-limit 상향식 텍스트가 숨겨지는 것을 수정했습니다.
- `_meta["anthropic/maxResultSizeChars"]`가 있는 MCP 도구가 token 기반 persist 레이어를 우회하지 못하는 것을 수정했습니다.
- 이전 transcript이 여전히 처리 중일 때 push-to-talk 키를 다시 누를 때 음성 모드가 수십 개의 공백 문자를 입력으로 유출하는 것을 수정했습니다.
- `DISABLE_AUTOUPDATER`가 npm 기반 설치에서 npm registry 버전 검사 및 symlink 수정을 완전히 억제하지 못하는 것을 수정했습니다.
- Remote Control 권한 처리기 항목이 세션 수명 동안 보존되는 메모리 누수를 수정했습니다.
- 오류로 실패한 백그라운드 subagent가 부분 진행을 부모 agent에 보고하지 않는 것을 수정했습니다.
- prompt-type Stop/SubagentStop 훅이 긴 세션에서 실패하고 훅 평가자 API 오류가 실제 메시지 대신 "JSON validation failed"를 표시하는 것을 수정했습니다.
- 피드백 설문조사 렌더링이 해제될 때 발생하는 것을 수정했습니다.
- Bash `grep -f FILE` / `rg -f FILE`이 작업 디렉토리 외부의 패턴 파일을 읽을 때 프롬프트되지 않는 것을 수정했습니다.
- 오래된 subagent worktree 정리가 추적되지 않은 파일을 포함하는 worktree를 제거하는 것을 수정했습니다.
- `sandbox.network.allowMachLookup`이 macOS에서 적용되지 않는 것을 수정했습니다.
- `/resume` 필터 힌트 레이블을 개선했으며 필터 표시기에 프로젝트/worktree/branch 이름을 추가했습니다.
- 좁은 터미널 너비에서 줄 바꿈 대신 mode-indicator 행에 머물러 있도록 footer 표시기(Focus, notifications)를 개선했습니다.
- 탭 레이아웃으로 `/agents`를 개선했습니다. Running 탭은 live subagent를 표시하고 Library 탭은 Run agent 및 View running instance 작업을 추가합니다.
- 재시작 없이 plugin 제공 skill을 선택하도록 `/reload-plugins`를 개선했습니다.
- 안전한 env var 또는 프로세스 래퍼가 접두어로 붙은 filesystem 명령을 자동 승인하도록 Accept Edits 모드를 개선했습니다.
- Vim 모드를 개선했습니다: NORMAL 모드의 `j`/`k`는 이제 history를 탐색하고 입력 경계의 footer pill을 선택합니다.
- hook 오류를 트랜스크립트에 개선하여 `--debug` 없이 자동 진단을 위해 stderr의 첫 번째 줄을 포함합니다.
- OTEL 트레이싱을 개선했습니다: interaction span은 이제 concurrent SDK 호출에서 전체 턴을 올바르게 래핑하고 headless 턴은 턴당 span을 종료합니다.
- 스트리밍 자리표시자 대신 최종 토큰 사용을 반영하도록 트랜스크립트 항목을 개선했습니다.
- Managed Agents와 함께 Claude API를 포괄하도록 `/claude-api` skill을 업데이트했습니다.
- [VSCode] `CLAUDE_CODE_GIT_BASH_PATH`가 설정되거나 Git이 기본 위치에 설치되어 있을 때 Windows에서 거짓 양성 "requires git-bash" 오류를 수정했습니다.
- `CLAUDE_CODE_MAX_CONTEXT_TOKENS`을 수정하여 `DISABLE_COMPACT`가 설정되어 있을 때 이를 인정합니다.
- `DISABLE_COMPACT`가 설정되어 있을 때 `/compact` 힌트를 삭제했습니다.
