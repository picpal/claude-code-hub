> 원문: [Claude Code v2.1.118 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.118)

## 변경사항

- Vim visual mode (`v`) 및 visual-line mode (`V`)를 추가했으며, selection, operators, visual feedback를 지원합니다
- `/cost`와 `/stats`를 `/usage`로 통합했습니다 — 두 항목 모두 관련 탭을 열기 위한 typing shortcuts으로 남아 있습니다
- `/theme`에서 명명된 custom themes을 만들고 전환하거나, `~/.claude/themes/`의 JSON 파일을 직접 편집할 수 있습니다. 플러그인은 또한 `themes/` 디렉토리를 통해 themes을 제공할 수 있습니다
- Hooks는 이제 `type: "mcp_tool"`을 통해 MCP tools을 직접 호출할 수 있습니다
- `DISABLE_UPDATES` env var를 추가했으며, 수동 `claude update`를 포함한 모든 update paths를 완전히 차단합니다 — `DISABLE_AUTOUPDATER`보다 더 엄격합니다
- Windows의 WSL은 이제 `wslInheritsWindowsSettings` policy key를 통해 Windows 측의 managed settings을 상속할 수 있습니다
- Auto mode: `autoMode.allow`, `autoMode.soft_deny` 또는 `autoMode.environment`에 `"$defaults"`를 포함하여 built-in list를 대체하는 대신 custom rules을 함께 추가합니다
- Auto mode opt-in prompt에 "다시 묻지 않기" 옵션을 추가했습니다
- Plugin release git tags을 버전 검증과 함께 만들기 위해 `claude plugin tag`를 추가했습니다
- `--continue`/`--resume`는 이제 `/add-dir`을 통해 현재 디렉토리를 추가한 sessions을 찾습니다
- Remote Control이 연결되었을 때 `/color`는 이제 session accent color을 claude.ai/code와 동기화합니다
- `/model` picker는 이제 custom `ANTHROPIC_BASE_URL` gateway를 사용할 때 `ANTHROPIC_DEFAULT_*_MODEL_NAME`/`_DESCRIPTION` overrides를 준수합니다
- Auto-update가 다른 플러그인의 version constraint로 인해 플러그인을 건너뛸 때, skip은 이제 `/doctor`와 `/plugin` Errors tab에 나타납니다
- `/mcp` menu가 `headersHelper`로 구성된 servers의 OAuth Authenticate/Re-authenticate actions을 숨기는 문제를 수정했으며, custom headers을 가진 HTTP/SSE MCP servers가 transient 401 이후에 "needs authentication" 상태에서 stuck되는 문제를 수정했습니다
- MCP servers의 OAuth token response가 `expires_in`을 생략할 때 매시간 re-authentication이 필요한 문제를 수정했습니다
- MCP step-up authorization이 server의 `insufficient_scope` 403에서 이름을 지정한 scope을 현재 token이 이미 가지고 있을 때 silent하게 새로고침되지 않고 re-consent를 prompt하도록 수정했습니다
- MCP server의 OAuth flow가 timeout되거나 cancelled될 때 unhandled promise rejection을 수정했습니다
- Contention 상태에서 MCP OAuth refresh가 cross-process lock 없이 진행되는 문제를 수정했습니다
- macOS keychain race에서 concurrent MCP token refresh가 freshly-refreshed OAuth token을 overwrite하여 unexpected "Please run /login" prompts을 유발하는 문제를 수정했습니다
- Server가 local expiry time 이전에 token을 revoke할 때 OAuth token refresh가 실패하는 문제를 수정했습니다
- Linux/Windows에서 credential save crash가 `~/.claude/.credentials.json`을 corrupting하는 문제를 수정했습니다
- `/login`이 `CLAUDE_CODE_OAUTH_TOKEN`으로 시작된 session에 영향을 주지 않는 문제를 수정했습니다 — env token이 이제 cleared되므로 disk credentials이 작동합니다
- "new messages" scroll pill 및 `/plugin` badges에서 읽을 수 없는 텍스트 문제를 수정했습니다
- `--dangerously-skip-permissions`로 실행할 때 plan acceptance dialog가 "bypass permissions" 대신 "auto mode"를 제공하는 문제를 수정했습니다
- Agent-type hooks가 `Stop` 또는 `SubagentStop` 이외의 events에 구성될 때 "Messages are required for agent hooks" 오류가 발생하는 문제를 수정했습니다
- `prompt` hooks이 agent-hook verifier subagent에 의해 만들어진 tool calls에서 re-firing되는 문제를 수정했습니다
- `/fork`가 fork 당 전체 parent conversation을 disk에 쓰는 문제를 수정했습니다 — 이제 pointer를 쓰고 read 시 hydrates합니다
- Alt+K / Alt+X / Alt+^ / Alt+_가 keyboard input을 freezing하는 문제를 수정했습니다
- Remote session에 연결할 때 local `model` setting이 `~/.claude/settings.json`에서 overwrite되는 문제를 수정했습니다
- Typeahead가 `/`로 시작하는 file paths를 paste할 때 "No commands match" 오류를 표시하는 문제를 수정했습니다
- 이미 설치된 플러그인에서 `plugin install`이 dependency installed at the wrong version을 다시 resolve하지 않는 문제를 수정했습니다
- Invalid paths 또는 fd exhaustion으로 인한 file watcher의 unhandled errors를 수정했습니다
- Remote Control sessions이 JWT refresh 중 transient CCR initialization blips에서 archived되는 문제를 수정했습니다
- `SendMessage`를 통해 resumed된 subagents가 spawn될 때의 explicit `cwd`를 restore하지 않는 문제를 수정했습니다
