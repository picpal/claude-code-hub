> 원문: [Claude Code v2.1.214 Release Notes](https://github.com/anthropics/claude-code/releases/tag/v2.1.214)

## 변경사항

- `Edit(src/**)`과 같은 단일 세그먼트 `dir/**` allow 규칙이 트리의 모든 곳에 있는 `<cwd>/dir` 디렉터리가 아닌 중첩 `dir/` 디렉터리에 대한 쓰기를 자동 승인하는 문제를 수정했습니다.
- Windows PowerShell 5.1 세션에서 실행되는 명령에 영향을 미치는 permission-check bypass를 수정했습니다.
- permission analyzer가 bash와 다르게 구문 분석하는 file-descriptor redirect 형식에서 Bash permission 확인이 실패하도록 수정했습니다.
- 10,000자 이상의 매우 긴 명령에 대해 자동으로 실행되지 않고 항상 프롬프트를 표시하도록 Bash permission 확인을 수정했습니다.
- `[[ ]]` 비교에서 zsh 변수 subscripts 및 modifiers를 정적 텍스트로 잘못 판단하는 Bash permission 확인을 수정했습니다.
- 안전하지 않은 옵션, command substitutions 또는 backslash 경로를 실행할 수 있는 특정 `help` 및 `man` 명령을 자동 승인하지 않도록 Bash permission 확인을 수정했습니다.
- 로컬 confirmation dialog 이전에 진행될 수 있는 원격 세션의 permission 프롬프트를 수정했습니다.
- EndConversation tool을 추가했습니다: Claude는 claude.ai에서 2025년 이후로 가능했던 것처럼 극도로 학대적인 사용자 또는 jailbreak 시도가 있는 세션을 종료할 수 있습니다 — https://www.anthropic.com/research/end-subset-conversations 참조
- 이전에 무음이었던 장시간 실행되는 tool 호출에 대한 주기적 진행 상황 heartbeat을 추가했습니다.
- memory 파일 frontmatter에 ISO `modified` timestamp을 추가했습니다.
- message 수준의 상관관계 및 tool provenance를 위해 OpenTelemetry 로그 이벤트에 `message.uuid`, `client_request_id` 및 `tool_source` 속성을 추가했습니다.
- OpenTelemetry 콘텐츠 속성의 60 KB truncation 한계를 구성하기 위해 `CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH`를 추가했습니다.
- 사용자 지정 agent 행이 모델 및 effort를 렌더링할 수 있도록 `subagentStatusLine` payload에 reasoning effort를 추가했습니다.
- 이전에 프롬프트 없이 실행되던 daemon-redirect 플래그(`--url`, `--connection`, `--identity` 및 Podman의 원격 모드)를 전달하는 `docker` 명령(Podman `docker` shim 포함)에 대한 permission 프롬프트를 추가했습니다.
- GrowthBook feature가 null로 평가되는 경우의 충돌과 malformed flag payload가 캐시된 feature flags를 지울 수 있는 버그를 수정했습니다.
- `pkill -f` 패턴이 실수로 CLI 자신의 프로세스와 일치할 때 Bash tool이 Claude 세션을 종료하는 문제를 수정했습니다(Linux).
- `--settings`가 device 파일 또는 multi-GB 파일을 가리킬 때 무제한 메모리 증가를 수정했습니다. oversized(>2 MiB) settings 파일은 이제 명확한 오류 메시지와 함께 시작 시 실패합니다.
- Windows의 corporate proxies 뒤에서 "Socket is closed" 오류로 실패하는 streaming turns를 수정했습니다.
- 느린 SDK/pipeline consumers를 위한 종료 시 stream-json 출력 truncation을 수정했습니다. 종료 drain은 이제 flat 2s cap 대신 큐에 대기 중인 바이트로 조정됩니다.
- scheduled tasks가 신뢰할 수 없는 입력으로 자신의 구성된 prompt를 거부하는 문제를 수정했습니다 — fired prompt는 이제 세션의 할당된 task로 전달됩니다.
- child 프로세스가 standard input을 대기할 때 PowerShell tool 명령이 timeout까지 hanging되는 문제를 수정했습니다(Windows).
- PowerShell tool 아래에서 실행되는 Python 스크립트가 standard input에서 non-UTF-8 데이터를 읽을 때 UnicodeDecodeError로 충돌하는 문제를 수정했습니다(Windows).
- PowerShell tool을 통해 실행되는 Python 스크립트가 non-ASCII 출력에서 UnicodeEncodeError로 충돌하고, PowerShell 7 오류 메시지가 raw ANSI escape sequences를 포함하는 문제를 수정했습니다(Windows).
- PowerShell tool이 유효한 음수 응답을 반환할 때 `where.exe`, `fc.exe` 및 `diff.exe`를 오류로 보고하는 문제를 수정했습니다(Windows).
- PowerShell tool의 Windows PowerShell 5.1에서 `>` 및 `>>`이 다른 tools가 UTF-8로 읽을 수 없는 UTF-16LE 파일을 쓰는 문제를 수정했습니다.
- 종료 시 대체 daemon의 control socket을 삭제하는 displaced background daemon을 수정했습니다. 이는 다음 client가 healthy replacement daemon을 종료하게 만들었습니다.
- `←` 또는 `/background`로 parked되고 유휴 상태로 남겨진 background 세션이 background daemon 및 worker 프로세스를 무한정 활성 상태로 유지하는 문제를 수정했습니다.
- completed background 세션이 background 서비스가 유휴 상태가 된 후 `claude rm` 또는 agent view를 통해 제거할 수 없는 문제를 수정했습니다.
- non-git 폴더에서 dispatched된 background 세션을 agents view에서 삭제할 수 없는 문제를 수정했습니다.
- session store에 읽을 수 없는 폴더가 존재할 때 stopped background 세션을 다시 열 때 저장된 대화를 복원하지 못하는 문제를 수정했습니다.
- Remote Control이 명시적으로 활성화되지 않은 세션에 대해 Remote Control "session ready" push notification이 firing되는 문제를 수정했습니다.
- `/install-github-app` 및 `/mcp` 설정 메뉴가 agent-view 세션에서 차단되는 문제를 수정했습니다 — 이제 terminal이 없는 background 세션에서만 거부됩니다.
- `--settings` CLI 플래그를 통해 활성화된 plugins이 로드되지 않는 문제를 수정했습니다(v2.1.181 이후 regression).
- OAuth token이 회전된 후 장시간 실행되는 세션에서 feature flags가 stale되는 문제를 수정했습니다.
- merge base가 없는 repos에서 `/ultrareview`가 실행을 거부하는 문제를 수정했습니다 — 이제 모든 tracked 파일을 검토하도록 제안합니다.
- shell-config 경로가 디렉터리일 때 `claude update` 및 `claude doctor`가 무음으로 hanging되고 `/status` System diagnostics 섹션이 공백이 되는 문제를 수정했습니다.
- memory 파일이 저장될 때 memory frontmatter 값이 inline `#`에서 무음으로 truncate되는 문제를 수정했습니다.
- session cost 및 token telemetry가 여러 cumulative `message_delta` frames를 내보내는 streams에서 double-counting되는 문제를 수정했습니다.
- advisor가 생각하는 동안 표시되는 spurious "check your network" 경고를 수정했습니다.
- exit code 2가 있는 hooks가 hook의 stdout JSON이 schema validation에 실패할 때 문서화된 대로 blocking되지 않는 문제를 수정했습니다.
- turn의 async 컨텍스트 외부에서 emit된 OTel 로그 이벤트가 interaction span의 trace context를 누락하는 문제를 수정했습니다.
- prompts/resources 새로 고침 중 MCP transient 오류가 server의 slash 명령 및 resources를 clearing하는 문제를 수정했습니다.
- home 디렉터리의 `claude rc` workspace-trust 오류를 개선했습니다. trust가 저장되지 않는다고 명시하고 프로젝트 디렉터리에서 실행하도록 제안합니다.
- 단일 세그먼트 `dir/**` hook `if:` 조건을 `<cwd>/dir`만 일치하도록 변경했습니다. 임의 깊이 일치를 위해 `**/dir/**`을 작성하세요. `deny`/`ask` permission 규칙은 임의 깊이 일치를 유지합니다.
- `-m`/`--magic-file` 또는 `-f`/`--files-from`을 사용하는 `file` 명령을 read-only로 자동 허용되지 않고 permission을 요구하도록 변경했습니다.
- keep-alive connection pooling을 stale-connection 오류 후 비활성화하도록 변경했습니다. 재시도는 fresh socket을 엽니다.
- SessionStart hooks를 `"resume"` 대신 `"fork"` source로 보고하도록 변경했습니다(세션이 fork로 시작될 때).
