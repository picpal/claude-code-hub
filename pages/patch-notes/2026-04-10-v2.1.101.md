## 변경사항

- `/team-onboarding` 명령을 추가하여 로컬 Claude Code 사용 내역으로부터 팀원 온보딩 가이드를 생성할 수 있도록 함
- OS CA 인증서 저장소 신뢰를 기본값으로 추가하여 엔터프라이즈 TLS 프록시가 추가 설정 없이 작동하도록 함 (`CLAUDE_CODE_CERT_STORE=bundled`로 설정하면 번들된 CA만 사용)
- `/ultraplan` 및 기타 remote-session 기능이 웹 설정을 먼저 요구하는 대신 기본 cloud environment를 자동으로 생성하도록 함
- brief mode를 개선하여 Claude가 구조화된 메시지 대신 일반 텍스트로 응답할 때 한 번 재시도하도록 함
- focus mode를 개선하여 Claude가 최종 메시지만 보인다는 것을 알기 때문에 더 자체 포함된 요약을 작성하도록 함
- tool-not-available 오류를 개선하여 모델이 존재하지만 현재 문맥에서 사용할 수 없는 도구를 호출할 때 이유와 진행 방법을 설명하도록 함
- rate-limit 재시도 메시지를 개선하여 불투명한 초 단위 카운트다운 대신 어느 한계가 도달했는지, 언제 리셋되는지를 표시하도록 함
- refusal 오류 메시지를 개선하여 가능할 때 API 제공 설명을 포함하도록 함
- `claude -p --resume <name>`을 개선하여 `/rename` 또는 `--name`으로 설정된 세션 제목을 허용하도록 함
- settings 복원력을 개선하여 `settings.json`의 인식되지 않는 hook event name이 더 이상 전체 파일을 무시하지 않도록 함
- managed settings에 의해 강제 활성화된 플러그인의 plugin hooks를 개선하여 `allowManagedHooksOnly`가 설정될 때 실행되도록 함
- `/plugin` 및 `claude plugin update`를 개선하여 마켓플레이스를 새로 고칠 수 없을 때 조용히 오래된 버전을 보고하는 대신 경고를 표시하도록 함
- plan mode를 개선하여 사용자의 조직 또는 인증 설정이 웹에서 Claude Code에 도달할 수 없을 때 "Refine with Ultraplan" 옵션을 숨기도록 함
- beta tracing을 개선하여 `OTEL_LOG_USER_PROMPTS`, `OTEL_LOG_TOOL_DETAILS`, 및 `OTEL_LOG_TOOL_CONTENT`를 준수하도록 함; 민감한 span attributes는 명시적으로 동의하지 않는 한 더 이상 발출되지 않음
- SDK `query()`를 개선하여 consumers가 `for await`에서 `break`하거나 `await using`을 사용할 때 subprocess 및 temp 파일을 정리하도록 함
- POSIX `which` fallback의 명령 주입 취약점을 수정하여 LSP binary detection에서 사용됨
- virtual scroller에서 긴 세션이 메시지 목록의 수십 개 역사적 복사본을 보유하던 메모리 누수를 수정함
- `--resume`/`--continue`가 로더가 활성 대화 대신 dead-end branch에 고정되었을 때 큰 세션에서 대화 문맥을 잃던 문제를 수정함
- `--resume` chain recovery가 subagent 메시지가 main-chain write gap 근처에 도달했을 때 관련 없는 subagent 대화로 연결되던 문제를 수정함
- persisted Edit/Write tool result가 `file_path`를 누락했을 때 `--resume`에서 충돌하던 문제를 수정함
- 5분으로 하드코딩된 요청 timeout이 `API_TIMEOUT_MS`와 관계없이 느린 backends(로컬 LLMs, extended thinking, 느린 gateways)를 중단하던 문제를 수정함
- `permissions.deny` 규칙이 PreToolUse hook의 `permissionDecision: "ask"`를 override하지 못하던 문제를 수정함 — 이전에 hook이 거부를 프롬프트로 다운그레이드할 수 있었음
- `--setting-sources`가 `user` 없이 background cleanup이 `cleanupPeriodDays`를 무시하고 30일보다 오래된 대화 기록을 삭제하던 문제를 수정함
- Bedrock SigV4 인증이 `ANTHROPIC_AUTH_TOKEN`, `apiKeyHelper`, 또는 `ANTHROPIC_CUSTOM_HEADERS`가 Authorization header를 설정할 때 403으로 실패하던 문제를 수정함
- `claude -w <name>`이 이전 세션의 worktree cleanup이 stale directory를 남겨둔 후 "already exists"로 실패하던 문제를 수정함
- subagents가 dynamically-injected servers로부터 MCP tools를 상속하지 않던 문제를 수정함
- 격리된 worktrees에서 실행되는 sub-agents가 자신의 worktree 내부 파일에 대한 Read/Edit 접근이 거부되던 문제를 수정함
- sandboxed Bash 명령이 fresh boot 후 `mktemp: No such file or directory`로 실패하던 문제를 수정함
- `claude mcp serve` tool calls가 `outputSchema`를 검증하는 MCP clients에서 "Tool execution failed"로 실패하던 문제를 수정함
- `RemoteTrigger` tool의 `run` action이 빈 body를 전송하고 server에 의해 거부되던 문제를 수정함
- 여러 `/resume` picker 문제를 수정함: 다른 프로젝트의 세션을 숨기는 좁은 기본 보기, Windows Terminal에서의 접근 불가능한 preview, worktrees의 잘못된 cwd, 세션 미발견 오류가 stderr에 표면화되지 않음, terminal title이 설정되지 않음, 그리고 resume hint가 프롬프트 입력과 겹침
- Grep tool ENOENT를 수정하여 임베드된 ripgrep binary path가 오래되었을 때(VS Code extension auto-update, macOS App Translocation) 발생하는 문제; 이제 system `rg`로 fallback하고 세션 중 자체 복구함
- `/btw`가 매번 사용할 때마다 전체 대화의 복사본을 디스크에 작성하던 문제를 수정함
- `/context` Free space 및 Messages breakdown이 header percentage와 불일치하던 문제를 수정함
- 여러 plugin 문제를 수정함: 중복된 `name:` frontmatter로 slash commands가 잘못된 플러그인으로 해석되는 문제, `/plugin update`가 `ENAMETOOLONG`으로 실패하는 문제, Discover가 이미 설치된 플러그인을 표시하는 문제, directory-source 플러그인이 stale version cache에서 로드되는 문제, 그리고 skills가 `context: fork` 및 `agent` frontmatter 필드를 준수하지 않는 문제
- `/mcp` 메뉴가 `headersHelper`로 구성된 MCP servers에 대해 OAuth 특정 작업을 제공하던 문제를 수정함; Reconnect가 대신 helper script를 다시 호출하기 위해 제공되도록 함
- `ctrl+]`, `ctrl+\`, 및 `ctrl+^` keybindings이 raw C0 control bytes를 전송하는 terminals(Terminal.app, 기본 iTerm2, xterm)에서 발화하지 않던 문제를 수정함
- `/login` OAuth URL이 깔끔한 마우스 선택을 방해하는 padding으로 렌더링되던 문제를 수정함
- 렌더링 문제를 수정함: 보이는 영역 위의 콘텐츠가 변경될 때 non-fullscreen mode의 깜빡임, 긴 세션 중 non-fullscreen mode의 terminal scrollback이 지워지는 문제, 그리고 mouse-scroll escape sequences가 때때로 프롬프트로 텍스트로 새나가는 문제
- `settings.json` env values가 문자열 대신 숫자일 때 충돌하던 문제를 수정함
- in-app settings writes(예: `/add-dir --remember`, `/config`)가 in-memory snapshot을 새로 고치지 않아 제거된 디렉터리가 세션 중 revoke되지 않던 문제를 수정함
- custom keybindings(`~/.claude/keybindings.json`)이 Bedrock, Vertex, 및 기타 third-party providers에서 로드되지 않던 문제를 수정함
- `claude --continue -p`가 `-p` 또는 SDK에 의해 생성된 세션을 올바르게 계속하지 못하던 문제를 수정함
- 여러 Remote Control 문제를 수정함: 세션 충돌 시 제거되는 worktrees, connection failures가 transcript에 유지되지 않는 문제, brief mode의 로컬 세션에 대한 거짓된 "Disconnected" indicator, 그리고 SSH를 통해 `CLAUDE_CODE_ORGANIZATION_UUID`만 설정되었을 때 `/remote-control`이 실패하는 문제
- `/insights`가 때때로 response에서 report file 링크를 생략하던 문제를 수정함
- [VSCode] 마지막 editor tab이 닫힐 때 chat input 아래의 파일 attachment가 지워지지 않던 문제를 수정함
